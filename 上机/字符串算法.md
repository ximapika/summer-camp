# 字符串算法

---

## 1. KMP 算法（Knuth-Morris-Pratt）

### 核心思想

KMP 算法用于在文本串 text 中查找模式串 pattern 的所有出现位置。核心在于利用**部分匹配表**（next 数组 / 前缀函数）避免回溯文本指针。

- **next[i]**（前缀函数 pi[i]）：pattern[0..i] 的最长**真前缀 = 真后缀**的长度。
- 匹配失败时，模式指针跳到 `next[j-1]` 位置而非重头开始，利用已匹配的部分信息。

### 时间复杂度

- 构建 next 数组：O(m)
- 匹配过程：O(n)
- 总体：O(n + m)

### 代码模板

```cpp
// 构建前缀函数（next 数组）
vector<int> buildNext(const string& pattern) {
    int m = pattern.size();
    vector<int> next(m, 0);
    for (int i = 1, j = 0; i < m; i++) {
        while (j > 0 && pattern[i] != pattern[j])
            j = next[j - 1];
        if (pattern[i] == pattern[j]) j++;
        next[i] = j;
    }
    return next;
}

// KMP 匹配，返回所有匹配位置（0-indexed）
vector<int> kmp(const string& text, const string& pattern) {
    int n = text.size(), m = pattern.size();
    vector<int> next = buildNext(pattern);
    vector<int> res;
    for (int i = 0, j = 0; i < n; i++) {
        while (j > 0 && text[i] != pattern[j])
            j = next[j - 1];
        if (text[i] == pattern[j]) j++;
        if (j == m) {
            res.push_back(i - m + 1);
            j = next[j - 1];
        }
    }
    return res;
}
```

### next 数组的其他应用

1. **求字符串的最小周期**：若 `n % (n - next[n-1]) == 0`，则最小周期为 `n - next[n-1]`。
2. **求所有周期**：从 next[n-1] 不断跳 next，每个位置对应一个周期。
3. **判断是否为某个串的重复拼接**：等价于最小周期能整除串长。

### 经典例题

**字符串匹配（洛谷 P3375）**
- 给定文本串 s 和模式串 p，输出 p 在 s 中所有出现的起始位置，以及 p 的 next 数组。直接套用 KMP 模板。

**重复子串（LeetCode 459）**
- 判断字符串 s 是否可以由某个子串重复多次构成。利用 next 数组：`len = n - next[n-1]`，若 `n % len == 0` 且 `len < n`，则是。

## 2. 字符串哈希

### 核心思想

将字符串映射为整数（哈希值），实现 O(1) 子串比较。采用多项式哈希：

```
h(s[0..n-1]) = s[0] * B^(n-1) + s[1] * B^(n-2) + ... + s[n-1]
```

其中 B 为基数（常取 131 或 13331），运算在模大素数下进行。预处理前缀哈希后，O(1) 求任意子串哈希值。

### 时间复杂度

- 预处理：O(n)
- 查询子串哈希：O(1)

### 2.1 单哈希模板

```cpp
typedef unsigned long long ull;
const ull BASE = 131;
const ull MOD = 1e9 + 7;

struct StringHash {
    vector<ull> h, pw;
    
    void build(const string& s) {
        int n = s.size();
        h.resize(n + 1);
        pw.resize(n + 1);
        h[0] = 0; pw[0] = 1;
        for (int i = 0; i < n; i++) {
            h[i + 1] = (h[i] * BASE + s[i]) % MOD;
            pw[i + 1] = pw[i] * BASE % MOD;
        }
    }
    
    ull get(int l, int r) { // 0-indexed [l, r]
        return (h[r + 1] - h[l] * pw[r - l + 1] % MOD + MOD) % MOD;
    }
};
```

### 2.2 双哈希防冲突

**核心思想**：单哈希存在冲突风险（生日悖论）。使用两组不同的 (BASE, MOD) 计算哈希值，只有两个值都相同才认为匹配。冲突概率降到约 10^(-18)。

```cpp
const ull BASE1 = 131, MOD1 = 1e9 + 7;
const ull BASE2 = 13331, MOD2 = 1e9 + 9;

struct DoubleHash {
    vector<ull> h1, h2, pw1, pw2;
    
    void build(const string& s) {
        int n = s.size();
        h1.resize(n + 1); h2.resize(n + 1);
        pw1.resize(n + 1); pw2.resize(n + 1);
        h1[0] = h2[0] = 0;
        pw1[0] = pw2[0] = 1;
        for (int i = 0; i < n; i++) {
            h1[i+1] = (h1[i] * BASE1 + s[i]) % MOD1;
            h2[i+1] = (h2[i] * BASE2 + s[i]) % MOD2;
            pw1[i+1] = pw1[i] * BASE1 % MOD1;
            pw2[i+1] = pw2[i] * BASE2 % MOD2;
        }
    }
    
    pair<ull, ull> get(int l, int r) { // 0-indexed [l, r]
        ull v1 = (h1[r+1] - h1[l] * pw1[r-l+1] % MOD1 + MOD1) % MOD1;
        ull v2 = (h2[r+1] - h2[l] * pw2[r-l+1] % MOD2 + MOD2) % MOD2;
        return {v1, v2};
    }
};
```

### 2.3 自然溢出哈希

利用 `unsigned long long` 的自然溢出替代取模，速度更快但安全性略低（容易被构造冲突数据 hack）。

```cpp
struct NaturalHash {
    vector<ull> h, pw;
    static const ull BASE = 131;
    
    void build(const string& s) {
        int n = s.size();
        h.resize(n + 1); pw.resize(n + 1);
        h[0] = 0; pw[0] = 1;
        for (int i = 0; i < n; i++) {
            h[i+1] = h[i] * BASE + s[i];
            pw[i+1] = pw[i] * BASE;
        }
    }
    
    ull get(int l, int r) {
        return h[r+1] - h[l] * pw[r-l+1];
    }
};
```

### 经典例题

**最长公共子串**
- 给定两个字符串，求最长公共子串。二分答案长度 + 哈希判断。O(n log n)。
- 思路：二分长度 len，将第一个串所有长度为 len 的子串哈希值放入 set，然后检查第二个串是否有长度为 len 的子串哈希值在 set 中。

**判断子串是否回文**
- 预处理正串和反串的哈希。子串 s[l..r] 是回文当且仅当正串 hash(l, r) == 反串 hash(n-1-r, n-1-l)。

## 3. Manacher 算法

### 核心思想

O(n) 求字符串的**所有回文子串信息**，特别是**最长回文子串**。

关键技巧：
1. **预处理**：在字符间插入特殊字符 `#`，将奇偶长度统一处理。例如 `"abc"` 变为 `"#a#b#c#"`。
2. 维护当前已知的**最右回文边界** `r` 和对应的**中心** `c`。
3. 对每个位置 i，利用已知的对称性快速初始化 `p[i]`，然后暴力扩展。

`p[i]` 表示以位置 i 为中心的最长回文半径（包括中心自身）。在变换后的串中，`p[i] - 1` 就是原始串中以该位置为中心的最长回文串长度。

### 时间复杂度

O(n)（每个字符最多被扩展一次）

### 代码模板

```cpp
string manacherPreprocess(const string& s) {
    string t = "#";
    for (char c : s) { t += c; t += '#'; }
    return t;
}

// 返回最长回文子串的长度
int manacher(const string& s) {
    string t = manacherPreprocess(s);
    int n = t.size();
    vector<int> p(n, 0);
    int c = 0, r = 0; // 当前最右回文的中心和右边界
    int maxLen = 0;
    
    for (int i = 0; i < n; i++) {
        int mirror = 2 * c - i; // i 关于 c 的对称点
        if (i < r)
            p[i] = min(r - i, p[mirror]);
        // 尝试扩展
        while (i + p[i] + 1 < n && i - p[i] - 1 >= 0 
               && t[i + p[i] + 1] == t[i - p[i] - 1])
            p[i]++;
        // 更新最右边界
        if (i + p[i] > r) {
            c = i;
            r = i + p[i];
        }
        maxLen = max(maxLen, p[i]);
    }
    return maxLen; // 这就是原始串中最长回文子串的长度
}

// 返回最长回文子串本身
string longestPalindrome(const string& s) {
    string t = manacherPreprocess(s);
    int n = t.size();
    vector<int> p(n, 0);
    int c = 0, r = 0;
    int maxLen = 0, center = 0;
    
    for (int i = 0; i < n; i++) {
        int mirror = 2 * c - i;
        if (i < r) p[i] = min(r - i, p[mirror]);
        while (i + p[i] + 1 < n && i - p[i] - 1 >= 0 
               && t[i + p[i] + 1] == t[i - p[i] - 1])
            p[i]++;
        if (i + p[i] > r) { c = i; r = i + p[i]; }
        if (p[i] > maxLen) { maxLen = p[i]; center = i; }
    }
    // 在原始串中的起始位置
    int start = (center - maxLen) / 2;
    return s.substr(start, maxLen);
}
```

### p 数组的含义

- 变换后位置 i 对应原串位置 `(i - 1) / 2`（仅当 t[i] != '#'）。
- `p[i]` 是变换串中以 i 为中心的回文半径。
- `p[i]` 等于原串中以该位置为中心的最长回文串长度。

### 经典例题

**最长回文子串（LeetCode 5）**
- 给定字符串 s，找到其中最长的回文子串。Manacher O(n) 是最优解。

**回文子串计数（LeetCode 647）**
- 统计所有回文子串的数量。可以用 Manacher 的 p 数组：每个中心的回文串数量为 `ceil(p[i] / 2)`。

## 4. 字典树（Trie）

### 核心思想

Trie（前缀树）是一种多叉树结构，每条边对应一个字符，从根到节点的路径表示一个字符串前缀。共享前缀的字符串共享路径，节省空间。

### 时间复杂度

- 插入：O(L)，L 为字符串长度
- 查询：O(L)
- 空间：O(Sigma * N)，Sigma 为字符集大小，N 为总节点数

### 代码模板

```cpp
const int MAXN = 500005; // 最大节点数（所有字符串总长度之和）
int ch[MAXN][26]; // ch[u][c] 表示节点 u 经过字符 c 到达的子节点
int cnt[MAXN];    // 以该节点为结尾的字符串数量
int prefCnt[MAXN]; // 以该节点为前缀的字符串数量
int tot = 0;       // 当前节点总数

void init() {
    tot = 0;
    memset(ch[0], 0, sizeof(ch[0]));
    cnt[0] = prefCnt[0] = 0;
}

int newNode() {
    ++tot;
    memset(ch[tot], 0, sizeof(ch[tot]));
    cnt[tot] = prefCnt[tot] = 0;
    return tot;
}

void insert(const string& s) {
    int u = 0;
    for (char c : s) {
        int v = c - 'a';
        if (!ch[u][v]) ch[u][v] = newNode();
        u = ch[u][v];
        prefCnt[u]++;
    }
    cnt[u]++;
}

// 精确查询：字符串 s 是否存在
int search(const string& s) {
    int u = 0;
    for (char c : s) {
        int v = c - 'a';
        if (!ch[u][v]) return 0;
        u = ch[u][v];
    }
    return cnt[u]; // 返回出现次数
}

// 前缀查询：有多少字符串以 s 为前缀
int countPrefix(const string& s) {
    int u = 0;
    for (char c : s) {
        int v = c - 'a';
        if (!ch[u][v]) return 0;
        u = ch[u][v];
    }
    return prefCnt[u];
}

// 删除一个字符串（需要保证字符串存在）
void erase(const string& s) {
    int u = 0;
    for (char c : s) {
        int v = c - 'a';
        u = ch[u][v];
        prefCnt[u]--;
    }
    cnt[u]--;
}
```

### 经典例题

**实现 Trie（LeetCode 208）**
- 实现 insert、search、startsWith 三个操作。直接使用上述模板。

**单词搜索 II（LeetCode 212）**
- 在二维字符矩阵中搜索多个单词。将所有单词插入 Trie，然后在矩阵中 DFS，沿 Trie 路径搜索。

**最长公共前缀**
- 多个字符串的最长公共前缀。插入 Trie 后，从根开始走到第一个分叉或结束节点。

## 5. AC 自动机（Aho-Corasick Automaton）

### 核心思想

AC 自动机是**多模式串匹配**算法，结合了 Trie 和 KMP 的思想：
1. 将所有模式串插入 Trie。
2. 在 Trie 上构建**失败指针**（fail pointer），类似 KMP 的 next 数组。
3. 在文本串上沿 Trie 匹配，失败时跳转 fail 指针，避免从头匹配。

**fail[u]**：从根到节点 u 的字符串的最长**真后缀**，且该后缀是 Trie 中某个节点对应的前缀。

### 时间复杂度

- 建 Trie：O(sum of |pattern_i|)
- 建 fail 指针：O(sum of |pattern_i|)
- 匹配：O(|text| + 匹配次数)

### 代码模板

```cpp
const int MAXN = 500005;
int ch[MAXN][26], fail[MAXN], val[MAXN];
int tot = 0;

void init() {
    tot = 0;
    memset(ch[0], 0, sizeof(ch[0]));
    val[0] = 0;
}

void insert(const string& s, int id) {
    int u = 0;
    for (char c : s) {
        int v = c - 'a';
        if (!ch[u][v]) {
            ch[u][v] = ++tot;
            memset(ch[tot], 0, sizeof(ch[tot]));
            val[tot] = 0;
        }
        u = ch[u][v];
    }
    val[u]++; // 标记模式串结尾（或记录 id）
}

void buildFail() {
    queue<int> q;
    fail[0] = 0;
    for (int c = 0; c < 26; c++) {
        if (ch[0][c]) {
            fail[ch[0][c]] = 0;
            q.push(ch[0][c]);
        }
    }
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int c = 0; c < 26; c++) {
            if (ch[u][c]) {
                fail[ch[u][c]] = ch[fail[u]][c];
                q.push(ch[u][c]);
            } else {
                ch[u][c] = ch[fail[u]][c]; // 路径压缩，补全 Trie 的转移
            }
        }
    }
}

// 在文本串 text 中查找所有模式串的出现次数
int query(const string& text) {
    int u = 0, res = 0;
    for (char c : text) {
        u = ch[u][c - 'a'];
        // 沿 fail 链统计所有匹配的模式串
        for (int t = u; t && val[t] != -1; t = fail[t]) {
            res += val[t];
            val[t] = -1; // 避免重复计数（若需多次匹配则不标记）
        }
    }
    return res;
}
```

### 构建过程详解

1. **BFS 构建 fail 指针**：按层次遍历 Trie，对每个节点 u 的每个子节点 `ch[u][c]`：
   - 若存在：`fail[ch[u][c]] = ch[fail[u]][c]`（跳到父节点的 fail 对应的 c 子节点）
   - 若不存在：`ch[u][c] = ch[fail[u]][c]`（路径压缩，直接指向 fail 链上的对应节点）

2. 路径压缩后的 `ch` 数组形成了一个**完整的自动机**，任何字符的转移都有定义，无需额外的 fail 跳转。

### 经典例题

**多模式匹配（洛谷 P3808 / P3796）**
- P3808：给定 n 个模式串和一个文本串，求有多少个模式串在文本串中出现过。
- P3796：求出现次数最多的模式串。在 query 中不标记 -1，改为累加次数。

**关键词过滤/敏感词检测**
- 给定一组敏感词和一段文本，找出文本中包含的所有敏感词。典型的 AC 自动机应用。

### AC 自动机 vs 其他算法

| 算法 | 模式串数量 | 时间复杂度 | 适用场景 |
|------|-----------|-----------|---------|
| KMP | 单个 | O(n + m) | 单模式串匹配 |
| 字符串哈希 | 单个/少量 | O(n + m) | 子串判等、匹配 |
| AC 自动机 | 多个 | O(n + sum(m_i)) | 多模式串匹配 |

---

## 总结

| 算法 | 核心功能 | 时间复杂度 | 关键点 |
|------|---------|-----------|--------|
| KMP | 单模式串匹配 | O(n + m) | next 数组（前缀函数） |
| 字符串哈希 | 子串快速比较 | 预处理 O(n)，查询 O(1) | 多项式哈希，双哈希防冲突 |
| Manacher | 最长回文子串 | O(n) | 插入分隔符统一奇偶，维护最右边界 |
| Trie | 字符串集合操作 | O(L) 每次操作 | 共享前缀，多叉树 |
| AC 自动机 | 多模式串匹配 | O(n + sum(m_i)) | Trie + fail 指针（类 KMP） |

### 选择建议

- **单模式串匹配** -> KMP
- **需要 O(1) 子串比较** -> 字符串哈希
- **最长回文子串** -> Manacher
- **多字符串前缀操作** -> Trie
- **多模式串同时匹配** -> AC 自动机

## 经典例题精讲

### 例题 1：LC 28 - 找出字符串中第一个匹配项的下标

**题意**：给定字符串 haystack 和 needle，返回 needle 在 haystack 中第一次出现的下标，不存在则返回 -1。

**思路**：KMP 模板题。构建 needle 的 next 数组（前缀函数），然后在 haystack 上匹配，找到第一个完整匹配即返回。

```cpp
class Solution {
public:
    int strStr(string haystack, string needle) {
        int n = haystack.size(), m = needle.size();
        if (m == 0) return 0;

        // 构建 next 数组
        vector<int> next(m, 0);
        for (int i = 1, j = 0; i < m; i++) {
            while (j > 0 && needle[i] != needle[j])
                j = next[j - 1];
            if (needle[i] == needle[j]) j++;
            next[i] = j;
        }

        // KMP 匹配
        for (int i = 0, j = 0; i < n; i++) {
            while (j > 0 && haystack[i] != needle[j])
                j = next[j - 1];
            if (haystack[i] == needle[j]) j++;
            if (j == m) return i - m + 1;
        }
        return -1;
    }
};
```

### 例题 2：LC 5 - 最长回文子串

**题意**：给定字符串 s，找出其中最长的回文子串。

**思路**：中心扩展法 O(n^2)：枚举每个中心（奇数和偶数长度分别处理），向两侧扩展直到不匹配。也可用 Manacher 做到 O(n)，此处给出中心扩展的写法。

```cpp
class Solution {
public:
    string longestPalindrome(string s) {
        int n = s.size();
        int start = 0, maxLen = 1;

        auto expand = [&](int l, int r) {
            while (l >= 0 && r < n && s[l] == s[r]) {
                if (r - l + 1 > maxLen) {
                    start = l;
                    maxLen = r - l + 1;
                }
                l--;
                r++;
            }
        };

        for (int i = 0; i < n; i++) {
            expand(i, i);     // 奇数长度
            expand(i, i + 1); // 偶数长度
        }
        return s.substr(start, maxLen);
    }
};
```

### 例题 3：LC 76 - 最小覆盖子串

**题意**：给定字符串 s 和 t，找出 s 中包含 t 所有字符（含重复）的最短子串。不存在则返回空串。

**思路**：滑动窗口。用计数器记录 t 中每个字符需要的个数，右指针扩展直到覆盖所有字符，然后左指针收缩寻找最短窗口。用 required 记录还差多少种字符未满足，满足时尝试收缩。

```cpp
class Solution {
public:
    string minWindow(string s, string t) {
        if (s.size() < t.size()) return "";

        unordered_map<char, int> need, window;
        for (char c : t) need[c]++;

        int required = need.size(); // 需要满足的不同字符种数
        int formed = 0;             // 当前窗口已满足的种数

        int left = 0, ansL = 0, ansLen = INT_MAX;

        for (int right = 0; right < (int)s.size(); right++) {
            char c = s[right];
            window[c]++;
            if (need.count(c) && window[c] == need[c])
                formed++;

            while (formed == required) {
                // 更新答案
                if (right - left + 1 < ansLen) {
                    ansLen = right - left + 1;
                    ansL = left;
                }
                // 收缩左边界
                char lc = s[left];
                window[lc]--;
                if (need.count(lc) && window[lc] < need[lc])
                    formed--;
                left++;
            }
        }
        return ansLen == INT_MAX ? "" : s.substr(ansL, ansLen);
    }
};
