# 计算机基础问答

本文件用于面试前快速过高频题。回答应先给结论，再补关键机制，最后给一两个容易加分的细节。

---

## 一、操作系统

### 1. 进程和线程有什么区别？

进程是资源分配的基本单位，线程是 CPU 调度的基本单位。同一进程内的线程共享地址空间、文件描述符等资源，但有各自的栈、寄存器和线程上下文。

补充：进程切换通常比线程切换更重，因为涉及地址空间等资源切换；线程共享内存方便通信，但也带来竞态条件，需要锁、信号量或条件变量同步。

### 2. 用户态和内核态为什么要区分？

目的是保护系统资源。用户态程序不能直接执行特权指令或访问内核数据，只能通过系统调用、中断、异常进入内核态，由操作系统代为完成敏感操作。

### 3. 死锁的必要条件是什么？如何处理？

四个必要条件是互斥、占有且等待、不可抢占、循环等待。处理策略包括死锁预防、死锁避免、死锁检测与恢复，以及在低风险场景下忽略。

加分点：银行家算法属于死锁避免，通过判断资源分配后是否仍处于安全状态来决定是否分配。

### 4. 虚拟内存有什么作用？

虚拟内存为进程提供独立、连续的地址空间，屏蔽物理内存细节；通过分页和换页机制，让程序可使用大于物理内存的地址空间；也提供进程隔离和内存保护。

### 5. 缺页中断的处理流程是什么？

CPU 访问页表发现目标页不在内存，触发缺页中断；操作系统检查访问是否合法；若合法，从磁盘调入页面，必要时按页面置换算法淘汰旧页；更新页表后恢复执行。

## 二、计算机网络

### 1. TCP 和 UDP 有什么区别？

TCP 面向连接、可靠、有序、字节流，提供流量控制和拥塞控制；UDP 无连接、不保证可靠和有序，报文边界保留，开销小、延迟低。

典型场景：TCP 用于 HTTP/HTTPS、文件传输；UDP 用于 DNS、实时音视频、QUIC/HTTP/3。

### 2. TCP 为什么需要三次握手？

三次握手用于确认双方收发能力正常，并同步初始序列号。两次握手无法避免历史连接请求导致服务端误建连接，也不能让客户端确认服务端的发送能力。

### 3. TIME_WAIT 为什么存在？

TIME_WAIT 主要有两个作用：保证最后一个 ACK 丢失时能够重传，确保连接可靠关闭；等待旧连接中的延迟报文在网络中消失，避免影响后续相同四元组连接。

### 4. HTTP 和 HTTPS 的区别是什么？

HTTPS = HTTP + TLS。TLS 提供机密性、完整性和身份认证，防止窃听、篡改和中间人攻击。证书用于证明服务端身份，对称密钥通常通过非对称密钥交换协商得到。

### 5. DNS 查询过程是什么？

客户端先查本地缓存和 hosts；若未命中，向本地 DNS 服务器查询。本地 DNS 可能递归查询根域名服务器、顶级域名服务器、权威域名服务器，最终返回 IP 并缓存结果。

## 三、数据库

### 1. 事务的 ACID 是什么？

- 原子性：事务要么全做，要么全不做。
- 一致性：事务前后数据库满足约束。
- 隔离性：并发事务之间互不干扰到指定程度。
- 持久性：提交后的修改不会因故障丢失。

### 2. 脏读、不可重复读、幻读有什么区别？

脏读是读到其他事务未提交的数据；不可重复读是同一事务内两次读同一行结果不同；幻读是同一事务内两次按条件查询得到的行集合不同。

### 3. B+ 树为什么适合数据库索引？

B+ 树高度低，磁盘 I/O 次数少；非叶子节点只存 key，扇出大；叶子节点按 key 有序并通过链表相连，适合范围查询。

### 4. MVCC 解决什么问题？

MVCC 通过保存数据的多个版本，让读操作不阻塞写操作、写操作不阻塞普通快照读，从而提高并发性能。它通常依赖版本链、事务 ID、Read View 和 undo log。

### 5. 最左前缀原则是什么？

对于联合索引 `(a,b,c)`，查询条件需要从最左列开始连续匹配才能充分利用索引。例如 `a=? and b=?` 可用，单独 `b=?` 通常不能有效使用该联合索引。

## 四、数据结构与算法

### 1. 快排、归并、堆排序怎么对比？

快排平均 `O(n log n)`、最坏 `O(n^2)`，原地但不稳定；归并排序稳定，时间稳定 `O(n log n)`，需要额外空间；堆排序原地、最坏 `O(n log n)`，但常数较大且不稳定。

### 2. 哈希表为什么平均 O(1)？

哈希函数把 key 映射到数组位置，理想情况下元素均匀分布，每个桶长度很短，因此查找、插入、删除平均为 `O(1)`。最坏情况下大量冲突会退化。

### 3. BFS 和 DFS 的区别是什么？

BFS 按层扩展，适合无权图最短路；DFS 沿路径深入，适合回溯、连通性、拓扑相关问题。BFS 通常需要队列，DFS 可递归或显式栈实现。

### 4. Dijkstra 为什么不能处理负权边？

Dijkstra 每次确定当前距离最小的未确定点，并认为其最短路已确定。负权边可能在之后把这个点的距离继续变小，破坏贪心前提。

## 五、计算机组成原理

### 1. 补码为什么常用？

补码把减法统一为加法，硬件实现简单；同时只有一个零，表示范围比原码/反码更方便。n 位补码范围是 `[-2^(n-1), 2^(n-1)-1]`。

### 2. Cache 的局部性原理是什么？

时间局部性指最近访问的数据未来可能再次访问；空间局部性指访问某个地址后，附近地址也可能被访问。Cache 利用局部性减少平均访存时间。

### 3. 流水线有哪些冲突？

结构冲突是硬件资源竞争；数据冲突是指令间数据依赖；控制冲突来自分支跳转。常用解决方法包括资源扩展、转发、暂停、分支预测等。

## 六、Linux 与工具

### 1. 硬链接和软链接有什么区别？

硬链接指向同一个 inode，删除原文件不影响硬链接访问，通常不能跨文件系统、不能链接目录。软链接保存目标路径，类似快捷方式，可跨文件系统，但目标删除后会失效。

### 2. 如何查看端口占用？

常用命令：

```bash
ss -lntp
lsof -i :8080
netstat -tulnp
```

`ss` 是较新的网络状态查看工具，通常比 `netstat` 更推荐。

### 3. Git merge 和 rebase 有什么区别？

merge 保留分支合并历史，会产生合并提交；rebase 把当前分支提交“搬到”目标分支之后，历史更线性，但会改写提交历史。公共分支上应谨慎 rebase。

## 七、操作系统（补充）

### 6. 进程间通信有哪些方式？

主要有管道（匿名管道/命名管道）、消息队列、共享内存、信号量、信号和 Socket。管道适合父子进程间的简单通信，共享内存效率最高但需要同步，Socket 可以跨主机通信。

### 7. 进程调度算法有哪些？如何选择？

常见有 FCFS、SJF、SRTF、RR（时间片轮转）、优先级调度、MLFQ（多级反馈队列）。交互式系统常用 RR 或 MLFQ 保证响应时间，批处理系统可用 SJF 提高吞吐量。Linux CFS 使用虚拟运行时间实现公平调度。

### 8. 什么是 Copy-on-Write？

fork 创建子进程时，父子进程共享相同的物理页，标记为只读。任一进程写入时触发缺页中断，操作系统才复制该页。好处是避免 fork 时的大量无用拷贝，特别是 fork 后紧接 exec 的场景。

### 9. 页面置换算法有哪些？

OPT（理论最优，不可实现）、FIFO（可能 Belady 异常）、LRU（性能好但硬件开销大）、Clock（LRU 近似，用引用位）、改进 Clock（同时考虑引用位和修改位）。实际系统多用 Clock 或其变种。

### 10. 协程和线程有什么区别？

线程由操作系统调度（内核态切换），协程由用户态调度（协作式切换）。协程的切换开销远小于线程，适合 I/O 密集型场景。协程不能利用多核并行（单线程内），需要配合多线程使用。Go 的 goroutine 是 M:N 模型（多协程映射到多线程），兼顾了并发和并行。

## 八、计算机网络（补充）

### 6. Cookie、Session、Token 有什么区别？

Cookie 存储在客户端浏览器中，随每次请求自动发送；Session 存储在服务端，通过 Session ID（通常放在 Cookie 中）关联用户；Token（如 JWT）是无状态的认证令牌，由服务端签发、客户端存储，不需要服务端维护状态。

分布式系统中 Session 需要共享存储（如 Redis），Token 天然支持分布式但无法主动失效。

### 7. TCP 拥塞控制的四个阶段是什么？

1. **慢启动**：cwnd 从 1 MSS 开始，每 RTT 翻倍（指数增长）
2. **拥塞避免**：cwnd 超过 ssthresh 后每 RTT 加 1（线性增长）
3. **快重传**：收到 3 个重复 ACK 立即重传丢失报文
4. **快恢复**：ssthresh = cwnd/2，cwnd = ssthresh + 3，进入拥塞避免（不回到慢启动）

超时则更严重：ssthresh = cwnd/2，cwnd = 1，回到慢启动。

### 8. HTTP/1.1、HTTP/2、HTTP/3 的主要区别？

HTTP/1.1：持久连接、管道化（但队头阻塞严重）。HTTP/2：二进制帧、多路复用（一个 TCP 连接上并行传输多个请求）、头部压缩（HPACK）、服务端推送，但 TCP 层丢包仍会阻塞所有流。HTTP/3：基于 QUIC（UDP），彻底解决 TCP 队头阻塞，连接建立更快（0-RTT），内建加密。

### 9. ARP 协议的工作原理？

主机知道目标 IP 但不知道 MAC 地址时，广播 ARP 请求包（目标 MAC 全 F）。目标主机收到后单播回复自己的 MAC 地址。发送方缓存 IP-MAC 映射。ARP 缓存有超时机制。ARP 欺骗是局域网安全的常见威胁。

### 10. HTTPS 的 TLS 握手过程？

TLS 1.3：(1) Client Hello（支持的密码套件、随机数、密钥共享参数）→ (2) Server Hello（选择的套件、证书、随机数、密钥共享参数）→ (3) 客户端验证证书 → (4) 双方通过 ECDHE 计算会话密钥 → (5) 后续用对称密钥加密通信。整个过程只需 1-RTT。

## 九、数据库（补充）

### 6. InnoDB 和 MyISAM 有什么区别？

InnoDB 支持事务、行级锁、外键、MVCC，使用聚簇索引（数据和主键索引一起存储）。MyISAM 不支持事务和行锁，使用表级锁，但全文索引和 COUNT(*) 更快。MySQL 5.5 后默认引擎是 InnoDB。

### 7. 索引失效的常见情况有哪些？

索引列上使用函数或运算（如 WHERE YEAR(date) = 2025）、隐式类型转换、LIKE 以通配符开头（如 '%abc'）、违反最左前缀原则、OR 条件中有非索引列、NOT/!=/<> 操作、数据量太小优化器选择全表扫描。

### 8. 什么是回表？什么是覆盖索引？

非聚簇索引的叶子节点存储的是主键值，查到主键后还需要回到聚簇索引查完整数据，这叫**回表**。如果查询的字段全部在非聚簇索引中，无需回表，这叫**覆盖索引**。覆盖索引能显著提升查询性能。

### 9. 如何优化慢查询？

先用 EXPLAIN 分析执行计划，看是否走了索引、扫描行数。常见优化：添加合适的索引、避免 SELECT *、减少子查询改用 JOIN、分页查询用游标代替 OFFSET、大表考虑分区或分库分表、缓存热点查询。

### 10. Redis 有哪些数据结构？各自适用什么场景？

| 数据结构 | 适用场景 |
|----------|----------|
| String | 缓存、计数器、分布式锁 |
| Hash | 对象属性存储（用户信息） |
| List | 消息队列、最新消息列表 |
| Set | 去重、交集/并集运算（共同好友） |
| Sorted Set (ZSet) | 排行榜、延迟队列 |

底层实现：String 用 SDS，Hash 小时用 ziplist 大时用 hashtable，ZSet 用 skiplist + hashtable。

## 十、设计模式与系统设计

### 1. 单例模式如何实现？有什么问题？

懒汉式（延迟初始化，需要考虑线程安全）和饿汉式（类加载时就创建，天然线程安全）。C++ 中推荐局部静态变量（Meyers' Singleton，C++11 后线程安全）。Java 中推荐枚举或静态内部类。问题：全局状态难以测试、违反单一职责、隐式依赖。

### 2. 工厂模式和策略模式的区别？

工厂模式关注**创建什么对象**（封装对象的创建过程），策略模式关注**使用什么算法**（封装可互换的行为）。工厂返回的是不同类型的对象，策略传入的是不同的行为实现。

### 3. 观察者模式的应用场景？

事件驱动系统、MVC 架构中 Model 通知 View 更新、消息订阅发布、GUI 事件处理。核心思想是解耦发布者和订阅者。缺点是如果观察者太多或通知链太长，可能导致性能问题或难以调试。

### 4. 如果让你设计一个短链接系统，怎么做？

核心是长 URL → 短码的映射。方案：(1) 用分布式 ID 生成器生成唯一 ID，转为 62 进制字符串作为短码；(2) 存入数据库（短码→长URL）；(3) 访问短链时查表 302 重定向。优化：Redis 缓存热点映射、布隆过滤器判断是否已存在、按日期分表。

### 5. 如果让你设计一个秒杀系统，关键点是什么？

核心挑战是高并发下保证库存不超卖。关键点：(1) 前端限流（按钮禁用、验证码）；(2) 后端限流+排队（令牌桶、消息队列削峰）；(3) Redis 预减库存（原子操作 DECR）；(4) 异步下单（MQ + 后台消费）；(5) 数据库乐观锁兜底（UPDATE ... WHERE stock > 0）。

## 十一、算法与复杂度

### 5. 动态规划和贪心有什么区别？

DP 考虑所有子问题的解再做选择，保证全局最优；贪心每步做局部最优选择，不回头。贪心需要证明贪心选择性质（交换论证），DP 需要最优子结构+重叠子问题。能贪心解的问题一般也能 DP，但 DP 能解的不一定能贪心。

### 6. 什么是 NP 完全问题？举例说明。

NP-complete 问题满足两个条件：(1) 属于 NP（给定解可在多项式时间验证）；(2) NP-hard（所有 NP 问题可多项式规约到它）。例如 SAT、Hamilton 回路、TSP、图着色、背包问题。目前没有任何 NPC 问题的多项式算法，也没有证明不存在。

### 7. 平均时间复杂度和平摊时间复杂度的区别？

平均时间复杂度是对所有可能输入的期望（涉及概率分布），平摊时间复杂度是对一系列操作的最坏总代价除以操作次数（不涉及概率，是确定性分析）。例如 vector 扩容：单次 push_back 最坏 O(n)，但平摊 O(1)。

## 十二、开放追问

### 1. 如果一个网站访问很慢，你怎么排查？

可以按链路分层排查：DNS 是否慢，TCP/TLS 建连是否慢，服务端处理是否慢，数据库/缓存是否慢，静态资源是否慢，客户端渲染是否慢。工具可用 `ping`、`traceroute`、`curl -w`、浏览器 DevTools、服务端日志和监控。

### 2. 如果项目 QPS 突然升高，你会怎么处理？

先确认瓶颈：CPU、内存、网络、数据库、锁竞争还是外部依赖。短期可限流、扩容、缓存热点、降级非核心功能；长期可优化索引、异步化、拆分服务、引入消息队列和容量规划。

### 3. 如何回答不会的问题？

先明确自己知道的边界，再基于相关知识推理，不要硬编。可以说：“这个细节我不确定，但我知道它和 XX 有关，我会从 YY 角度分析。”
