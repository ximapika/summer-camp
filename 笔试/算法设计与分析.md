## 一、算法复杂度分析

### 1.1 渐近记号

| 记号 | 含义 | 直觉 |
|------|------|------|
| O(g(n)) | 渐近上界 | f(n) 增长不超过 g(n) |
| Ω(g(n)) | 渐近下界 | f(n) 增长不低于 g(n) |
| Θ(g(n)) | 渐近紧界 | f(n) 增长与 g(n) 同阶 |
| o(g(n)) | 严格上界 | f(n) 增长严格慢于 g(n) |
| ω(g(n)) | 严格下界 | f(n) 增长严格快于 g(n) |

### 1.2 主定理 (Master Theorem)

对形如 T(n) = aT(n/b) + f(n) 的递推（a ≥ 1, b > 1）：

设 c_crit = log_b(a)：

| 情况 | 条件 | 结论 |
|------|------|------|
| 1 | f(n) = O(n^{c_crit - ε})，ε > 0 | T(n) = Θ(n^{c_crit}) |
| 2 | f(n) = Θ(n^{c_crit} · log^k n) | T(n) = Θ(n^{c_crit} · log^{k+1} n) |
| 3 | f(n) = Ω(n^{c_crit + ε})，且 af(n/b) ≤ cf(n) | T(n) = Θ(f(n)) |

**常见例子**：
- T(n) = 2T(n/2) + O(n)（归并排序）：c_crit = 1，f(n) = Θ(n)，情况 2 → T(n) = Θ(n log n)
- T(n) = 2T(n/2) + O(1)（二叉树遍历）：c_crit = 1，f(n) = O(n^0)，情况 1 → T(n) = Θ(n)
- T(n) = T(n/2) + O(1)（二分查找）：c_crit = 0，f(n) = Θ(1)，情况 2 → T(n) = Θ(log n)

### 1.3 递推方程求解方法

1. **代入法（substitution）**：猜测解的形式，用数学归纳法验证。
2. **递归树法**：展开递推为树形结构，逐层求和。
3. **主定理**：适用于分治型递推（见上）。
4. **特征方程法**：适用于线性齐次递推 a_n = c₁a_{n-1} + c₂a_{n-2} + ...，求特征根。

## 二、分治法

### 2.1 基本思想

1. **分解（Divide）**：将问题分解为若干子问题
2. **解决（Conquer）**：递归求解子问题
3. **合并（Combine）**：将子问题的解合并为原问题的解

### 2.2 经典分治问题

**归并排序**：T(n) = 2T(n/2) + O(n) = O(n log n)

**快速排序**：
- 最好/平均：T(n) = 2T(n/2) + O(n) = O(n log n)
- 最坏（每次选到极值）：T(n) = T(n-1) + O(n) = O(n²)
- 随机化选 pivot 可使期望复杂度为 O(n log n)

**大整数乘法（Karatsuba）**：

普通乘法 O(n²)。将 n 位数分为高低各 n/2 位：

```
x = x₁ · 10^{n/2} + x₀
y = y₁ · 10^{n/2} + y₀
xy = x₁y₁ · 10^n + [(x₁+x₀)(y₁+y₀) - x₁y₁ - x₀y₀] · 10^{n/2} + x₀y₀
```

只需 3 次 n/2 位乘法：T(n) = 3T(n/2) + O(n) = O(n^{log₂3}) ≈ O(n^{1.585})

**Strassen 矩阵乘法**：将 2×2 分块矩阵乘法从 8 次乘法减少到 7 次。T(n) = 7T(n/2) + O(n²) = O(n^{log₂7}) ≈ O(n^{2.807})

**平面最近点对**：
1. 按 x 坐标排序，分为左右两半
2. 递归求左右两半各自的最近点对距离 δ = min(δ_L, δ_R)
3. 只需检查中线附近宽 2δ 的带状区域中跨两半的点对
4. 带状区域中每个点最多只需与后续 7 个点比较
5. T(n) = 2T(n/2) + O(n log n) = O(n log² n)，预排序可优化到 O(n log n)

**线性时间选择（中位数的中位数）**：
- 将 n 个元素分为 ⌈n/5⌉ 组，每组 5 个
- 求每组的中位数，再递归求这些中位数的中位数作为 pivot
- 保证每次至少淘汰 3n/10 的元素
- T(n) = T(n/5) + T(7n/10) + O(n) = O(n)

### 2.3 分治法的复杂度分析

分治算法的复杂度通常用主定理分析。关键在于：
- a：子问题个数
- b：子问题缩小比例
- f(n)：分解和合并的代价

减少 a（如 Karatsuba 从 4 减到 3）或减少 f(n) 是优化分治的核心手段。

## 三、贪心法的理论基础

### 3.1 贪心选择性质与最优子结构

贪心算法正确的两个条件：

1. **贪心选择性质**：通过局部最优选择能达到全局最优。即存在某个最优解包含贪心所做的第一步选择。
2. **最优子结构**：问题的最优解包含子问题的最优解。

### 3.2 证明贪心正确性

**交换论证法（Exchange Argument）**：
1. 假设存在一个最优解 OPT 与贪心解 G 不同
2. 找到 OPT 与 G 的第一个不同之处
3. 证明可以将 OPT 中的选择"交换"为 G 的选择，结果不变差
4. 重复交换直到 OPT 变为 G，说明 G 也是最优解

**经典例子**：
- **活动选择问题**：按结束时间排序，每次选最早结束的。交换论证：如果最优解的第一个活动结束更晚，换成最早结束的不会冲突，且不减少后续可选活动数。
- **Huffman 编码**：频率最低的两个字符应在最深层且互为兄弟。
- **Kruskal/Prim 最小生成树**：切割性质保证贪心正确。

### 3.3 拟阵 (Matroid)

拟阵是一种抽象结构 M = (S, I)，其中 S 是有限集，I 是独立集族，满足：
1. 空集 ∈ I
2. 遗传性：A ∈ I 且 B ⊆ A → B ∈ I
3. 交换性：A, B ∈ I 且 |A| < |B| → ∃ x ∈ B\A 使 A ∪ {x} ∈ I

**定理**：如果问题可以建模为带权拟阵上的最大权独立集问题，则贪心算法（按权值从大到小加入不破坏独立性的元素）一定得到最优解。

典型拟阵：图拟阵（最小生成树）、均匀拟阵。

## 四、动态规划的理论基础

### 4.1 最优子结构与重叠子问题

DP 的两个前提条件：
1. **最优子结构**：问题的最优解可由子问题的最优解构造
2. **重叠子问题**：不同的子问题共享更小的子问题（否则分治即可）

### 4.2 DP 与贪心的区别

| 对比 | 动态规划 | 贪心 |
|------|----------|------|
| 选择依据 | 基于子问题的最优解做选择 | 基于局部最优做选择 |
| 子问题求解 | 先求解子问题，再做选择 | 先做选择，再求解子问题 |
| 适用条件 | 最优子结构 + 重叠子问题 | 最优子结构 + 贪心选择性质 |
| 时间复杂度 | 通常较高（多项式） | 通常较低 |

### 4.3 DP 与分治的区别

| 对比 | 动态规划 | 分治 |
|------|----------|------|
| 子问题 | 大量重叠 | 互不重叠 |
| 求解方式 | 记表避免重复计算 | 直接递归 |
| 典型 | 背包、LCS、矩阵链乘 | 归并排序、快排 |

### 4.4 矩阵链乘法

给定矩阵链 A₁A₂...Aₙ，不同的加括号方式导致不同的乘法次数。

状态：dp[i][j] = 计算 A_i...A_j 的最少乘法次数

转移：dp[i][j] = min_{i ≤ k < j} { dp[i][k] + dp[k+1][j] + p_{i-1} · p_k · p_j }

复杂度：O(n³) 时间，O(n²) 空间。

### 4.5 最优二叉搜索树

给定 n 个关键字及其查找概率，构造使期望查找代价最小的 BST。

类似矩阵链乘法的区间 DP：dp[i][j] 表示用关键字 i..j 构造最优 BST 的代价。

## 五、回溯法与分支限界

### 5.1 回溯法

回溯法是深度优先搜索 + 剪枝的系统化方法。将解空间组织为树形结构（子集树或排列树），在搜索过程中通过约束条件和限界函数剪去不可能产生最优解的分支。

**解空间树**：
- **子集树**：每个元素"选"或"不选"，共 2^n 个叶结点。如 0-1 背包。
- **排列树**：第 i 层从剩余元素中选一个，共 n! 个叶结点。如 TSP。

**剪枝策略**：
- **约束剪枝**：当前部分解违反约束条件，不再扩展
- **限界剪枝**：当前部分解的上界（最优估计）不优于已知最优解，不再扩展

### 5.2 分支限界法

分支限界法用**广度优先**或**最优优先**搜索解空间树，每个结点只有一次成为活结点的机会。

**与回溯法的核心区别**：

| 对比 | 回溯法 | 分支限界法 |
|------|--------|-----------|
| 搜索方式 | DFS | BFS 或 优先队列 |
| 目标 | 找所有可行解或最优解 | 找一个最优解 |
| 活结点策略 | 一次只扩展一个活结点（栈） | 可一次扩展多个活结点（队列/堆） |
| 剪枝方式 | 约束 + 限界 | 限界为主（bound function） |

**分支限界的类型**：
- **FIFO 分支限界**：用队列，按 BFS 顺序扩展
- **优先队列分支限界（LC 分支限界）**：用优先队列，每次扩展限界值最优的结点

**经典应用**：
- **0-1 背包的分支限界**：
  - 将物品按单位价值排序
  - 上界函数：当前价值 + 剩余容量按单位价值贪心装入的价值
  - 用最大堆按上界排序扩展
- **TSP 的分支限界**：
  - 下界函数：当前路径长度 + 剩余城市最小出边之和
  - 用最小堆按下界排序扩展

## 六、平摊分析 (Amortized Analysis)

平摊分析用于分析一系列操作的**总代价**，证明虽然单次操作可能很贵，但平均到每次操作的代价很低。注意：平摊分析是最坏情况分析，不涉及概率。

### 6.1 聚合分析 (Aggregate Analysis)

计算 n 次操作的总代价 T(n)，平摊代价 = T(n)/n。

**例：多重栈操作**

栈支持 Push、Pop 和 MultiPop(k)（弹出 min(k, size) 个元素）。

- Push 和 Pop 各 O(1)，MultiPop(k) 为 O(k)
- n 次操作中，每个元素最多被 Push 一次、Pop 一次
- 总代价 T(n) ≤ 2n = O(n)，平摊代价 = O(1)

**例：二进制计数器**

从 0 开始，每次 +1。第 i 位在每 2^i 次操作翻转一次。

n 次操作总翻转次数 = n + ⌊n/2⌋ + ⌊n/4⌋ + ... < 2n

平摊代价 = O(1)。

### 6.2 记账法 (Accounting Method)

为每种操作赋予一个"平摊代价"（可高于或低于实际代价）。多出的部分作为"信用"存储在数据结构中，不足的部分从信用中支付。

**要求**：任何时刻信用总额 ≥ 0（即 Σ平摊代价 ≥ Σ实际代价）。

**例：多重栈**
- Push 的平摊代价 = 2（实际 1，存 1 个信用在该元素上）
- Pop 的平摊代价 = 0（用元素上的信用支付）
- MultiPop 的平摊代价 = 0（每个弹出元素用自己的信用支付）

### 6.3 势能法 (Potential Method)

定义势函数 Φ：数据结构状态 → 实数。

平摊代价 = 实际代价 + ΔΦ = c_i + Φ(D_i) - Φ(D_{i-1})

n 次操作的总平摊代价 = Σc_i + Φ(D_n) - Φ(D_0)

**要求**：Φ(D_n) ≥ Φ(D_0)，通常令 Φ(D_0) = 0 且 Φ ≥ 0。

**例：二进制计数器**
- Φ = 计数器中 1 的个数
- +1 操作：设翻转了 t 个 1→0 和 1 个 0→1
  - 实际代价 = t + 1
  - ΔΦ = 1 - t
  - 平摊代价 = (t + 1) + (1 - t) = 2 = O(1)

**例：动态表扩展**

动态数组（如 vector）插入时若满则倍增容量：
- Φ = 2·size - capacity
- 普通插入：实际代价 1，ΔΦ = 2，平摊代价 = 3
- 扩展插入：实际代价 size+1（复制+插入），ΔΦ = 2 - size，平摊代价 = 3
- 所以每次插入的平摊代价为 O(1)

### 6.4 三种方法的对比

| 方法 | 核心思想 | 适用场景 |
|------|----------|----------|
| 聚合分析 | 直接算总代价除以 n | 结构简单、容易直接求和 |
| 记账法 | 为操作分配信用，贵的操作用便宜操作攒的信用 | 不同操作代价差异大 |
| 势能法 | 用势函数刻画数据结构状态 | 最通用，适合复杂数据结构（如 Splay Tree） |

## 七、线性规划 (Linear Programming)

### 7.1 基本概念

线性规划求解形如以下问题的最优解：

```
最大化    c^T x
约束条件  Ax ≤ b
          x ≥ 0
```

- **决策变量**：x = (x₁, x₂, ..., xₙ)^T
- **目标函数**：c^T x（线性）
- **约束条件**：线性不等式或等式

### 7.2 标准型与松弛型

**标准型**：最大化，所有约束为 ≤，所有变量 ≥ 0。

任何线性规划都可以转化为标准型：
- 最小化 → 取负最大化
- ≥ 约束 → 两端取负变 ≤
- 等式约束 → 拆为 ≤ 和 ≥
- 无约束变量 x → 替换为 x⁺ - x⁻（x⁺, x⁻ ≥ 0）

**松弛型**：引入松弛变量 s ≥ 0，将不等式变为等式：Ax + s = b。

### 7.3 单纯形法概述

**基本可行解**：在松弛型中，选 n 个变量为非基变量（令其为 0），解出 m 个基变量。若基变量 ≥ 0，则为基本可行解，对应多面体的一个**顶点**。

**单纯形法过程**：
1. 从一个基本可行解出发
2. 选择一个能改善目标函数的非基变量进基（入基变量）
3. 通过最小比值测试选择一个基变量出基（出基变量）
4. 进行转轴操作（pivot），得到新的基本可行解
5. 重复直到没有能改善目标的非基变量

**复杂度**：最坏指数时间，但实际中表现很好（平均多项式时间）。

### 7.4 对偶性 (Duality)

每个线性规划（原问题）都有一个**对偶问题**：

| 原问题 (Primal) | 对偶问题 (Dual) |
|-----------------|-----------------|
| max c^T x | min b^T y |
| Ax ≤ b | A^T y ≥ c |
| x ≥ 0 | y ≥ 0 |

**弱对偶定理**：对任意原问题可行解 x 和对偶可行解 y，c^T x ≤ b^T y。

**强对偶定理**：若原问题有最优解，则对偶也有最优解，且最优值相等：c^T x* = b^T y*。

**互补松弛条件**：在最优解处，原问题约束取等 ↔ 对偶变量 > 0。

**应用**：对偶性可用于验证最优性、设计近似算法、经济学中的影子价格解释。

### 7.5 整数线性规划 (ILP)

要求决策变量取整数值。ILP 是 NP-hard 的。

常见处理方法：
- **LP 松弛**：去掉整数约束求 LP，用舍入法得到近似整数解
- **分支定界法**：在 LP 松弛基础上，对非整数变量分支
- **割平面法**：添加新约束（割平面）排除非整数最优解

## 八、NP 完全性理论

### 8.1 基本概念

**判定问题**：答案为"是"或"否"的问题。

**P 类**：存在确定性图灵机在多项式时间内求解的判定问题集合。

**NP 类**：存在非确定性图灵机在多项式时间内求解的判定问题集合。等价地，给定一个"证书"（certificate），可以在多项式时间内验证答案为"是"。

**co-NP**：补问题属于 NP 的问题集合。

**关系**：P ⊆ NP ∩ co-NP。P = NP？是计算机科学最大的未解问题。

### 8.2 规约 (Reduction)

**多项式时间规约**：如果问题 A 可以在多项式时间内转化为问题 B，使得 A 的答案为"是"当且仅当转化后的 B 的答案为"是"，则记 A ≤_p B。

含义：B 至少和 A 一样难。如果 B 有多项式算法，那么 A 也有。

### 8.3 NP-hard 与 NP-complete

**NP-hard**：所有 NP 问题都可以多项式时间规约到它。（不要求自身属于 NP）

**NP-complete (NPC)**：既是 NP-hard，又属于 NP。NPC 是 NP 中"最难"的一类问题。

**Cook-Levin 定理**：SAT（布尔可满足性问题）是 NP-complete 的。这是第一个被证明的 NPC 问题。

**证明问题 X 是 NPC 的步骤**：
1. 证明 X ∈ NP（给出多项式时间验证算法）
2. 选择一个已知的 NPC 问题 Y
3. 证明 Y ≤_p X（构造多项式时间规约）

### 8.4 经典 NPC 问题

| 问题 | 描述 | 常用规约链 |
|------|------|-----------|
| **SAT** | 给定布尔公式，是否存在使其为真的赋值 | 第一个 NPC 问题 |
| **3-SAT** | SAT 的特例：每个子句恰好 3 个文字 | SAT ≤_p 3-SAT |
| **Clique（团）** | 图中是否存在大小 ≥ k 的完全子图 | 3-SAT ≤_p Clique |
| **Vertex Cover** | 是否存在大小 ≤ k 的顶点集覆盖所有边 | Clique ≤_p Vertex Cover（取补图） |
| **Independent Set** | 是否存在大小 ≥ k 的独立集 | Clique ≤_p Independent Set（取补图） |
| **Hamilton Cycle** | 图中是否存在经过每个顶点恰好一次的回路 | 3-SAT ≤_p Hamilton Cycle |
| **TSP（判定版）** | 是否存在总长 ≤ k 的 Hamilton 回路 | Hamilton Cycle ≤_p TSP |
| **SubsetSum** | 集合中是否存在子集和为目标值 | 3-SAT ≤_p SubsetSum |
| **Set Cover** | 是否能用 ≤ k 个子集覆盖全集 | Vertex Cover ≤_p Set Cover |
| **Graph Coloring** | 图是否可用 ≤ k 种颜色正常着色 | 3-SAT ≤_p 3-Coloring |

### 8.5 经典规约示例

**3-SAT ≤_p Clique**：

给定 3-SAT 公式有 k 个子句。构造图 G：
- 每个子句的 3 个文字各对应一个顶点
- 两个顶点之间连边，当且仅当它们不在同一子句中且不互补（x 和 ¬x 不连边）
- 3-SAT 有满足赋值 ↔ G 有大小为 k 的团

**Clique ≤_p Vertex Cover**：

图 G 有大小 k 的团 ↔ 补图 Ḡ 有大小 n-k 的顶点覆盖。

### 8.6 处理 NPC 问题的策略

既然 NPC 问题（除非 P=NP）没有多项式精确算法，实际中：
1. **近似算法**：在多项式时间内求近似最优解（有保证的近似比）
2. **随机算法**：用随机性换取效率
3. **参数化算法**：对某些参数小的实例高效求解
4. **启发式算法**：如模拟退火、遗传算法（无理论保证）
5. **特殊情况**：很多 NPC 问题在特殊结构上有多项式算法（如二部图上的匹配）

## 九、近似算法

### 9.1 近似比

对最优化问题，算法 A 的近似比 ρ(n) 定义为：

```
对所有输入，max(C/C*, C*/C) ≤ ρ(n)
```

C 为算法结果，C* 为最优解。ρ(n) 越接近 1 越好。

ρ(n)-近似算法 = 多项式时间 + 近似比 ≤ ρ(n)。

### 9.2 经典近似算法

**顶点覆盖（2-近似）**：

```
C = ∅
while 图中还有边:
    选择任意一条边 (u, v)
    将 u 和 v 都加入 C
    删除与 u 或 v 关联的所有边
return C
```

分析：选的每条边至少有一个端点在最优解中，所以 |C| ≤ 2|C*|。

**集合覆盖（贪心 ln n-近似）**：

每次选覆盖未覆盖元素最多的集合。近似比 H(n) = 1 + 1/2 + ... + 1/n = O(ln n)。

这是集合覆盖能达到的最好近似比（除非 P=NP）。

**TSP（三角不等式下，2-近似）**：

1. 求最小生成树 MST
2. 对 MST 做 DFS 得到遍历序列
3. 跳过重复顶点得到 Hamilton 回路

分析：MST 权值 ≤ OPT（去掉 Hamilton 回路的一条边就是生成树）。DFS 遍历每条边走两次，权值 ≤ 2·MST ≤ 2·OPT。三角不等式保证跳过顶点不增加距离。

**Christofides 算法（TSP，1.5-近似）**：MST + 奇度顶点最小权完美匹配 + 欧拉回路 → Hamilton 回路。

**背包问题（FPTAS）**：

背包问题有伪多项式时间的 DP 算法 O(nW)。通过缩放价值（除以 ε·v_max/n 后取整），可以得到 (1+ε)-近似算法，时间 O(n³/ε)。这是一个 FPTAS（Fully Polynomial-Time Approximation Scheme）。

### 9.3 近似方案

**PTAS (Polynomial-Time Approximation Scheme)**：对任意 ε > 0，有 (1+ε)-近似算法，运行时间关于 n 是多项式（但可能关于 1/ε 是指数）。

**FPTAS**：运行时间同时关于 n 和 1/ε 都是多项式。比 PTAS 更强。

## 十、随机算法

### 10.1 分类

**Las Vegas 算法**：结果一定正确，运行时间是随机变量。例：随机快排。

**Monte Carlo 算法**：运行时间确定（多项式），结果可能出错。
- 单侧错误：若输出"是"则一定正确，输出"否"可能出错（或反之）
- 双侧错误：任何输出都可能出错，但错误概率有界

### 10.2 经典随机算法

**随机快排**：随机选择 pivot，期望时间复杂度 O(n log n)。

分析：元素 i 和 j 被比较的概率 = 2/(j-i+1)（当且仅当 [i..j] 中第一个被选为 pivot 的是 i 或 j）。

期望比较次数 = Σ_{i<j} 2/(j-i+1) = O(n log n)。

**Karger 最小割算法**：
1. 随机选一条边，收缩（合并两端点）
2. 重复直到只剩两个顶点
3. 剩余的边就是一个割

单次成功概率 ≥ 2/n(n-1)。重复 O(n² log n) 次，以高概率得到最小割。总时间 O(n⁴ log n)，优化后 O(n² log³ n)。

**Miller-Rabin 素性测试**：Monte Carlo 算法，单次错误概率 ≤ 1/4，重复 k 次错误概率 ≤ (1/4)^k。

### 10.3 概率分析工具

- **期望的线性性**：E[X₁ + X₂ + ... + Xₙ] = E[X₁] + ... + E[Xₙ]（即使不独立）
- **指示器随机变量**：X_i = 1（事件发生）或 0，E[X_i] = P(事件发生)
- **Markov 不等式**：P(X ≥ a) ≤ E[X]/a（X ≥ 0）
- **Chebyshev 不等式**：P(|X - μ| ≥ kσ) ≤ 1/k²
- **Chernoff 界**：对独立随机变量之和，给出指数级的尾概率界
