# 数据结构

数据结构研究数据的组织、存储和操作方式。选择合适的数据结构可以极大地影响程序的效率——同一个问题，用数组解可能是 O(n²)，用哈希表解可能是 O(n)。数据结构与算法是计算机科学的两大核心，密不可分。

## 一、线性结构

线性结构中的数据元素之间存在一对一的关系，每个元素最多有一个前驱和一个后继。

### 1.1 数组 (Array)

数组是最基本的数据结构——在内存中占据一段**连续**的空间，所有元素类型相同、大小固定。由于内存连续，可以通过**首地址 + 下标 × 元素大小**直接计算任意元素的地址，实现 O(1) 的随机访问。

- 随机访问：O(1)——这是数组最大的优势
- 插入/删除：O(n)——需要移动大量元素来维持连续性
- 查找（无序）：O(n)——需要逐个比较
- 查找（有序）：O(log n)——可以使用二分查找
- 适合：频繁随机访问、元素数量已知或变化不大的场景

### 1.2 链表 (Linked List)

链表通过**指针**将分散在内存各处的节点串联起来。每个节点包含数据域和指针域。链表不需要连续内存，可以灵活地插入和删除，但失去了随机访问的能力。

| 类型 | 结构 | 特点 |
|------|------|------|
| **单链表** | 每个节点有 data + next 指针 | 只能从前向后遍历 |
| **双链表** | 每个节点有 prev + data + next 指针 | 可以双向遍历，删除更方便（不需要找前驱） |
| **循环链表** | 尾节点指向头节点（形成环） | 适合需要循环访问的场景（如约瑟夫问题） |

- 插入/删除（已知位置）：O(1)——只需修改指针
- 查找：O(n)——必须从头遍历
- 不支持随机访问
- 适合：频繁插入删除、元素数量不确定的场景

**数组 vs 链表**：

| 对比 | 数组 | 链表 |
|------|------|------|
| 内存布局 | 连续 | 离散 |
| 随机访问 | O(1) | O(n) |
| 插入/删除 | O(n) | O(1)（已知位置） |
| 内存利用 | 可能浪费（预分配） | 精确分配，但指针占用额外空间 |
| Cache 友好 | 好（空间局部性） | 差（节点分散在内存中） |

### 1.3 栈 (Stack)

栈是一种**后进先出 (LIFO, Last In First Out)** 的线性表，只能在一端（栈顶）进行插入和删除。就像一摞盘子——最后放上去的最先拿走。

操作：push（入栈）、pop（出栈）、top/peek（查看栈顶），均为 O(1)。

**典型应用**：
- **括号匹配**：遇到左括号入栈，遇到右括号时弹出栈顶并检查是否匹配。若最终栈空则匹配成功
- **表达式求值**：
  - 中缀转后缀：用运算符栈，遇到运算符时弹出栈中优先级 ≥ 当前的运算符
  - 后缀表达式求值：用操作数栈，遇到数字入栈，遇到运算符弹出两个操作数计算后结果入栈
- **函数调用栈**：每次函数调用创建栈帧保存返回地址和局部变量，函数返回时弹出栈帧
- **DFS（深度优先搜索）**：递归本质上使用系统栈，也可以用显式栈模拟

### 1.4 队列 (Queue)

队列是一种**先进先出 (FIFO, First In First Out)** 的线性表，从一端（队尾）插入，从另一端（队首）删除。就像排队买票——先来的先处理。

操作：enqueue（入队）、dequeue（出队）、front（队首），均为 O(1)。

| 类型 | 特点 | 应用 |
|------|------|------|
| **普通队列** | 数组实现时有**假溢出**问题（队首前方有空位但无法使用） | — |
| **循环队列** | 用取模运算解决假溢出。判满：`(rear+1) % size == front`（牺牲一个位置区分空和满） | 操作系统缓冲区 |
| **双端队列 (Deque)** | 两端都可以入出 | C++ STL `deque` |
| **优先队列 (Priority Queue)** | 按**优先级**出队而非先进先出，通常用**堆**实现 | Dijkstra 算法、任务调度 |

## 二、树

树是一种**层次结构**的数据结构，由节点和边组成。每个节点最多有一个父节点（根节点无父节点），可以有零个或多个子节点。树在文件系统（目录结构）、数据库（索引）、编译器（语法树）等领域有广泛应用。

### 2.1 二叉树基础

**二叉树 (Binary Tree)** 是每个节点最多有两个子节点（左子和右子）的树。二叉树是最常用的树结构，许多复杂的树（如 B 树）在概念上都可以用二叉树来理解。

**重要性质**：
- 第 i 层最多有 2^(i-1) 个节点（根为第 1 层）
- 深度为 k 的二叉树最多有 2^k - 1 个节点（此时称为**满二叉树**）
- **n₀ = n₂ + 1**：叶子节点数 = 度为 2 的节点数 + 1（利用总边数 = 总节点数 - 1 推导）

**完全二叉树**：除最后一层外每层都是满的，最后一层从左到右连续填充。完全二叉树特别适合用数组存储——节点 i 的左子 = 2i，右子 = 2i+1，父 = ⌊i/2⌋。堆就是一种完全二叉树。

**四种遍历方式**：

| 遍历 | 访问顺序 | 典型应用 | 递归实现思路 |
|------|----------|----------|------------|
| **前序 (Preorder)** | 根 → 左 → 右 | 复制树、序列化、前缀表达式 | visit(root); preorder(left); preorder(right) |
| **中序 (Inorder)** | 左 → 根 → 右 | BST 有序输出、中缀表达式 | inorder(left); visit(root); inorder(right) |
| **后序 (Postorder)** | 左 → 右 → 根 | 删除树、计算目录大小、后缀表达式 | postorder(left); postorder(right); visit(root) |
| **层序 (Level-order)** | 从上到下、从左到右逐层 | BFS、完全二叉树判定 | 用队列实现 |

**由遍历序列还原二叉树**：
- **前序 + 中序** → 唯一确定一棵二叉树。前序的第一个元素是根，在中序中找到根的位置将中序分为左右子树，递归处理
- **后序 + 中序** → 唯一确定。后序的最后一个元素是根，同理
- **前序 + 后序** → 不能唯一确定（除非是满二叉树）。因为无法确定只有一个子节点时是左子还是右子

### 2.2 二叉搜索树 (BST, Binary Search Tree)

BST 是一种有序的二叉树，核心性质：对于任意节点，**左子树所有节点的值 < 节点值 < 右子树所有节点的值**。这使得 BST 能像有序数组一样进行二分查找，同时像链表一样灵活地插入删除。

| 操作 | 平均时间 | 最坏时间（退化为链表） | 说明 |
|------|----------|----------------------|------|
| 查找 | O(log n) | O(n) | 从根开始，比节点小往左走，比节点大往右走 |
| 插入 | O(log n) | O(n) | 查找到空位后插入为叶子节点 |
| 删除 | O(log n) | O(n) | 最复杂的操作（见下文） |

**删除操作**的三种情况：
- 删除**叶子节点**：直接删除
- 删除**只有一个子树**的节点：用子树替代被删节点
- 删除**有两个子树**的节点：用**中序后继**（右子树的最小值）或**中序前驱**（左子树的最大值）替代被删节点的值，然后删除那个后继/前驱节点

BST 的问题：当数据按有序顺序插入时（如 1,2,3,4,5），BST 退化为链表，所有操作变为 O(n)。这就是平衡二叉树（AVL、红黑树）要解决的问题。

### 2.3 AVL 树

AVL 树是最早提出的**自平衡二叉搜索树**（由 Adelson-Velsky 和 Landis 在 1962 年提出）。

**平衡条件**：任意节点的左右子树**高度差（平衡因子）不超过 1**。即 |height(left) - height(right)| ≤ 1。

当插入或删除操作导致某个节点的平衡因子超过 1 时，通过**旋转**操作恢复平衡。旋转分为四种情况：

| 失衡类型 | 形状 | 旋转方式 | 说明 |
|----------|------|----------|------|
| **LL** | 左子树的左子树过高 | 对失衡节点**右旋** | 一次旋转 |
| **RR** | 右子树的右子树过高 | 对失衡节点**左旋** | 一次旋转 |
| **LR** | 左子树的右子树过高 | 先对左子**左旋**，再对失衡节点**右旋** | 两次旋转 |
| **RL** | 右子树的左子树过高 | 先对右子**右旋**，再对失衡节点**左旋** | 两次旋转 |

AVL 树保证所有操作的时间复杂度严格为 O(log n)。

### 2.4 红黑树

红黑树是另一种自平衡 BST，比 AVL 树的平衡条件更宽松，因此插入删除时需要的旋转次数更少（最多 3 次），适合插入删除频繁的场景。

**五条性质**（红黑树的不变量）：
1. 每个节点要么是**红色**要么是**黑色**
2. **根节点**是黑色
3. 所有**叶子节点（NIL/空节点）** 是黑色
4. 红色节点的子节点必须是黑色（**不能有连续的红节点**）
5. 从任一节点到其所有叶子的路径上，**黑色节点数相同**（称为黑高度）

性质 4 和 5 共同保证了最长路径不超过最短路径的两倍，因此红黑树是"近似平衡"的。

**AVL 树 vs 红黑树**：

| 对比 | AVL 树 | 红黑树 |
|------|--------|--------|
| 平衡条件 | 严格平衡（高度差 ≤ 1） | 近似平衡（最长路径 ≤ 2×最短路径） |
| 树高度 | 更矮（1.44 log n） | 稍高（2 log n） |
| 查找性能 | 更快（树更矮，比较次数少） | 稍慢 |
| 插入/删除 | 可能需要 O(log n) 次旋转 | 最多 3 次旋转（但可能需要 O(log n) 次重着色） |
| 适用场景 | 查找密集型（如数据库索引） | 插入删除频繁型（如内存中的有序集合） |
| 典型应用 | 数据库、Windows 内存管理 | Java TreeMap/TreeSet、C++ std::map、Linux CFS 调度器、epoll |

### 2.5 B 树 / B+ 树

B 树和 B+ 树是为**磁盘存储**优化的多路搜索树。与二叉树不同，B 树的每个节点可以有多个键和多个子节点，从而降低树的高度、减少磁盘 I/O 次数。

**B 树（m 阶）** 的性质：
- 每个节点最多有 **m 个子节点**、**m-1 个关键字**
- 每个非根节点至少有 **⌈m/2⌉ 个子节点**（保证节点不会太空）
- 所有**叶子节点在同一层**（完美平衡）
- 关键字在节点内有序排列，子节点按关键字范围划分

**B+ 树**：B 树的变种，在数据库索引中更常用。与 B 树的关键区别：
- **内部节点只存索引**（不存数据），数据全部在叶子节点
- **叶子节点通过指针链接成有序链表**——支持高效的范围查询
- 内部节点可以存储更多关键字（因为不存数据），树更矮

详见数据库篇。

### 2.6 哈夫曼树与哈夫曼编码

**哈夫曼树 (Huffman Tree)** 是一种**最优二叉树**——带权路径长度 (WPL) 最小的二叉树。WPL = 所有叶子节点的"权值 × 根到该叶子的路径长度"之和。

**构建算法**（贪心）：
1. 将所有节点放入一个最小优先队列
2. 每次取出权值**最小的两个**节点，合并为一个新节点（权值为两者之和）
3. 将新节点放回优先队列
4. 重复直到只剩一个节点（根节点）

**哈夫曼编码**：哈夫曼树的重要应用。为每个字符分配变长编码——左分支为 0，右分支为 1。从根到叶子的路径就是该字符的编码。

关键性质：
- **前缀编码**：任何字符的编码都不是另一个字符编码的前缀，因此解码无歧义
- **频率高的字符编码短，频率低的字符编码长**——实现最优压缩
- 哈夫曼编码被用于 JPEG、DEFLATE（zip/gzip）等压缩算法中

### 2.7 字典树 (Trie)

字典树（也称前缀树）是一种用于高效存储和查找字符串的树结构。每条边代表一个字符，从根到某节点的路径拼接起来形成一个字符串的前缀。

- 查找/插入时间 O(L)，L 为字符串长度（与字符串总数无关！）
- 空间开销较大（每个节点最多有 |字符集| 个子节点）
- 适合：前缀匹配（自动补全）、词频统计、拼写检查、IP 路由查找

## 三、堆 (Heap)

堆是一种特殊的**完全二叉树**，满足堆序性质。堆是实现优先队列的标准数据结构。

### 3.1 基本概念

- **最大堆 (Max Heap)**：每个节点的值 ≥ 其所有子节点的值（根节点是最大值）
- **最小堆 (Min Heap)**：每个节点的值 ≤ 其所有子节点的值（根节点是最小值）
- 堆是完全二叉树，因此用**数组**存储最为紧凑高效：节点 i 的左子 = 2i+1，右子 = 2i+2，父 = (i-1)/2

### 3.2 核心操作

| 操作 | 方法 | 时间复杂度 | 说明 |
|------|------|-----------|------|
| **插入** | 放到数组末尾（堆的最后），然后**上浮 (sift up)**——与父节点比较，如果违反堆序则交换，重复 | O(log n) | 最多上浮到根 |
| **删除堆顶** | 用数组末尾元素替换堆顶，然后**下沉 (sift down)**——与较大/较小的子节点交换，重复 | O(log n) | 最多下沉到叶子 |
| **建堆** | 从最后一个**非叶节点**开始，逐个**下沉** | **O(n)** | 看似 O(n log n)，但数学分析为 O(n) |
| **取堆顶** | 直接返回 arr[0] | O(1) | 堆的最大优势 |

**建堆为什么是 O(n)**：虽然每个节点下沉最多 O(log n)，但大部分节点在底层（叶子层不需要下沉，倒数第二层最多下沉 1 层...），总工作量的精确分析是 O(n)。

### 3.3 堆排序

堆排序利用堆的性质实现原地 O(n log n) 排序：

1. **建最大堆**：O(n)
2. 重复 n-1 次：将堆顶（最大值）与当前末尾交换，堆大小减 1，对新堆顶执行下沉。每次 O(log n)
3. 总时间 O(n log n)，空间 O(1)（原地排序），**不稳定**

### 3.4 优先队列

优先队列是一种抽象数据类型，支持两个核心操作：插入元素和取出最高优先级的元素。底层通常用堆实现。

- 插入：O(log n)
- 取出最值：O(1)
- 删除最值：O(log n)
- C++ STL：`priority_queue`（默认最大堆）

应用场景：Dijkstra 最短路算法、Prim 最小生成树、哈夫曼编码、任务调度、TopK 问题。

## 四、图

图是最一般的数据结构，可以表示任意的多对多关系。图由**顶点 (Vertex)** 和**边 (Edge)** 组成，边连接两个顶点表示它们之间的关系。社交网络、互联网、交通网络、依赖关系等都可以用图来建模。

### 4.1 图的存储

| 方式 | 空间 | 查边 (u,v) | 遍历 u 的邻接点 | 适用 |
|------|------|-----------|----------------|------|
| **邻接矩阵** | O(V²) | O(1)——直接查 M[u][v] | O(V)——扫描整行 | **稠密图**（边数接近 V²） |
| **邻接表** | O(V+E) | O(度)——遍历 u 的邻接链表 | O(度) | **稀疏图**（边数远小于 V²）——大多数实际图 |

邻接矩阵用二维数组表示，M[i][j] = 1（或权重）表示 i 到 j 有边。空间浪费在稀疏图上。

邻接表为每个顶点维护一个链表（或动态数组），存储所有与该顶点相邻的顶点。空间利用率高。

### 4.2 DFS 与 BFS

DFS 和 BFS 是图的两种基本遍历方式，是几乎所有图算法的基础。

| 对比 | DFS (深度优先搜索) | BFS (广度优先搜索) |
|------|-----|-----|
| 策略 | 沿一条路径尽可能深入，走到底再回溯 | 从起点出发逐层向外扩展 |
| 数据结构 | **栈**（递归使用系统栈，也可用显式栈） | **队列** |
| 空间复杂度 | O(V)（递归深度） | O(V)（队列中的节点数） |
| 时间复杂度 | O(V+E)（邻接表） | O(V+E)（邻接表） |
| 应用 | 连通性判断、拓扑排序、强连通分量(Tarjan)、回溯搜索、欧拉路径 | **最短路径（无权图）**、层序遍历、多源 BFS |

**关键区别**：BFS 找到的路径一定是最短路径（在无权图中），DFS 不保证。

### 4.3 最小生成树 (MST)

对于一个连通加权无向图，**最小生成树**是包含所有顶点、边权之和最小的树（n 个顶点，n-1 条边，无环）。

| 算法 | 策略 | 时间复杂度 | 适用场景 |
|------|------|-----------|----------|
| **Prim** | 从一个顶点出发，每次将**距已选顶点集最近的**未选顶点加入 | O(V²)（朴素）或 O(E log V)（堆优化） | **稠密图**（朴素版 O(V²) 与边数无关） |
| **Kruskal** | 将所有边按权重排序，每次选**最小权重**且**不形成环**的边加入 | O(E log E) | **稀疏图**（只需对边排序） |

Prim 类似 Dijkstra（贪心选最近的顶点）；Kruskal 使用**并查集**高效判断加入一条边是否会形成环。

**切割定理 (Cut Property)**：对于图的任意切割（将顶点分为两个不相交集合），横跨切割的最小权重边一定属于某棵最小生成树。这是 Prim 和 Kruskal 正确性的理论基础。

### 4.4 最短路径

最短路径问题是图论中最重要的问题之一。

| 算法 | 问题类型 | 时间复杂度 | 适用条件 | 核心思想 |
|------|----------|-----------|----------|----------|
| **Dijkstra** | 单源最短路 | O(V²) 朴素 / O(E log V) 堆优化 | **无负权边** | 贪心——每次选距离最小的未确定顶点 |
| **Bellman-Ford** | 单源最短路 | O(VE) | 可以有**负权边** | 动态规划——对所有边松弛 V-1 轮。第 V 轮若还能松弛则存在负权环 |
| **SPFA** | 单源最短路 | 平均 O(kE)，k 常数 | 可以有负权边 | BF 的优化，用队列记录被更新的顶点（最坏仍 O(VE)，可被卡） |
| **Floyd** | **全源**最短路 | O(V³) | 可以有负权边 | 动态规划——枚举中间顶点 k，d[i][j] = min(d[i][j], d[i][k]+d[k][j]) |

**松弛操作 (Relaxation)**：如果 d[u] + w(u,v) < d[v]，则更新 d[v] = d[u] + w(u,v)。这是所有最短路算法的核心操作。

### 4.5 拓扑排序

对**有向无环图 (DAG)** 的所有顶点排成线性序列，使得对每条有向边 (u,v)，u 在 v 前面。拓扑排序反映了事件的先后依赖关系（如课程先修关系、编译依赖）。

**Kahn 算法**（BFS 方法）：
1. 将所有入度为 0 的顶点入队（它们没有前驱，可以先执行）
2. 出队一个顶点，加入结果序列；删除该顶点的所有出边，更新相关顶点的入度
3. 新产生的入度为 0 的顶点入队
4. 重复直到队列为空。如果结果序列包含所有顶点则无环，否则图中有环

时间复杂度 O(V+E)。拓扑排序结果不唯一。

### 4.6 关键路径

关键路径用于分析项目管理中的工期问题。在 **AOE (Activity On Edge) 网**中：
- 顶点代表事件（里程碑），边代表活动（任务），边的权重代表活动的持续时间
- **关键路径**是从源点到汇点的**最长路径**，决定了项目的**最短完成时间**
- 关键路径上的活动称为**关键活动**，其特征是**最早开始时间 = 最迟开始时间**（没有任何松弛余量，延误就会影响整个工期）

计算步骤：
1. 正向求各事件的**最早发生时间 (Earliest Time)**
2. 反向求各事件的**最迟发生时间 (Latest Time)**
3. 对每个活动计算最早/最迟开始时间，两者相等的就是关键活动

## 五、散列 (Hash)

散列（哈希）的核心思想是：通过一个函数将**键 (Key)** 直接映射为存储位置（数组下标），从而实现**平均 O(1)** 的查找、插入和删除。这比有序数组的 O(log n) 和链表的 O(n) 快得多。

### 5.1 散列函数

好的散列函数应该使键值均匀分布在散列表中，减少冲突。

- **除留余数法**：H(key) = key % p，p 通常取**不大于表长的最大素数**（素数可以减少冲突模式）。这是最常用的方法
- **数字分析法**：分析键值中各位数字的分布，选择分布均匀的位作为散列地址
- **平方取中法**：取键值平方后的中间几位
- **折叠法**：将键值分为几段相加

### 5.2 冲突处理

不同的键可能映射到同一位置（**冲突/碰撞**），必须有处理机制。

| 方法 | 描述 | 优缺点 |
|------|------|--------|
| **开放寻址—线性探测** | H(key)+1, +2, +3, ... | 简单，但产生**聚集 (clustering)**——冲突的元素连成一片，加剧后续冲突 |
| **开放寻址—二次探测** | H(key)+1², -1², +2², -2², ... | 减轻聚集，但可能访问不到所有位置 |
| **开放寻址—双散列** | H(key)+i×H₂(key)，H₂ 是另一个散列函数 | 效果好，几乎消除聚集 |
| **链地址法（拉链法）** | 每个槽是一个链表，冲突元素挂在同一链表上 | **最常用**（Java HashMap）。无聚集，装填因子可 > 1 |

### 5.3 装填因子与性能

**装填因子 α = 已存元素数 / 表长**。α 越大，冲突越严重，性能越差。

- 链地址法：α 可以 > 1（每个槽可以挂多个元素），但 α 过大时链表变长，退化为 O(n)。Java HashMap 在 α > 0.75 时扩容
- 开放寻址法：α 必须 < 1（因为需要空位放新元素），通常保持 α < 0.75
- 当 α 过大时需要**扩容 (rehash)**——创建更大的表，重新插入所有元素

## 六、排序算法对比

排序是最基本的算法问题。不同排序算法在时间复杂度、空间复杂度和稳定性上各有特点，选择合适的算法取决于具体场景。

**稳定性**：如果排序前两个相等的元素 a 和 b 的相对顺序是 a 在 b 前，排序后 a 仍在 b 前，则该排序算法是**稳定的**。稳定性在多关键字排序时很重要。

| 算法 | 平均 | 最好 | 最坏 | 空间 | 稳定 | 核心思想 |
|------|------|------|------|------|------|----------|
| **冒泡排序** | O(n²) | O(n) | O(n²) | O(1) | 稳定 | 相邻元素两两比较交换，每轮将最大值冒泡到末尾 |
| **选择排序** | O(n²) | O(n²) | O(n²) | O(1) | 不稳定 | 每轮选出最小值放到已排序部分末尾 |
| **插入排序** | O(n²) | O(n) | O(n²) | O(1) | 稳定 | 将元素插入已排序部分的正确位置（类似整理扑克牌） |
| **希尔排序** | O(n^1.3) | O(n) | O(n²) | O(1) | 不稳定 | 插入排序的改进，先用大步长使数组大致有序，再逐步缩小步长 |
| **归并排序** | O(n log n) | O(n log n) | O(n log n) | O(n) | 稳定 | 分治——将数组分为两半分别排序后合并 |
| **快速排序** | O(n log n) | O(n log n) | O(n²) | O(log n) | 不稳定 | 分治——选 pivot 将数组分为小于和大于两部分，递归 |
| **堆排序** | O(n log n) | O(n log n) | O(n log n) | O(1) | 不稳定 | 建堆后反复取堆顶并下沉 |
| **计数排序** | O(n+k) | O(n+k) | O(n+k) | O(k) | 稳定 | 统计每个值出现次数，直接算出位置。k 为值域范围 |
| **基数排序** | O(d(n+k)) | O(d(n+k)) | O(d(n+k)) | O(n+k) | 稳定 | 按位排序（低位到高位），每位用稳定排序。d 为位数 |
| **桶排序** | O(n+k) | O(n) | O(n²) | O(n+k) | 稳定 | 将元素分到多个桶中分别排序后合并 |

**选择依据**：
- **数据量小**（n < 50）：插入排序（常数小，代码简单，对小数据集很快）
- **一般用途**：**快速排序**（平均最快，大多数标准库使用）
- **需要稳定**：**归并排序**（O(n log n) 的稳定排序中最常用）
- **空间受限**：**堆排序**（O(1) 额外空间的 O(n log n) 排序）
- **数据范围有限**：**计数排序 / 基数排序**（线性时间，但需要值域不太大）
- **数据基本有序**：插入排序（接近 O(n)）

**快速排序的最坏情况优化**：
- **随机选取 pivot**：避免对有序数据退化
- **三数取中法**：取首、尾、中间三个元素的中位数作为 pivot
- **混合排序**：当子数组长度小于阈值时改用插入排序（如 C++ STL 的 `std::sort` 使用 IntroSort = 快排 + 堆排 + 插入排序）

## 七、查找算法

| 算法 | 时间复杂度 | 要求 | 适用场景 |
|------|-----------|------|----------|
| **顺序查找** | O(n) | 无 | 无序小数据集 |
| **二分查找** | O(log n) | **有序**数组 | 有序静态数据 |
| **插值查找** | O(log log n) 平均 | 有序且**均匀分布** | 数据分布均匀时 |
| **散列查找** | O(1) 平均 | 散列表 | 等值查找（最快） |
| **BST 查找** | O(log n) 平均 | BST | 需要有序性的动态数据 |

**二分查找**是最重要的查找算法。核心思想：每次将搜索范围缩小一半。要求数据**有序**且支持**随机访问**（因此只能用于数组，不能用于链表）。

## 八、算法分析

### 8.1 时间复杂度

时间复杂度用**大 O 记号**描述算法运行时间随输入规模增长的趋势。只关注最高阶项，忽略常数和低阶项。

常见复杂度从快到慢：O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)

| 复杂度 | 名称 | 典型算法 | n=10⁶ 的数量级 |
|--------|------|----------|---------------|
| O(1) | 常数 | 数组访问、哈希查找 | 1 |
| O(log n) | 对数 | 二分查找 | 20 |
| O(n) | 线性 | 遍历、线性查找 | 10⁶ |
| O(n log n) | 线性对数 | 快排、归并排序 | 2×10⁷ |
| O(n²) | 平方 | 冒泡排序、暴力二重循环 | 10¹² (超时) |
| O(2ⁿ) | 指数 | 子集枚举 | 天文数字 |

### 8.2 主定理 (Master Theorem)

主定理用于求解**分治递推关系**的时间复杂度。对于形如 T(n) = aT(n/b) + O(n^d) 的递推：

| 条件 | 复杂度 | 直觉理解 |
|------|--------|----------|
| d < log_b(a) | T(n) = O(n^(log_b(a))) | 叶子层的工作量主导 |
| d = log_b(a) | T(n) = O(n^d × log n) | 每层工作量相同 |
| d > log_b(a) | T(n) = O(n^d) | 根层的工作量主导 |

**常见应用**：
- **二分查找**：T(n) = T(n/2) + O(1) → a=1, b=2, d=0 → d = log₂1 = 0 → O(log n)
- **归并排序**：T(n) = 2T(n/2) + O(n) → a=2, b=2, d=1 → d = log₂2 = 1 → O(n log n)
- **Karatsuba 乘法**：T(n) = 3T(n/2) + O(n) → a=3, b=2, d=1 → d < log₂3 ≈ 1.58 → O(n^1.58)
- **Strassen 矩阵乘法**：T(n) = 7T(n/2) + O(n²) → O(n^(log₂7)) ≈ O(n^2.81)

## 九、递推方程求解

递推方程是分析递归算法时间复杂度的核心工具。对于形如 T(n) = aT(n/b) + f(n) 的递推关系，有三种主要求解方法。

### 9.1 迭代归纳法

反复将递推式展开，观察规律后求和。

**示例**：T(n) = 2T(n/2) + n，T(1) = 1
- T(n) = 2T(n/2) + n = 2[2T(n/4) + n/2] + n = 4T(n/4) + 2n
- 第 k 层：T(n) = 2^k T(n/2^k) + kn
- 令 n/2^k = 1，得 k = log₂n，故 T(n) = nT(1) + n log n = O(n log n)

**常用求和公式**：
- 等比级数：a₁(1 + q + q² + ... + qᵏ) = a₁(1 - q^(k+1))/(1 - q)
- 调和级数：1 + 1/2 + ... + 1/n = O(log n)
- 对数和：log1 + log2 + ... + logn = O(n log n)

### 9.2 递归树法

将递推式展开为一棵树，每个节点代表一次递归调用的非递归工作量。树的每一层的总工作量求和，再对所有层求和。

**关键步骤**：
1. 画出递归树——根节点工作量为 f(n)，每个节点有 a 个子节点，子问题规模为 n/b
2. 第 k 层有 aᵏ 个节点，每个节点工作量为 f(n/bᵏ)
3. 树高为 log_b(n)，叶子层共 a^(log_b n) = n^(log_b a) 个节点
4. 总工作量 = 各层工作量之和

### 9.3 主定理（详细版）

设 a >= 1, b > 1 为常数，f(n) 为函数，T(n) = aT(n/b) + f(n)，则：

1. 若 f(n) = O(n^(log_b a - ε))，ε > 0，则 **T(n) = Θ(n^(log_b a))**——叶子层主导
2. 若 f(n) = Θ(n^(log_b a))，则 **T(n) = Θ(n^(log_b a) · log n)**——每层工作量相同
3. 若 f(n) = Ω(n^(log_b a + ε))，ε > 0，且 af(n/b) ≤ cf(n)（某常数 c < 1），则 **T(n) = Θ(f(n))**——根层主导

**注意**：存在不满足主定理三种情况的递推式（如 f(n) = n log n 时 T(n) = 2T(n/2) + n log n 不直接适用简化版，需用高级形式）。

**简化版常用结论**（f(n) = af(n/b) + d(n)）：
- 当 d(n) 为常数：T(n) = O(n^(log_b a))（a ≠ 1）或 O(log n)（a = 1）
- 当 d(n) = cn：T(n) = O(n)（a < b）或 O(n log n)（a = b）或 O(n^(log_b a))（a > b）

## 十、分治策略 (Divide and Conquer)

分治是将问题分解为若干个规模更小的**同类子问题**，递归求解后合并结果的算法设计范式。

### 10.1 分治策略框架

**三步骤**：
1. **Divide（分解）**：将原问题分为 a 个规模为 n/b 的子问题
2. **Conquer（求解）**：递归地解决各子问题；当子问题规模足够小时直接求解
3. **Combine（合并）**：将子问题的解合并为原问题的解

**适用条件**：
- 子问题与原问题类型相同（自相似性）
- 子问题之间相互独立（无重叠，否则应考虑动态规划）
- 存在基准情况（递归终止条件）

**均衡划分原则**：子问题的规模应尽量相等。不均衡的划分（如 n-1 和 1）会导致递归树退化为链，复杂度升高。例如快速排序最坏 O(n²) 就是因为不均衡划分。

### 10.2 经典分治算法

#### Strassen 矩阵乘法

传统矩阵乘法需要 O(n³) 次乘法。Strassen 算法通过巧妙地构造 7 个子矩阵乘法（而非 8 个），将复杂度降低：

- 将 n×n 矩阵分为 4 个 n/2 × n/2 的子矩阵
- 构造 7 个辅助矩阵乘积 M₁...M₇（仅需 7 次递归乘法）
- 用 M₁...M₇ 的加减组合得到结果矩阵的 4 个块
- T(n) = 7T(n/2) + O(n²) → **O(n^(log₂7)) ≈ O(n^2.81)**

#### 最近点对问题

给定平面上 n 个点，找距离最近的一对点。

- 按 x 坐标排序，用中线将点集分为左右两半
- 递归求左半和右半的最近点对距离 d₁, d₂，令 d = min(d₁, d₂)
- **关键**：检查跨越中线的点对——只需考虑中线两侧宽度为 d 的带状区域内的点
- 带状区域内每个点最多与 7 个点比较（按 y 坐标排序后扫描）
- T(n) = 2T(n/2) + O(n) → **O(n log n)**

#### 线性时间选择（中位数的中位数）

在无序数组中找第 k 小的元素，最坏情况 O(n)。

**算法步骤**：
1. 将 n 个元素分为 ⌈n/5⌉ 组，每组 5 个
2. 对每组排序取中位数，得到 ⌈n/5⌉ 个中位数
3. 递归地找这些中位数的中位数 m*（作为 pivot）
4. 用 m* 划分数组，根据 k 的位置递归到一侧

**为什么选 5**：保证 pivot 的好坏——至少有 3n/10 个元素 ≤ m*，至少 3n/10 个元素 ≥ m*。递推 T(n) ≤ T(n/5) + T(7n/10) + O(n)，解为 **O(n)**。

#### 芯片测试问题

n 片芯片中好芯片多于坏芯片，两两测试。分治思路：
- 每轮让芯片两两配对测试，淘汰不一致的对和一致为"坏"的对
- 一致为"好"的对保留其中一片
- 问题规模至少减半，T(n) = T(n/2) + O(n) → **O(n)**

#### 快速傅里叶变换 (FFT)

将多项式从系数表示转为点值表示，或反向转换。

- 利用单位复根的对称性，将 n 次 DFT 分为两个 n/2 次 DFT
- T(n) = 2T(n/2) + O(n) → **O(n log n)**
- 应用：大数乘法、多项式乘法、信号处理

## 十一、动态规划 (Dynamic Programming)

动态规划适用于具有**最优子结构**和**重叠子问题**性质的优化问题。通过保存子问题的解（避免重复计算），自底向上地构建最终解。

### 11.1 动态规划框架

**适用条件**：
- **最优子结构**：问题的最优解包含子问题的最优解
- **重叠子问题**：不同的决策序列会产生相同的子问题（区分于分治）
- 多阶段决策过程

**设计步骤**：
1. 确定**子问题边界**和**状态**定义
2. 建立关于**目标函数的递推方程**（状态转移方程）
3. 确定**计算顺序**（自底向上）
4. 用**表格存储**中间结果
5. 通过**标记函数追踪**最优解的构成

**与分治的区别**：分治的子问题不重叠，各自独立；动态规划的子问题大量重叠，必须存储避免重复计算。

### 11.2 经典 DP 问题

#### 矩阵链乘法

给定 n 个矩阵 A₁A₂...Aₙ 的链乘，通过不同的加括号方式，使标量乘法次数最少。

- **状态**：m[i][j] 表示计算 Aᵢ...Aⱼ 的最少乘法次数
- **转移**：m[i][j] = min_{i≤k<j} { m[i][k] + m[k+1][j] + pᵢ₋₁·pₖ·pⱼ }
  - 其中矩阵 Aᵢ 的维度为 p_{i-1} × pᵢ
- **边界**：m[i][i] = 0
- **计算顺序**：按链长 L = j - i + 1 从小到大
- **时间**：O(n³)，**空间**：O(n²)
- 用标记函数 s[i][j] = k 记录最优分割点，可追踪最优加括号方案

#### 最长公共子序列 (LCS)

给定序列 X = x₁...xₘ 和 Y = y₁...yₙ，求最长公共子序列。

- **状态**：c[i][j] = LCS(X₁...ᵢ, Y₁...ⱼ) 的长度
- **转移**：
  - 若 xᵢ = yⱼ：c[i][j] = c[i-1][j-1] + 1
  - 否则：c[i][j] = max(c[i-1][j], c[i][j-1])
- **边界**：c[i][0] = c[0][j] = 0
- **时间**：O(mn)，**空间**：O(mn)（可优化为 O(min(m,n))）
- 通过回溯标记数组可还原具体的 LCS

#### 最大子段和

给定整数序列，找连续子段使其和最大。

- **Kadane 算法**（DP）：b[j] = max(b[j-1] + a[j], a[j])，即以 a[j] 结尾的最大子段和
- **答案**：max_{1≤j≤n} b[j]
- **时间**：**O(n)**，空间 O(1)
- 也可用分治法 O(n log n) 解决

#### 0-1 背包问题

n 个物品，每个有重量 wᵢ 和价值 vᵢ，背包容量 W，每个物品只能选或不选。

- **状态**：dp[i][j] = 前 i 个物品、容量为 j 时的最大价值
- **转移**：dp[i][j] = max(dp[i-1][j], dp[i-1][j-wᵢ] + vᵢ)（若 j ≥ wᵢ）
- **时间**：O(nW)——**伪多项式时间**（W 的二进制表示长度为 log W）
- 空间可优化为 O(W)（一维数组，逆序遍历容量）

#### 最优二叉搜索树

给定 n 个键 k₁ < k₂ < ... < kₙ 及其搜索概率 pᵢ（命中）和 qᵢ（未命中），构造搜索期望代价最小的 BST。

- **状态**：e[i][j] = 包含键 kᵢ...kⱼ 的最优 BST 的期望搜索代价
- **转移**：e[i][j] = min_{i≤r≤j} { e[i][r-1] + e[r+1][j] + w(i,j) }
  - w(i,j) = Σpₖ + Σqₖ 为概率之和
- **时间**：O(n³)（可用 Knuth 优化到 O(n²)）

#### 图像压缩

将 n 个像素分成若干段，每段用相同的位数编码，使总存储量最小。

- 本质是一个最优分段问题
- **状态**：s[i] = 前 i 个像素的最小存储位数
- **转移**：s[i] = min_{max(1,i-255)≤j≤i} { s[j-1] + (i-j+1)·bₘₐₓ(j,i) + 11 }
  - bₘₐₓ(j,i) 为第 j 到第 i 个像素所需的最大位数，11 位为段头信息
- **时间**：O(n)（每段最长 256）

## 十二、贪心算法 (Greedy Algorithm)

贪心算法在每一步都做出当前看起来最好的选择（局部最优），期望最终得到全局最优解。贪心算法不一定总能得到最优解，需要证明其正确性。

### 12.1 贪心算法框架

**适用条件**：
- **贪心选择性质**：全局最优解可以通过局部最优选择构成——即存在一个最优解包含贪心选择
- **最优子结构**：做出贪心选择后，剩余子问题的最优解与贪心选择组合仍是原问题的最优解

**正确性证明方法**：
1. **归纳法**：证明每一步贪心选择后问题仍满足贪心选择性质
2. **交换论证 (Exchange Argument)**：假设存在一个不包含贪心选择的最优解，通过交换元素将其改造为包含贪心选择的解，且不变差

**贪心 vs 动态规划**：
- 贪心是"做选择再解子问题"（自顶向下，不回头）
- 动态规划是"解完所有子问题再做选择"（考虑所有可能）
- 贪心效率更高（通常排序后一遍扫描），但适用范围更窄

### 12.2 经典贪心问题

#### 活动选择问题

n 个活动，每个有开始和结束时间 [sᵢ, fᵢ)，选最多的互不冲突的活动。

- **贪心策略**：按结束时间升序排序，每次选结束最早且与已选活动不冲突的活动
- **正确性**：交换论证——选结束最早的活动至少不会比选其他活动差
- **时间**：O(n log n)（排序）

#### 最优装载问题

n 个集装箱，重量为 wᵢ，船的载重为 C，求最多能装多少个集装箱。

- **贪心策略**：按重量升序排序，依次装载直到超出容量
- **正确性**：轻的先装，能装的个数最多
- **时间**：O(n log n)

#### 最小延迟调度

n 个任务，每个需要 tᵢ 时间完成且有截止时间 dᵢ，单机调度使最大延迟最小。

- **贪心策略**：按截止时间 dᵢ 升序排序（EDD 规则）
- **时间**：O(n log n)

#### 哈夫曼编码（贪心证明）

前面已介绍哈夫曼树的构建。其正确性可用贪心来证明：
- **贪心选择性质**：频率最小的两个字符一定在最优编码树中是兄弟（最深叶子）
- **证明**：反证法 + 交换论证。若最优树中频率最小的字符不在最深层，交换后 WPL 不增

### 12.3 贪心在图论中的应用

- **Dijkstra 最短路**：贪心选当前距离最小的未确定顶点（无负权边时正确）
- **Prim 最小生成树**：贪心选距已选集合最近的顶点
- **Kruskal 最小生成树**：贪心选权重最小的不形成环的边
- 正确性均基于**切割定理**（横跨切割的最小权边属于某棵 MST）

## 十三、回溯法 (Backtracking)

回溯法是一种系统搜索问题解空间的算法，本质是带**剪枝**的深度优先搜索。

### 13.1 回溯法框架

**基本思想**：
- 将解表示为向量 (x₁, x₂, ..., xₙ)，解空间组织为一棵**搜索树**
- 从根节点出发，按 DFS 搜索解空间树
- 在每个节点检查**约束条件**——如果当前部分解不可能扩展为可行解，则**剪枝**（回溯到父节点）
- 对于优化问题，还可以用**限界函数**剪去不可能产生更优解的分支

**解空间树的类型**：
- **子集树**：每个元素选或不选，共 2ⁿ 个叶子（如 0-1 背包）
- **排列树**：确定 n 个元素的排列，共 n! 个叶子（如 TSP）

**通用框架**（递归版）：
```
Backtrack(k):
    if k > n:  // 到达叶子，得到一个完整解
        检查/记录解
        return
    for each possible value v for x_k:
        x_k = v
        if 约束条件满足:  // 剪枝
            Backtrack(k + 1)
```

### 13.2 经典回溯问题

#### n 皇后问题

在 n×n 棋盘上放 n 个皇后，使任意两个皇后不同行、不同列、不同对角线。

- **解向量**：x[i] = 皇后在第 i 行的列号（1 ≤ x[i] ≤ n）
- **约束条件**：x[i] ≠ x[j] 且 |x[i] - x[j]| ≠ |i - j|（不同列且不同对角线）
- 搜索树是排列树的子集
- 8 皇后共有 92 个解

#### 0-1 背包问题（回溯版）

- **解向量**：x[i] ∈ {0, 1}，子集树
- **约束条件**：当前重量不超过背包容量
- **限界函数**：计算当前价值 + 剩余物品按单位价值贪心装入的上界，若 ≤ 当前最优值则剪枝
- 先按**单位价值**降序排列物品可改善剪枝效果

#### TSP（旅行商问题）

访问 n 个城市各一次并回到出发城市，使总路径最短。

- **解向量**：(x₁, x₂, ..., xₙ) 为城市访问顺序，排列树
- **约束**：x[k] 未被前面访问过，且存在边 (x[k-1], x[k])
- **限界**：当前路径长度 + 下界估计（如剩余城市最小出边之和）> 当前最优值则剪枝

### 13.3 搜索树节点数估计（Monte Carlo 方法）

回溯算法的实际搜索量难以精确估计。Monte Carlo 方法通过**随机采样**估计搜索树的节点数：
- 从根出发，在每个节点随机选择一个可行子节点深入
- 设第 i 层有 mᵢ 个可行子节点，则整棵树节点数的估计为：1 + m₁ + m₁·m₂ + m₁·m₂·m₃ + ...
- 重复多次取平均可提高估计精度

## 十四、分支限界法 (Branch and Bound)

分支限界法与回溯法类似，也是搜索解空间树，但采用**广度优先**或**最佳优先**策略，并利用更强的**代价函数（限界函数）**进行剪枝。

### 14.1 分支限界法框架

**与回溯法的区别**：

| 对比 | 回溯法 | 分支限界法 |
|------|--------|----------|
| 搜索策略 | DFS | BFS 或最佳优先 |
| 目标 | 找所有解或某个最优解 | 通常找一个最优解 |
| 数据结构 | 栈（递归） | 队列（BFS）或优先队列（最佳优先） |
| 剪枝依据 | 约束条件 + 简单限界 | 代价函数（上界/下界） |

**代价函数**：
- 对最大化问题：计算每个节点的**上界**（该节点子树中可能的最大值），若上界 ≤ 当前最优值则剪枝
- 对最小化问题：计算每个节点的**下界**，若下界 ≥ 当前最优值则剪枝

### 14.2 经典应用

#### 最大团问题

给定无向图 G = (V, E)，找最大完全子图（团）。

- **上界函数**：当前团大小 + 剩余可加入的顶点数
- 若上界 ≤ 当前最大团大小则剪枝
- 按顶点度数降序排列可改善效率

#### TSP 的分支限界

- 用**最佳优先**策略：优先扩展下界最小的节点
- **下界**：当前路径长度 + 每个未访问城市的最小出边权重
- 比回溯法更快找到最优解（但空间开销更大）

## 十五、平摊分析 (Amortized Analysis)

平摊分析用于分析一系列操作的**总体**时间复杂度，而非单个操作的最坏情况。某些操作虽然偶尔很昂贵，但"平摊"到所有操作上后平均代价很低。

**注意**：平摊分析不是平均情况分析——它给出的是**最坏情况下**一系列操作的总代价的上界，不涉及概率。

### 15.1 聚集分析 (Aggregate Analysis)

计算 n 次操作的**总代价** T(n)，平摊代价 = T(n)/n。

**示例——栈的 MULTIPOP**：
- 栈支持 PUSH、POP、MULTIPOP(k)（弹出 min(k, 栈大小) 个元素）
- 单次 MULTIPOP 最坏 O(n)，但 n 次操作总共最多 push n 次，因此总共最多 pop n 次
- 总代价 T(n) ≤ 2n = O(n)，**平摊代价 O(1)**

**示例——二进制计数器**：
- INCREMENT 操作翻转若干位。位 i 每 2ⁱ 次操作翻转一次
- n 次 INCREMENT 的总翻转次数：n + n/2 + n/4 + ... < 2n
- **平摊代价 O(1)**

### 15.2 记账法 (Accounting Method)

为每种操作分配一个**平摊代价**（可以高于或低于实际代价）。多付的部分作为**信用 (credit)** 存储在数据结构中，少付时用信用补足。要求任何时刻总信用 ≥ 0。

**栈的 MULTIPOP 示例**：
- PUSH 的平摊代价 = 2（1 用于 push，1 存为信用）
- POP 的平摊代价 = 0（用存储的信用支付）
- MULTIPOP 的平摊代价 = 0（每个弹出的元素有 1 单位信用）
- 每次操作平摊代价 ≤ 2 = O(1)

### 15.3 势能法 (Potential Method)

定义**势能函数** Φ(Dᵢ)：数据结构状态 Dᵢ 到实数的映射。

- 第 i 次操作的**平摊代价** = 实际代价 + Φ(Dᵢ) - Φ(Dᵢ₋₁)
- 总平摊代价 = 总实际代价 + Φ(Dₙ) - Φ(D₀)
- 若 Φ(Dₙ) ≥ Φ(D₀)，则总平摊代价是总实际代价的上界

**二进制计数器示例**：
- 势能函数 Φ = 计数器中 1 的个数
- INCREMENT 翻转 tᵢ 个 1→0，1 个 0→1，实际代价 = tᵢ + 1
- ΔΦ = 1 - tᵢ
- 平摊代价 = (tᵢ + 1) + (1 - tᵢ) = 2 = **O(1)**

### 15.4 动态表 (Dynamic Table)

当表满时倍增容量，当表不到 1/4 满时减半容量。

- 插入的平摊代价：O(1)（势能函数 Φ = 2·元素数 - 表大小，扩容时势能释放支付复制代价）
- 删除的平摊代价：O(1)
- 这解释了为什么 C++ `vector` 的 `push_back` 平摊 O(1)

## 十六、网络流 (Network Flow)

### 16.1 基本概念

**流网络**：有向图 G = (V, E)，每条边 (u,v) 有**容量** c(u,v) ≥ 0。有一个**源点 s** 和**汇点 t**。

**可行流**：满足以下条件的函数 f：
1. **容量约束**：0 ≤ f(u,v) ≤ c(u,v)——每条边的流量不超过容量
2. **流量守恒**：对每个内部节点 v（非 s 和 t），流入量 = 流出量

**流的值** |f| = 从源点 s 流出的净流量。

**最大流问题**：在满足约束的前提下，使流的值 |f| 最大。

### 16.2 最大流-最小割定理

**s-t 割**：将顶点集 V 分为 S 和 T = V\S，使 s ∈ S, t ∈ T。割的**容量** = Σ c(u,v)（u ∈ S, v ∈ T 的边容量之和）。

**最大流-最小割定理**（Ford-Fulkerson 定理）：以下三者等价：
1. f 是最大流
2. 残余网络中不存在增广路径
3. |f| = 某个 s-t 割的容量

即 **最大流的值 = 最小割的容量**。

### 16.3 求解算法

#### 残余网络与增广路径

- **残余网络 Gf**：对原网络中的每条边 (u,v)：
  - 若 f(u,v) < c(u,v)，保留前向边，残余容量 c_f(u,v) = c(u,v) - f(u,v)
  - 若 f(u,v) > 0，添加反向边，残余容量 c_f(v,u) = f(u,v)
- **增广路径**：残余网络中从 s 到 t 的一条路径
- 增广路径的**瓶颈容量** = 路径上最小的残余容量

#### Ford-Fulkerson 算法

```
初始化：所有边流量为 0
while 残余网络 Gf 中存在增广路径 P:
    计算路径 P 的瓶颈容量 Δ
    沿 P 增广流量 Δ
return f
```
- 时间复杂度：**O(m · C)**，其中 m = |E|，C = 最大流值（整数容量时）
- 如果容量是无理数，算法可能不终止

#### Edmonds-Karp 算法

Ford-Fulkerson 的改进：每次用 BFS 找**最短增广路径**（边数最少）。

- 时间复杂度：**O(V · E²)**，与容量无关
- 增广路径的长度单调不减，最多增广 O(VE) 次

#### Dinic 算法

进一步改进：构建**层次图**（BFS 分层），在层次图中用 DFS 找阻塞流。

- 时间复杂度：**O(V² · E)**
- 对于单位容量网络：O(E · √V)

### 16.4 应用

- **二分图最大匹配**：建模为网络流，添加超级源点和超级汇点
- **最小路径覆盖**（DAG）
- **项目选择问题**（最大权闭合子图）

## 十七、计算复杂性与 NP 完全理论

### 17.1 问题的下界

**问题的下界**是解决该问题的任何算法在最坏情况下所需时间的下限。

**下界分析技术**：
1. **平凡下界**：基于输入/输出规模。如排序至少需要读 n 个元素 → Ω(n)
2. **决策树模型**：基于比较的算法可以用决策树建模
   - **查找下界**：n 个元素中查找，决策树有 n 个叶子，高度 ≥ ⌈log₂(n+1)⌉ → **Ω(log n)**
   - **排序下界**：n 个元素排序，决策树有 n! 个叶子，高度 ≥ ⌈log₂(n!)⌉ ≈ n log n → **Ω(n log n)**
3. **对手策略**：构造最坏情况输入，迫使算法做更多工作
4. **归约**：若问题 A 可归约到问题 B，则 B 的下界 ≥ A 的下界

**最优算法**：算法的上界与问题的下界匹配时，该算法是最优的。
- 二分查找 O(log n) 对查找问题是最优的
- 归并排序 O(n log n) 对基于比较的排序问题是最优的

### 17.2 P 类与 NP 类

**判定问题**：输出为 Yes/No 的问题。

| 复杂度类 | 定义 | 直觉 |
|---------|------|------|
| **P** | 可以在**多项式时间**内**求解**的判定问题集合 | "容易解的问题" |
| **NP** | 可以在**多项式时间**内**验证**解的判定问题集合 | "容易验证的问题" |
| **NP-hard** | 所有 NP 问题都可以多项式时间归约到它 | "至少和 NP 中最难的一样难" |
| **NP-complete (NPC)** | 既是 NP 又是 NP-hard | "NP 中最难的问题" |

**关键关系**：P ⊆ NP（能解的当然能验证）。**P = NP?** 是计算机科学最重要的未解问题。

### 17.3 多项式时间归约

**多项式时间归约**：问题 A 可多项式归约到问题 B（记为 A ≤_p B），如果存在多项式时间函数 f 将 A 的每个实例映射为 B 的实例，使得 A(x) = Yes 当且仅当 B(f(x)) = Yes。

**意义**：
- 若 A ≤_p B 且 B ∈ P，则 A ∈ P
- 若 A ≤_p B 且 A 是 NP-hard，则 B 也是 NP-hard

### 17.4 证明 NP 完全性

**证明问题 X 是 NPC 的步骤**：
1. 证明 X ∈ NP（给出多项式时间验证算法）
2. 选择一个已知的 NPC 问题 Y
3. 证明 Y ≤_p X（构造多项式时间归约）

### 17.5 经典 NPC 问题与归约链

**Cook-Levin 定理**：**SAT（布尔可满足性问题）是 NP 完全的**。这是第一个被证明的 NPC 问题，证明思路是将任意 NP 问题的非确定性图灵机的计算过程编码为布尔公式。

**重要 NPC 问题及归约链**：

```
SAT
├── 3-SAT（将 SAT 子句限制为恰好 3 个文字）
│   ├── 独立集 (Independent Set)
│   ├── 顶点覆盖 (Vertex Cover)
│   ├── 团 (Clique)
│   ├── 哈密顿回路 (Hamiltonian Cycle)
│   │   └── TSP（旅行商问题）
│   └── 图着色 (Graph Coloring)
├── 精确覆盖 (Exact Cover)
│   └── 子集和 (Subset Sum)
│       └── 0-1 背包 (0-1 Knapsack)
└── 划分问题 (Partition)
```

**经典 NPC 问题定义**：
- **SAT**：给定布尔公式，是否存在使其为真的变量赋值？
- **3-SAT**：SAT 的限制版本，每个子句恰好 3 个文字
- **顶点覆盖**：给定图 G 和整数 k，是否存在大小 ≤ k 的顶点集覆盖所有边？
- **独立集**：给定图 G 和整数 k，是否存在大小 ≥ k 的顶点集，任意两顶点不相邻？
- **团 (Clique)**：给定图 G 和整数 k，是否存在大小 ≥ k 的完全子图？
- **哈密顿回路**：给定图 G，是否存在经过每个顶点恰好一次的回路？
- **子集和**：给定整数集 S 和目标值 W，是否存在子集和恰为 W？

**顶点覆盖与独立集的关系**：S 是图 G 的独立集 ⟺ V\S 是图 G 的顶点覆盖。

### 17.6 NPC 问题的应对策略

既然 NPC 问题在 P ≠ NP 假设下不存在多项式时间精确算法，实际中有以下策略：
1. **近似算法**——在多项式时间内找到近似最优解
2. **启发式算法**——没有理论保证，但实际效果好（如遗传算法、模拟退火）
3. **参数化算法**——当某个参数较小时可以高效求解
4. **特殊情况**——某些 NPC 问题在特殊图/输入结构上可以多项式求解
5. **随机算法**——利用随机性加速

## 十八、近似算法 (Approximation Algorithm)

对于 NP 难的优化问题，近似算法在多项式时间内找到一个"足够好"的解。

### 18.1 近似比

设 A 是优化问题 Π 的近似算法，对实例 I：
- 最小化问题：r_A(I) = A(I) / OPT(I) ≥ 1
- 最大化问题：r_A(I) = OPT(I) / A(I) ≥ 1

A 是 **r-近似算法**：对所有实例 I，r_A(I) ≤ r。r 越接近 1 越好。

### 18.2 可近似性分类

假设 P ≠ NP，NP 难的优化问题按可近似性分为三类：
1. **完全可近似的 (FPTAS)**：对任意 ε > 0，存在 (1+ε)-近似算法，且运行时间关于 1/ε 和 n 都是多项式。例：背包问题
2. **可近似的**：存在常数近似比的多项式时间算法。例：最小顶点覆盖（2-近似）、多机调度
3. **不可近似的**：不存在常数近似比的多项式时间算法（除非 P = NP）。例：一般 TSP

### 18.3 经典近似算法

#### 最小顶点覆盖（2-近似）

**算法 MVC**：
1. V' = 空集
2. 任取一条边 (u,v)，将 u 和 v 都加入 V'
3. 删去 u 和 v 的所有关联边
4. 重复直到没有边

**近似比分析**：设选了 k 条互不关联的边，则 |V'| = 2k。覆盖这 k 条边至少需要 k 个顶点，故 OPT ≥ k。因此 MVC(I)/OPT(I) ≤ 2k/k = **2**。
- 紧实例：k 条互不关联的边构成的图，MVC = 2k，OPT = k

#### 多机调度问题

n 个任务分配给 m 台机器，使最大完成时间（makespan）最小。NP 难问题。

**贪心算法 G-MPS**：按输入顺序，将每个任务分给当前负载最小的机器。
- 近似比：G-MPS(I) ≤ (2 - 1/m) · OPT(I)，即 **(2 - 1/m)-近似算法**
- 证明依赖两个事实：OPT ≥ (1/m)·Σt(a)（平均负载下界）和 OPT ≥ max t(a)（单任务下界）
- 紧实例：m(m-1)+1 个任务，前 m(m-1) 个时间为 1，最后一个时间为 m

**改进贪心 DG-MPS**：先将任务按处理时间**降序排序**，再运行 G-MPS。
- 近似比改进为 **(3/2 - 1/(2m))-近似算法**
- 关键观察：排序后最后分配的任务 aᵢ 满足 t(aᵢ) ≤ OPT/2

#### TSP 近似算法（满足三角不等式）

**最邻近法 (NN)**：从任意城市出发，每步选最近的未访问城市。
- 近似比：O(log n)，不是常数近似比
- 上界：NN(I) ≤ (1/2)(⌈log₂n⌉ + 1) · OPT(I)

**最小生成树法**：求 MST，DFS 遍历得到回路（跳过已访问顶点）。
- 近似比：2（利用三角不等式，MST 权重 ≤ OPT）

**最小权匹配法 (Christofides)**：MST + 奇度顶点的最小权完美匹配 + 欧拉回路。
- 近似比：3/2（目前满足三角不等式的 TSP 最好的近似比）

#### 背包问题的近似方案

**贪心算法**：按单位价值降序装入物品。近似比可以任意差。

**FPTAS（完全多项式时间近似方案）**：
- 将价值缩放：vᵢ' = ⌊vᵢ · (n / (ε · v_max))⌋
- 用动态规划求解缩放后的背包问题
- 时间 O(n³/ε)，近似比 1 + ε

## 十九、随机算法 (Randomized Algorithm)

随机算法在执行过程中使用随机数来做决策。分为两大类。

### 19.1 Las Vegas 算法

**特点**：结果一定正确，但运行时间是随机变量。期望运行时间为多项式。

#### 随机快速排序

随机选择 pivot 元素进行划分。

- **期望比较次数**：T(n) ≤ 2n ln n ≈ 1.39n log₂n
- 证明：随机选取的 pivot 位于排序后第 i 位置的概率为 1/n，得递推 T(n) = (n-1) + (2/n)·Σ T(i)，解为 Θ(n log n)
- 消除了确定性快排对特定输入退化为 O(n²) 的风险

#### 随机选择算法

从 n 个元素中选第 k 小：
1. 随机选 pivot，划分为小于(A)、等于(B)、大于(C) 三组
2. 根据 k 与 |A|, |B| 的关系递归到 A 或 C

- **期望时间** O(n)（最坏情况的概率极小）

#### 随机 n 后放置

将 n 皇后问题的确定性回溯改为随机选择：每行随机选一个与前面不冲突的列。
- 一次运行可能失败（找不到解），重复调用直到成功
- 可与回溯结合：前 stopVegas 行随机放置，剩余用回溯
- n=12 时 stopVegas=5 效率最高（平均访问 80 个节点 vs 纯回溯 262 个）

### 19.2 Monte Carlo 算法

**特点**：运行时间确定（多项式），但结果可能出错。错误概率可控。

#### 主元素测试

判断数组中是否存在出现超过 n/2 次的元素（主元素）。

**算法**：随机选一个元素，检查它是否是主元素。
- 若存在主元素，一次正确概率 > 1/2
- 重复 k 次：错误概率 < (1/2)^k → 每多一次，错误概率减半
- 若要错误概率 ≤ ε，则 k ≥ ⌈log(1/ε)⌉

#### 串相等测试

A 有长串 x，B 有长串 y，A 和 B 要判断 x = y（通信场景）。

**随机算法**：
- 将 x, y 视为整数 I(x), I(y)
- 随机选素数 p < M，计算**指纹** I_p(x) = I(x) mod p
- A 发送 (p, I_p(x)) 给 B，B 检查 I_p(x) = I_p(y)？
- 若 x = y 则必然通过；若 x ≠ y 则可能误判
- 取 M ≥ 2n²（n 为位数），一次测试出错概率 ≤ 1/n
- 重复 ⌈log log n⌉ 次，出错概率 ≤ 1/n^(⌈log log n⌉)
- 每次仅传送 O(log M) = O(log n) 位（而非传送整个长串）

### 19.3 两类随机算法的对比

| 对比 | Las Vegas | Monte Carlo |
|------|-----------|-------------|
| 正确性 | 总是正确 | 可能出错 |
| 运行时间 | 随机（期望多项式） | 确定（多项式） |
| 失败处理 | 重复运行 | 增加重复次数降低出错率 |
| 典型应用 | 随机快排、随机选择 | 主元素测试、素数测试、串相等 |

### 19.4 素数测试（了解）

给定大整数 n，判断是否为素数。Miller-Rabin 测试是 Monte Carlo 算法，一次测试出错概率 ≤ 1/4，重复 k 次出错概率 ≤ (1/4)^k。广泛用于密码学。

## 二十、算法设计技术总结

| 设计技术 | 核心思想 | 适用条件 | 典型问题 | 解题要点 |
|---------|--------|---------|---------|--------|
| **分治** | 分解→递归→合并 | 子问题独立、同类 | 排序、选择、最近点对 | 划分方法、递推方程、合并代价 |
| **动态规划** | 自底向上填表 | 最优子结构、子问题重叠 | 矩阵链、LCS、背包 | 状态定义、转移方程、计算顺序、解的追踪 |
| **贪心** | 局部最优→全局最优 | 贪心选择性质、最优子结构 | 活动选择、Huffman、MST | 贪心策略、正确性证明（交换论证） |
| **回溯** | DFS + 剪枝 | 搜索问题 | n 皇后、背包、TSP | 解空间树、约束条件、限界函数 |
| **分支限界** | BFS/最佳优先 + 限界 | 优化问题的搜索 | TSP、最大团 | 代价函数（上/下界）、优先队列 |
| **网络流** | 增广路径 | 流量/匹配问题 | 最大流、二分匹配 | 残余网络、最大流-最小割定理 |
| **近似算法** | 多项式时间近似解 | NP 难优化问题 | 顶点覆盖、调度、TSP | 近似比分析、紧实例构造 |
| **随机算法** | 引入随机决策 | 消除最坏情况/简化算法 | 快排、主元素、串相等 | 期望分析、错误概率控制 |
