# 数据结构

数据结构研究数据的组织、存储和操作方式。选择合适的数据结构可以极大地影响程序的效率——同一个问题，用数组解可能是 O(n²)，用哈希表解可能是 O(n)。数据结构与算法是计算机科学的两大核心，密不可分。

## 一、线性结构

线性结构中的数据元素之间存在一对一的关系，每个元素最多有一个前驱和一个后继。

### 1.1 数组 (Array)

数组是最基本的数据结构——在内存中占据一段**连续**的空间，所有元素类型相同、大小固定。由于内存连续，可以通过**首地址 + 下标 × 元素大小**直接计算任意元素的地址，实现 O(1) 的随机访问。

- 随机访问：O(1)——这是数组最大的优势
- 插入/删除：O(n)——需要移动大量元素来维持连续性
- 查找（无序）：O(n)——需要逐个比较
- 查找（有序）：O(log n)——可以使用二分查找
- 适合：频繁随机访问、元素数量已知或变化不大的场景

### 1.2 链表 (Linked List)

链表通过**指针**将分散在内存各处的节点串联起来。每个节点包含数据域和指针域。链表不需要连续内存，可以灵活地插入和删除，但失去了随机访问的能力。

| 类型 | 结构 | 特点 |
|------|------|------|
| **单链表** | 每个节点有 data + next 指针 | 只能从前向后遍历 |
| **双链表** | 每个节点有 prev + data + next 指针 | 可以双向遍历，删除更方便（不需要找前驱） |
| **循环链表** | 尾节点指向头节点（形成环） | 适合需要循环访问的场景（如约瑟夫问题） |

- 插入/删除（已知位置）：O(1)——只需修改指针
- 查找：O(n)——必须从头遍历
- 不支持随机访问
- 适合：频繁插入删除、元素数量不确定的场景

**数组 vs 链表**：

| 对比 | 数组 | 链表 |
|------|------|------|
| 内存布局 | 连续 | 离散 |
| 随机访问 | O(1) | O(n) |
| 插入/删除 | O(n) | O(1)（已知位置） |
| 内存利用 | 可能浪费（预分配） | 精确分配，但指针占用额外空间 |
| Cache 友好 | 好（空间局部性） | 差（节点分散在内存中） |

### 1.3 栈 (Stack)

栈是一种**后进先出 (LIFO, Last In First Out)** 的线性表，只能在一端（栈顶）进行插入和删除。就像一摞盘子——最后放上去的最先拿走。

操作：push（入栈）、pop（出栈）、top/peek（查看栈顶），均为 O(1)。

**典型应用**：
- **括号匹配**：遇到左括号入栈，遇到右括号时弹出栈顶并检查是否匹配。若最终栈空则匹配成功
- **表达式求值**：
  - 中缀转后缀：用运算符栈，遇到运算符时弹出栈中优先级 ≥ 当前的运算符
  - 后缀表达式求值：用操作数栈，遇到数字入栈，遇到运算符弹出两个操作数计算后结果入栈
- **函数调用栈**：每次函数调用创建栈帧保存返回地址和局部变量，函数返回时弹出栈帧
- **DFS（深度优先搜索）**：递归本质上使用系统栈，也可以用显式栈模拟

### 1.4 队列 (Queue)

队列是一种**先进先出 (FIFO, First In First Out)** 的线性表，从一端（队尾）插入，从另一端（队首）删除。就像排队买票——先来的先处理。

操作：enqueue（入队）、dequeue（出队）、front（队首），均为 O(1)。

| 类型 | 特点 | 应用 |
|------|------|------|
| **普通队列** | 数组实现时有**假溢出**问题（队首前方有空位但无法使用） | — |
| **循环队列** | 用取模运算解决假溢出。判满：`(rear+1) % size == front`（牺牲一个位置区分空和满） | 操作系统缓冲区 |
| **双端队列 (Deque)** | 两端都可以入出 | C++ STL `deque` |
| **优先队列 (Priority Queue)** | 按**优先级**出队而非先进先出，通常用**堆**实现 | Dijkstra 算法、任务调度 |

## 二、树

树是一种**层次结构**的数据结构，由节点和边组成。每个节点最多有一个父节点（根节点无父节点），可以有零个或多个子节点。树在文件系统（目录结构）、数据库（索引）、编译器（语法树）等领域有广泛应用。

### 2.1 二叉树基础

**二叉树 (Binary Tree)** 是每个节点最多有两个子节点（左子和右子）的树。二叉树是最常用的树结构，许多复杂的树（如 B 树）在概念上都可以用二叉树来理解。

**重要性质**：
- 第 i 层最多有 2^(i-1) 个节点（根为第 1 层）
- 深度为 k 的二叉树最多有 2^k - 1 个节点（此时称为**满二叉树**）
- **n₀ = n₂ + 1**：叶子节点数 = 度为 2 的节点数 + 1（利用总边数 = 总节点数 - 1 推导）

**完全二叉树**：除最后一层外每层都是满的，最后一层从左到右连续填充。完全二叉树特别适合用数组存储——节点 i 的左子 = 2i，右子 = 2i+1，父 = ⌊i/2⌋。堆就是一种完全二叉树。

**四种遍历方式**：

| 遍历 | 访问顺序 | 典型应用 | 递归实现思路 |
|------|----------|----------|------------|
| **前序 (Preorder)** | 根 → 左 → 右 | 复制树、序列化、前缀表达式 | visit(root); preorder(left); preorder(right) |
| **中序 (Inorder)** | 左 → 根 → 右 | BST 有序输出、中缀表达式 | inorder(left); visit(root); inorder(right) |
| **后序 (Postorder)** | 左 → 右 → 根 | 删除树、计算目录大小、后缀表达式 | postorder(left); postorder(right); visit(root) |
| **层序 (Level-order)** | 从上到下、从左到右逐层 | BFS、完全二叉树判定 | 用队列实现 |

**由遍历序列还原二叉树**：
- **前序 + 中序** → 唯一确定一棵二叉树。前序的第一个元素是根，在中序中找到根的位置将中序分为左右子树，递归处理
- **后序 + 中序** → 唯一确定。后序的最后一个元素是根，同理
- **前序 + 后序** → 不能唯一确定（除非是满二叉树）。因为无法确定只有一个子节点时是左子还是右子

### 2.2 二叉搜索树 (BST, Binary Search Tree)

BST 是一种有序的二叉树，核心性质：对于任意节点，**左子树所有节点的值 < 节点值 < 右子树所有节点的值**。这使得 BST 能像有序数组一样进行二分查找，同时像链表一样灵活地插入删除。

| 操作 | 平均时间 | 最坏时间（退化为链表） | 说明 |
|------|----------|----------------------|------|
| 查找 | O(log n) | O(n) | 从根开始，比节点小往左走，比节点大往右走 |
| 插入 | O(log n) | O(n) | 查找到空位后插入为叶子节点 |
| 删除 | O(log n) | O(n) | 最复杂的操作（见下文） |

**删除操作**的三种情况：
- 删除**叶子节点**：直接删除
- 删除**只有一个子树**的节点：用子树替代被删节点
- 删除**有两个子树**的节点：用**中序后继**（右子树的最小值）或**中序前驱**（左子树的最大值）替代被删节点的值，然后删除那个后继/前驱节点

BST 的问题：当数据按有序顺序插入时（如 1,2,3,4,5），BST 退化为链表，所有操作变为 O(n)。这就是平衡二叉树（AVL、红黑树）要解决的问题。

### 2.3 AVL 树

AVL 树是最早提出的**自平衡二叉搜索树**（由 Adelson-Velsky 和 Landis 在 1962 年提出）。

**平衡条件**：任意节点的左右子树**高度差（平衡因子）不超过 1**。即 |height(left) - height(right)| ≤ 1。

当插入或删除操作导致某个节点的平衡因子超过 1 时，通过**旋转**操作恢复平衡。旋转分为四种情况：

| 失衡类型 | 形状 | 旋转方式 | 说明 |
|----------|------|----------|------|
| **LL** | 左子树的左子树过高 | 对失衡节点**右旋** | 一次旋转 |
| **RR** | 右子树的右子树过高 | 对失衡节点**左旋** | 一次旋转 |
| **LR** | 左子树的右子树过高 | 先对左子**左旋**，再对失衡节点**右旋** | 两次旋转 |
| **RL** | 右子树的左子树过高 | 先对右子**右旋**，再对失衡节点**左旋** | 两次旋转 |

AVL 树保证所有操作的时间复杂度严格为 O(log n)。

### 2.4 红黑树

红黑树是另一种自平衡 BST，比 AVL 树的平衡条件更宽松，因此插入删除时需要的旋转次数更少（最多 3 次），适合插入删除频繁的场景。

**五条性质**（红黑树的不变量）：
1. 每个节点要么是**红色**要么是**黑色**
2. **根节点**是黑色
3. 所有**叶子节点（NIL/空节点）** 是黑色
4. 红色节点的子节点必须是黑色（**不能有连续的红节点**）
5. 从任一节点到其所有叶子的路径上，**黑色节点数相同**（称为黑高度）

性质 4 和 5 共同保证了最长路径不超过最短路径的两倍，因此红黑树是"近似平衡"的。

**AVL 树 vs 红黑树**：

| 对比 | AVL 树 | 红黑树 |
|------|--------|--------|
| 平衡条件 | 严格平衡（高度差 ≤ 1） | 近似平衡（最长路径 ≤ 2×最短路径） |
| 树高度 | 更矮（1.44 log n） | 稍高（2 log n） |
| 查找性能 | 更快（树更矮，比较次数少） | 稍慢 |
| 插入/删除 | 可能需要 O(log n) 次旋转 | 最多 3 次旋转（但可能需要 O(log n) 次重着色） |
| 适用场景 | 查找密集型（如数据库索引） | 插入删除频繁型（如内存中的有序集合） |
| 典型应用 | 数据库、Windows 内存管理 | Java TreeMap/TreeSet、C++ std::map、Linux CFS 调度器、epoll |

### 2.5 B 树 / B+ 树

B 树和 B+ 树是为**磁盘存储**优化的多路搜索树。与二叉树不同，B 树的每个节点可以有多个键和多个子节点，从而降低树的高度、减少磁盘 I/O 次数。

**B 树（m 阶）** 的性质：
- 每个节点最多有 **m 个子节点**、**m-1 个关键字**
- 每个非根节点至少有 **⌈m/2⌉ 个子节点**（保证节点不会太空）
- 所有**叶子节点在同一层**（完美平衡）
- 关键字在节点内有序排列，子节点按关键字范围划分

**B+ 树**：B 树的变种，在数据库索引中更常用。与 B 树的关键区别：
- **内部节点只存索引**（不存数据），数据全部在叶子节点
- **叶子节点通过指针链接成有序链表**——支持高效的范围查询
- 内部节点可以存储更多关键字（因为不存数据），树更矮

详见数据库篇。

### 2.6 哈夫曼树与哈夫曼编码

**哈夫曼树 (Huffman Tree)** 是一种**最优二叉树**——带权路径长度 (WPL) 最小的二叉树。WPL = 所有叶子节点的"权值 × 根到该叶子的路径长度"之和。

**构建算法**（贪心）：
1. 将所有节点放入一个最小优先队列
2. 每次取出权值**最小的两个**节点，合并为一个新节点（权值为两者之和）
3. 将新节点放回优先队列
4. 重复直到只剩一个节点（根节点）

**哈夫曼编码**：哈夫曼树的重要应用。为每个字符分配变长编码——左分支为 0，右分支为 1。从根到叶子的路径就是该字符的编码。

关键性质：
- **前缀编码**：任何字符的编码都不是另一个字符编码的前缀，因此解码无歧义
- **频率高的字符编码短，频率低的字符编码长**——实现最优压缩
- 哈夫曼编码被用于 JPEG、DEFLATE（zip/gzip）等压缩算法中

### 2.7 字典树 (Trie)

字典树（也称前缀树）是一种用于高效存储和查找字符串的树结构。每条边代表一个字符，从根到某节点的路径拼接起来形成一个字符串的前缀。

- 查找/插入时间 O(L)，L 为字符串长度（与字符串总数无关！）
- 空间开销较大（每个节点最多有 |字符集| 个子节点）
- 适合：前缀匹配（自动补全）、词频统计、拼写检查、IP 路由查找

## 三、堆 (Heap)

堆是一种特殊的**完全二叉树**，满足堆序性质。堆是实现优先队列的标准数据结构。

### 3.1 基本概念

- **最大堆 (Max Heap)**：每个节点的值 ≥ 其所有子节点的值（根节点是最大值）
- **最小堆 (Min Heap)**：每个节点的值 ≤ 其所有子节点的值（根节点是最小值）
- 堆是完全二叉树，因此用**数组**存储最为紧凑高效：节点 i 的左子 = 2i+1，右子 = 2i+2，父 = (i-1)/2

### 3.2 核心操作

| 操作 | 方法 | 时间复杂度 | 说明 |
|------|------|-----------|------|
| **插入** | 放到数组末尾（堆的最后），然后**上浮 (sift up)**——与父节点比较，如果违反堆序则交换，重复 | O(log n) | 最多上浮到根 |
| **删除堆顶** | 用数组末尾元素替换堆顶，然后**下沉 (sift down)**——与较大/较小的子节点交换，重复 | O(log n) | 最多下沉到叶子 |
| **建堆** | 从最后一个**非叶节点**开始，逐个**下沉** | **O(n)** | 看似 O(n log n)，但数学分析为 O(n) |
| **取堆顶** | 直接返回 arr[0] | O(1) | 堆的最大优势 |

**建堆为什么是 O(n)**：虽然每个节点下沉最多 O(log n)，但大部分节点在底层（叶子层不需要下沉，倒数第二层最多下沉 1 层...），总工作量的精确分析是 O(n)。

### 3.3 堆排序

堆排序利用堆的性质实现原地 O(n log n) 排序：

1. **建最大堆**：O(n)
2. 重复 n-1 次：将堆顶（最大值）与当前末尾交换，堆大小减 1，对新堆顶执行下沉。每次 O(log n)
3. 总时间 O(n log n)，空间 O(1)（原地排序），**不稳定**

### 3.4 优先队列

优先队列是一种抽象数据类型，支持两个核心操作：插入元素和取出最高优先级的元素。底层通常用堆实现。

- 插入：O(log n)
- 取出最值：O(1)
- 删除最值：O(log n)
- C++ STL：`priority_queue`（默认最大堆）

应用场景：Dijkstra 最短路算法、Prim 最小生成树、哈夫曼编码、任务调度、TopK 问题。

## 四、图

图是最一般的数据结构，可以表示任意的多对多关系。图由**顶点 (Vertex)** 和**边 (Edge)** 组成，边连接两个顶点表示它们之间的关系。社交网络、互联网、交通网络、依赖关系等都可以用图来建模。

### 4.1 图的存储

| 方式 | 空间 | 查边 (u,v) | 遍历 u 的邻接点 | 适用 |
|------|------|-----------|----------------|------|
| **邻接矩阵** | O(V²) | O(1)——直接查 M[u][v] | O(V)——扫描整行 | **稠密图**（边数接近 V²） |
| **邻接表** | O(V+E) | O(度)——遍历 u 的邻接链表 | O(度) | **稀疏图**（边数远小于 V²）——大多数实际图 |

邻接矩阵用二维数组表示，M[i][j] = 1（或权重）表示 i 到 j 有边。空间浪费在稀疏图上。

邻接表为每个顶点维护一个链表（或动态数组），存储所有与该顶点相邻的顶点。空间利用率高。

### 4.2 DFS 与 BFS

DFS 和 BFS 是图的两种基本遍历方式，是几乎所有图算法的基础。

| 对比 | DFS (深度优先搜索) | BFS (广度优先搜索) |
|------|-----|-----|
| 策略 | 沿一条路径尽可能深入，走到底再回溯 | 从起点出发逐层向外扩展 |
| 数据结构 | **栈**（递归使用系统栈，也可用显式栈） | **队列** |
| 空间复杂度 | O(V)（递归深度） | O(V)（队列中的节点数） |
| 时间复杂度 | O(V+E)（邻接表） | O(V+E)（邻接表） |
| 应用 | 连通性判断、拓扑排序、强连通分量(Tarjan)、回溯搜索、欧拉路径 | **最短路径（无权图）**、层序遍历、多源 BFS |

**关键区别**：BFS 找到的路径一定是最短路径（在无权图中），DFS 不保证。

### 4.3 最小生成树 (MST)

对于一个连通加权无向图，**最小生成树**是包含所有顶点、边权之和最小的树（n 个顶点，n-1 条边，无环）。

| 算法 | 策略 | 时间复杂度 | 适用场景 |
|------|------|-----------|----------|
| **Prim** | 从一个顶点出发，每次将**距已选顶点集最近的**未选顶点加入 | O(V²)（朴素）或 O(E log V)（堆优化） | **稠密图**（朴素版 O(V²) 与边数无关） |
| **Kruskal** | 将所有边按权重排序，每次选**最小权重**且**不形成环**的边加入 | O(E log E) | **稀疏图**（只需对边排序） |

Prim 类似 Dijkstra（贪心选最近的顶点）；Kruskal 使用**并查集**高效判断加入一条边是否会形成环。

**切割定理 (Cut Property)**：对于图的任意切割（将顶点分为两个不相交集合），横跨切割的最小权重边一定属于某棵最小生成树。这是 Prim 和 Kruskal 正确性的理论基础。

### 4.4 最短路径

最短路径问题是图论中最重要的问题之一。

| 算法 | 问题类型 | 时间复杂度 | 适用条件 | 核心思想 |
|------|----------|-----------|----------|----------|
| **Dijkstra** | 单源最短路 | O(V²) 朴素 / O(E log V) 堆优化 | **无负权边** | 贪心——每次选距离最小的未确定顶点 |
| **Bellman-Ford** | 单源最短路 | O(VE) | 可以有**负权边** | 动态规划——对所有边松弛 V-1 轮。第 V 轮若还能松弛则存在负权环 |
| **SPFA** | 单源最短路 | 平均 O(kE)，k 常数 | 可以有负权边 | BF 的优化，用队列记录被更新的顶点（最坏仍 O(VE)，可被卡） |
| **Floyd** | **全源**最短路 | O(V³) | 可以有负权边 | 动态规划——枚举中间顶点 k，d[i][j] = min(d[i][j], d[i][k]+d[k][j]) |

**松弛操作 (Relaxation)**：如果 d[u] + w(u,v) < d[v]，则更新 d[v] = d[u] + w(u,v)。这是所有最短路算法的核心操作。

### 4.5 拓扑排序

对**有向无环图 (DAG)** 的所有顶点排成线性序列，使得对每条有向边 (u,v)，u 在 v 前面。拓扑排序反映了事件的先后依赖关系（如课程先修关系、编译依赖）。

**Kahn 算法**（BFS 方法）：
1. 将所有入度为 0 的顶点入队（它们没有前驱，可以先执行）
2. 出队一个顶点，加入结果序列；删除该顶点的所有出边，更新相关顶点的入度
3. 新产生的入度为 0 的顶点入队
4. 重复直到队列为空。如果结果序列包含所有顶点则无环，否则图中有环

时间复杂度 O(V+E)。拓扑排序结果不唯一。

### 4.6 关键路径

关键路径用于分析项目管理中的工期问题。在 **AOE (Activity On Edge) 网**中：
- 顶点代表事件（里程碑），边代表活动（任务），边的权重代表活动的持续时间
- **关键路径**是从源点到汇点的**最长路径**，决定了项目的**最短完成时间**
- 关键路径上的活动称为**关键活动**，其特征是**最早开始时间 = 最迟开始时间**（没有任何松弛余量，延误就会影响整个工期）

计算步骤：
1. 正向求各事件的**最早发生时间 (Earliest Time)**
2. 反向求各事件的**最迟发生时间 (Latest Time)**
3. 对每个活动计算最早/最迟开始时间，两者相等的就是关键活动

## 五、散列 (Hash)

散列（哈希）的核心思想是：通过一个函数将**键 (Key)** 直接映射为存储位置（数组下标），从而实现**平均 O(1)** 的查找、插入和删除。这比有序数组的 O(log n) 和链表的 O(n) 快得多。

### 5.1 散列函数

好的散列函数应该使键值均匀分布在散列表中，减少冲突。

- **除留余数法**：H(key) = key % p，p 通常取**不大于表长的最大素数**（素数可以减少冲突模式）。这是最常用的方法
- **数字分析法**：分析键值中各位数字的分布，选择分布均匀的位作为散列地址
- **平方取中法**：取键值平方后的中间几位
- **折叠法**：将键值分为几段相加

### 5.2 冲突处理

不同的键可能映射到同一位置（**冲突/碰撞**），必须有处理机制。

| 方法 | 描述 | 优缺点 |
|------|------|--------|
| **开放寻址—线性探测** | H(key)+1, +2, +3, ... | 简单，但产生**聚集 (clustering)**——冲突的元素连成一片，加剧后续冲突 |
| **开放寻址—二次探测** | H(key)+1², -1², +2², -2², ... | 减轻聚集，但可能访问不到所有位置 |
| **开放寻址—双散列** | H(key)+i×H₂(key)，H₂ 是另一个散列函数 | 效果好，几乎消除聚集 |
| **链地址法（拉链法）** | 每个槽是一个链表，冲突元素挂在同一链表上 | **最常用**（Java HashMap）。无聚集，装填因子可 > 1 |

### 5.3 装填因子与性能

**装填因子 α = 已存元素数 / 表长**。α 越大，冲突越严重，性能越差。

- 链地址法：α 可以 > 1（每个槽可以挂多个元素），但 α 过大时链表变长，退化为 O(n)。Java HashMap 在 α > 0.75 时扩容
- 开放寻址法：α 必须 < 1（因为需要空位放新元素），通常保持 α < 0.75
- 当 α 过大时需要**扩容 (rehash)**——创建更大的表，重新插入所有元素

## 六、排序算法对比

排序是最基本的算法问题。不同排序算法在时间复杂度、空间复杂度和稳定性上各有特点，选择合适的算法取决于具体场景。

**稳定性**：如果排序前两个相等的元素 a 和 b 的相对顺序是 a 在 b 前，排序后 a 仍在 b 前，则该排序算法是**稳定的**。稳定性在多关键字排序时很重要。

| 算法 | 平均 | 最好 | 最坏 | 空间 | 稳定 | 核心思想 |
|------|------|------|------|------|------|----------|
| **冒泡排序** | O(n²) | O(n) | O(n²) | O(1) | 稳定 | 相邻元素两两比较交换，每轮将最大值冒泡到末尾 |
| **选择排序** | O(n²) | O(n²) | O(n²) | O(1) | 不稳定 | 每轮选出最小值放到已排序部分末尾 |
| **插入排序** | O(n²) | O(n) | O(n²) | O(1) | 稳定 | 将元素插入已排序部分的正确位置（类似整理扑克牌） |
| **希尔排序** | O(n^1.3) | O(n) | O(n²) | O(1) | 不稳定 | 插入排序的改进，先用大步长使数组大致有序，再逐步缩小步长 |
| **归并排序** | O(n log n) | O(n log n) | O(n log n) | O(n) | 稳定 | 分治——将数组分为两半分别排序后合并 |
| **快速排序** | O(n log n) | O(n log n) | O(n²) | O(log n) | 不稳定 | 分治——选 pivot 将数组分为小于和大于两部分，递归 |
| **堆排序** | O(n log n) | O(n log n) | O(n log n) | O(1) | 不稳定 | 建堆后反复取堆顶并下沉 |
| **计数排序** | O(n+k) | O(n+k) | O(n+k) | O(k) | 稳定 | 统计每个值出现次数，直接算出位置。k 为值域范围 |
| **基数排序** | O(d(n+k)) | O(d(n+k)) | O(d(n+k)) | O(n+k) | 稳定 | 按位排序（低位到高位），每位用稳定排序。d 为位数 |
| **桶排序** | O(n+k) | O(n) | O(n²) | O(n+k) | 稳定 | 将元素分到多个桶中分别排序后合并 |

**选择依据**：
- **数据量小**（n < 50）：插入排序（常数小，代码简单，对小数据集很快）
- **一般用途**：**快速排序**（平均最快，大多数标准库使用）
- **需要稳定**：**归并排序**（O(n log n) 的稳定排序中最常用）
- **空间受限**：**堆排序**（O(1) 额外空间的 O(n log n) 排序）
- **数据范围有限**：**计数排序 / 基数排序**（线性时间，但需要值域不太大）
- **数据基本有序**：插入排序（接近 O(n)）

**快速排序的最坏情况优化**：
- **随机选取 pivot**：避免对有序数据退化
- **三数取中法**：取首、尾、中间三个元素的中位数作为 pivot
- **混合排序**：当子数组长度小于阈值时改用插入排序（如 C++ STL 的 `std::sort` 使用 IntroSort = 快排 + 堆排 + 插入排序）

## 七、查找算法

| 算法 | 时间复杂度 | 要求 | 适用场景 |
|------|-----------|------|----------|
| **顺序查找** | O(n) | 无 | 无序小数据集 |
| **二分查找** | O(log n) | **有序**数组 | 有序静态数据 |
| **插值查找** | O(log log n) 平均 | 有序且**均匀分布** | 数据分布均匀时 |
| **散列查找** | O(1) 平均 | 散列表 | 等值查找（最快） |
| **BST 查找** | O(log n) 平均 | BST | 需要有序性的动态数据 |

**二分查找**是最重要的查找算法。核心思想：每次将搜索范围缩小一半。要求数据**有序**且支持**随机访问**（因此只能用于数组，不能用于链表）。

## 八、算法分析

### 8.1 时间复杂度

时间复杂度用**大 O 记号**描述算法运行时间随输入规模增长的趋势。只关注最高阶项，忽略常数和低阶项。

常见复杂度从快到慢：O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)

| 复杂度 | 名称 | 典型算法 | n=10⁶ 的数量级 |
|--------|------|----------|---------------|
| O(1) | 常数 | 数组访问、哈希查找 | 1 |
| O(log n) | 对数 | 二分查找 | 20 |
| O(n) | 线性 | 遍历、线性查找 | 10⁶ |
| O(n log n) | 线性对数 | 快排、归并排序 | 2×10⁷ |
| O(n²) | 平方 | 冒泡排序、暴力二重循环 | 10¹² (超时) |
| O(2ⁿ) | 指数 | 子集枚举 | 天文数字 |

### 8.2 主定理 (Master Theorem)

主定理用于求解**分治递推关系**的时间复杂度。对于形如 T(n) = aT(n/b) + O(n^d) 的递推：

| 条件 | 复杂度 | 直觉理解 |
|------|--------|----------|
| d < log_b(a) | T(n) = O(n^(log_b(a))) | 叶子层的工作量主导 |
| d = log_b(a) | T(n) = O(n^d × log n) | 每层工作量相同 |
| d > log_b(a) | T(n) = O(n^d) | 根层的工作量主导 |

**常见应用**：
- **二分查找**：T(n) = T(n/2) + O(1) → a=1, b=2, d=0 → d = log₂1 = 0 → O(log n)
- **归并排序**：T(n) = 2T(n/2) + O(n) → a=2, b=2, d=1 → d = log₂2 = 1 → O(n log n)
- **Karatsuba 乘法**：T(n) = 3T(n/2) + O(n) → a=3, b=2, d=1 → d < log₂3 ≈ 1.58 → O(n^1.58)
- **Strassen 矩阵乘法**：T(n) = 7T(n/2) + O(n²) → O(n^(log₂7)) ≈ O(n^2.81)
