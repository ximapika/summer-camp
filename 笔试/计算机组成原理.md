# 计算机组成原理

计算机组成原理研究计算机硬件系统的基本组成和工作原理。理解计组有助于编写更高效的代码——了解 Cache 就知道为什么按行遍历数组比按列快，了解流水线就知道为什么分支预测对性能影响巨大。

## 一、数据表示

计算机中所有的数据（数字、文字、图像、声音）最终都是用二进制（0 和 1）表示的。数值在计算机中有多种编码方式，不同编码方式决定了计算机如何进行算术运算。

### 1.1 数值编码

计算机中的带符号整数使用最高位作为符号位（0 正 1 负），但如何表示数值部分有多种编码方式。

**原码**：最直观的编码方式。符号位 + 绝对值的二进制表示。例如 +5 = 0101，-5 = 1101（4 位）。
- 缺点：加减法运算需要先判断符号再决定操作，+0 (0000) 和 -0 (1000) 有两种表示

**反码**：正数同原码，负数将原码除符号位外各位取反。例如 -5 = 1010。
- 仍然有 +0 和 -0 两种表示，主要作为求补码的中间步骤

**补码**：现代计算机实际使用的编码。正数同原码，负数 = 反码 + 1。例如 -5 = 1011。
- **核心优势**：0 的表示唯一（0000），加减法统一为加法运算（符号位参与运算），硬件实现简单
- 多表示一个负数：n 位补码范围为 [-2^(n-1), 2^(n-1)-1]，例如 8 位 [-128, 127]
- [-128] 的补码 = 1000 0000（没有对应的原码和反码）
- **补码的补码 = 原码**

**移码**：补码的符号位取反。例如 +5 的补码 0101 → 移码 1101；-5 的补码 1011 → 移码 0011。
- 特点：移码的数值大小顺序与无符号整数一致，**便于直接比较大小**
- 应用：IEEE 754 浮点数的阶码使用移码表示

| 编码 | 正数 | 负数 | 范围 (n位) | 零的表示 | 应用 |
|------|------|------|-----------|----------|------|
| 原码 | 0+数值 | 1+数值 | -(2^(n-1)-1) ~ 2^(n-1)-1 | +0 和 -0 | 乘除法 |
| 反码 | 同原码 | 符号位1，其余取反 | 同原码 | +0 和 -0 | 求补码的中间步骤 |
| 补码 | 同原码 | 反码+1 | -2^(n-1) ~ 2^(n-1)-1 | 唯一的 0 | 加减法、存储 |
| 移码 | 补码符号位取反 | 补码符号位取反 | 同补码 | 唯一 | 浮点数阶码 |

### 1.2 IEEE 754 浮点数

浮点数用于表示实数（带小数点的数），采用科学计数法的思想：一个数 = 符号 × 尾数 × 2^指数。

**格式**：`(-1)^S × 1.M × 2^(E-Bias)`

- S：符号位（0 正 1 负）
- E：阶码（指数部分），用移码表示（加上偏置值 Bias）
- M：尾数（有效数字部分），隐含前导 1（规格化数）

| 格式 | 总位数 | S | E(阶码) | M(尾数) | Bias | 有效位数 | 十进制精度 |
|------|--------|---|---------|---------|------|----------|-----------|
| 单精度 float | 32 | 1 | 8 | 23 | 127 | 24 位 | 约 7 位 |
| 双精度 double | 64 | 1 | 11 | 52 | 1023 | 53 位 | 约 15-16 位 |

**举例**：十进制 -6.5 的 float 表示
1. -6.5 = -110.1₂ = -1.101 × 2²
2. S = 1（负数），E = 2 + 127 = 129 = 10000001₂，M = 10100...0
3. 存储：1 10000001 10100000000000000000000

**特殊值**：阶码 E 全 0 或全 1 时表示特殊数值：

| 阶码 E | 尾数 M | 表示 | 含义 |
|--------|--------|------|------|
| 全 0 | 全 0 | ±0 | 正零和负零 |
| 全 0 | ≠ 0 | 非规格化数 | 用于表示极其接近 0 的小数（隐含位为 0 而非 1） |
| 全 1 | 全 0 | ±∞ | 正无穷和负无穷（如 1.0/0.0） |
| 全 1 | ≠ 0 | NaN | Not a Number（如 0.0/0.0、sqrt(-1)） |

**浮点运算步骤**：
1. **对阶**：将两个操作数的阶码调整为相同（小阶对大阶，避免高位丢失）
2. **尾数运算**：对齐后的尾数进行加减
3. **规格化**：调整结果使其满足 1.xxx 的形式（左规或右规）
4. **舍入**：因为尾数位数有限，需要舍入。常用方法：就近舍入（银行家舍入）、向零舍入
5. **溢出判断**：阶码超出范围则上溢（→ ±∞）或下溢（→ 0 或非规格化数）

**浮点数精度问题**：0.1 + 0.2 ≠ 0.3（在大多数编程语言中），因为 0.1 在二进制中是无限循环小数，存储时产生舍入误差。

### 1.3 溢出检测

整数运算结果超出表示范围时发生溢出。CPU 需要检测溢出以便处理（如触发异常）。

- **单符号位法**：两个同号数相加结果异号 → 溢出。具体地：正+正=负（正溢出）或 负+负=正（负溢出）
- **双符号位法（变形补码）**：用两位表示符号。正常结果符号位为 00(正) 或 11(负)；如果出现 01(正溢出) 或 10(负溢出) 则溢出。第一位是真正的符号位
- **进位判断法**：最高数据位向符号位的进位 C₁ 与符号位的进位 C₀ 不同时溢出。即 C₁ ⊕ C₀ = 1 → 溢出

## 二、运算器

运算器（也称 ALU，Arithmetic Logic Unit）是 CPU 中执行算术和逻辑运算的核心部件。

### 2.1 加法器

加法是最基本的运算，减法可以用加法实现（减去一个数 = 加上它的补码），乘法和除法也可以分解为移位和加法。因此加法器是运算器的核心。

- **串行进位加法器 (Ripple Carry Adder, RCA)**：n 位加法器由 n 个全加器级联，每个全加器的进位输出连接到下一个全加器的进位输入。进位逐级传递（"涟漪式"），总延迟与位数成正比 O(n)。结构简单但速度慢
- **超前进位加法器 (Carry Lookahead Adder, CLA)**：通过预先计算所有位的进位来加速。定义：
  - 生成函数：G_i = A_i · B_i（该位自己产生进位）
  - 传播函数：P_i = A_i ⊕ B_i（该位传播来自低位的进位）
  - 进位公式：C_i = G_i + P_i · C_{i-1}（展开后所有进位只依赖初始进位 C₀）
  - 所有位的进位可以并行计算，延迟 O(log n)

### 2.2 乘法器与除法器

- **原码一位乘法**：符号位单独异或确定，绝对值部分用"加法+右移"方式逐位相乘
- **补码一位乘法 (Booth 算法)**：直接对补码进行乘法，不需要先转换为原码。根据乘数**末两位**决定操作：
  - 00 或 11：只右移
  - 01：加被乘数，再右移
  - 10：减被乘数，再右移
  - Booth 算法可以处理负数乘法，且当乘数中有连续的 1 时效率更高
- **除法器**：
  - 原码恢复余数除法：试商后如果余数为负，恢复（加回除数）
  - 不恢复余数除法（加减交替法）：余数为正则左移后减除数，余数为负则左移后加除数。效率更高

### 2.3 ALU

算术逻辑单元是运算器的核心，由加法器 + 逻辑电路 + 多路选择器组成。能执行：
- **算术运算**：加、减、乘、除、增量、减量
- **逻辑运算**：与 (AND)、或 (OR)、非 (NOT)、异或 (XOR)
- **移位运算**：逻辑移位、算术移位、循环移位

ALU 通过控制信号选择执行哪种运算，并输出运算结果和状态标志（零标志 ZF、进位标志 CF、溢出标志 OF、符号标志 SF）。

## 三、存储系统

### 3.1 存储器层次结构

计算机使用多级存储层次来平衡速度、容量和成本之间的矛盾。越靠近 CPU 的存储越快、越贵、容量越小。

```
寄存器 ← 最快（< 1ns）、最贵、最小（几十~几百个）
  ↕ 
Cache（高速缓存）← 快（1-10ns）、SRAM、KB~MB 级
  ↕
主存（内存）← 较快（50-100ns）、DRAM、GB 级
  ↕
外存（磁盘/SSD）← 慢（ms 级/μs 级）、最便宜、TB 级
```

这种层次结构之所以有效，是因为**局部性原理**：
- **时间局部性**：刚被访问的数据很可能很快再次被访问（如循环变量）
- **空间局部性**：刚被访问的数据附近的数据很可能很快被访问（如数组遍历）

### 3.2 SRAM vs DRAM

主存和 Cache 使用不同的半导体存储技术。

| 对比 | SRAM (静态随机存储器) | DRAM (动态随机存储器) |
|------|------|------|
| 存储元件 | 双稳态触发器（通常 6 个晶体管） | 微小电容 + 1 个晶体管 (1T1C) |
| 速度 | 快（与 CPU 速度接近） | 较慢（需要充放电） |
| 刷新 | 不需要（触发器能保持状态） | **需要定期刷新**（电容会漏电，通常每 64ms 全部刷新一次） |
| 集成度 | 低（每个存储单元需要更多晶体管） | 高（结构简单，可以做得很密集） |
| 成本 | 高 | 低 |
| 功耗 | 低（静态功耗小） | 较高（刷新消耗能量） |
| 用途 | **Cache**（容量小但需要快） | **主存**（容量大但可以稍慢） |

### 3.3 Cache（高速缓存）

Cache 是 CPU 和主存之间的高速缓冲存储器，用于缓解 CPU 和主存之间巨大的速度差距。CPU 访问数据时先查 Cache，命中则直接获取（快），未命中则从主存加载到 Cache 再获取（慢）。由于局部性原理，Cache 的命中率通常 > 95%。

**映射方式**（决定主存中的块可以放到 Cache 的哪个位置）：

| 方式 | 原理 | 冲突概率 | 硬件复杂度 | 实际应用 |
|------|------|----------|-----------|----------|
| **直接映射** | 主存块号 % Cache 行数 = Cache 行号。每个主存块只能放在固定的一行 | 高（不同块竞争同一行） | 低（只需比较一个标记） | L1 Cache（对速度要求极高） |
| **全相联映射** | 主存块可以放在 Cache 的**任意一行** | 低（只有 Cache 满了才冲突） | 高（需要与所有行并行比较标记） | TLB（容量很小） |
| **组相联映射** | Cache 分为多个组，主存块号 % 组数 = 组号，块可以放在该组内的**任意一行** | 折中 | 折中 | **最常用**（L2/L3 Cache） |

- **n 路组相联**：每组有 n 行。n=1 退化为直接映射，n=Cache行数 退化为全相联。现代 CPU 通常用 4-16 路组相联

**Cache 地址结构**（以组相联为例）：
- 主存地址 = | 标记 (Tag) | 组号 (Index) | 块内偏移 (Offset) |
- 组号用于定位 Cache 组，标记用于在组内比较是否命中，偏移用于定位块内的具体字节

**替换算法**（Cache 满了需要换出哪一行）：

| 算法 | 策略 | 特点 |
|------|------|------|
| **Random** | 随机选择 | 简单，效果还行 |
| **FIFO** | 替换最先进入的行 | 简单，但可能替换常用行 |
| **LRU** | 替换**最近最久未使用**的行 | 效果最好，实现复杂（需要维护使用时间信息） |
| **LFU** | 替换**使用频率最低**的行 | 需要维护计数器 |

**写策略**（CPU 写数据时如何处理 Cache 和主存的一致性）：

| 策略 | 命中时 | 未命中时 | 特点 |
|------|--------|----------|------|
| **写直达 (Write Through)** | 同时写 Cache 和主存 | 写分配或非写分配 | 实现简单，主存总是最新的，但每次写都访问主存（慢） |
| **写回 (Write Back)** | 只写 Cache，设**脏位 (Dirty Bit)** | 通常写分配 | 减少主存写次数，但需要替换时回写脏块 |

- **写分配 (Write Allocate)**：写未命中时先把块从主存调入 Cache，再写 Cache。利用空间局部性
- **非写分配 (No Write Allocate)**：写未命中时直接写主存，不调入 Cache

现代 CPU 通常使用**写回 + 写分配**的组合。

### 3.4 主存与 CPU 的连接

当单个存储芯片的容量或字长不满足需求时，需要将多个芯片组合使用。

- **位扩展**：增加每个存储单元的位数。将多个芯片的数据引脚并联，地址线和控制线共享。例如用 4 片 1K×4bit 芯片组成 1K×16bit 的存储器
- **字扩展**：增加存储单元的数量（扩大地址空间）。用高位地址通过地址译码器选择芯片（片选信号 CS）。例如用 4 片 1K×8bit 芯片组成 4K×8bit 的存储器
- **字位同时扩展**：两者结合，同时增加字长和地址空间

## 四、指令系统

指令系统（也称指令集架构, ISA）是硬件和软件的接口——它定义了 CPU 能执行哪些操作、操作数在哪里、如何编码指令。ISA 是计算机体系结构中最重要的抽象层。

### 4.1 指令格式

一条指令由**操作码 (Opcode)** 和**地址码 (Operand)** 两部分组成。操作码指定要执行的操作，地址码指定操作数的位置。

| 指令类型 | 地址码个数 | 示例 | 说明 |
|----------|-----------|------|------|
| **零地址指令** | 0 | `NOP`, `HALT`, `PUSH`, `POP` | 无需操作数，或操作数隐含在栈中 |
| **一地址指令** | 1 | `INC R1`, `NOT R1` | 单操作数，结果存回原处 |
| **二地址指令** | 2 | `ADD R1, R2` | 最常见，R1 = R1 + R2 |
| **三地址指令** | 3 | `ADD R1, R2, R3` | R1 = R2 + R3，RISC 常用 |

### 4.2 寻址方式

寻址方式决定了如何从指令中找到操作数的实际地址（有效地址 EA, Effective Address）。

| 寻址方式 | 有效地址 EA | 特点 | 典型用途 |
|----------|------------|------|----------|
| **立即寻址** | 操作数直接在指令中 | 最快，但操作数大小受指令字长限制 | 常量赋值 `MOV R1, #5` |
| **直接寻址** | EA = 指令中给出的地址 A | 简单，一次内存访问 | 访问全局变量 |
| **间接寻址** | EA = (A)，即 A 指向的内存单元的内容是真正的地址 | 可扩大寻址范围，但需要两次内存访问 | 指针操作 |
| **寄存器寻址** | EA = 寄存器 R_i 的内容 | **最快**（不访问内存） | 频繁使用的变量 |
| **寄存器间接寻址** | EA = (R_i)，即寄存器中存放的是地址 | 较快，一次内存访问 | 通过指针访问数据 |
| **基址寻址** | EA = (BR) + A，基址寄存器 + 偏移量 | 支持程序**重定位**（改变 BR 即可） | 操作系统加载程序到不同地址 |
| **变址寻址** | EA = (IX) + A，变址寄存器 + 基地址 | 适合**数组访问**（IX 作为下标递增） | `a[i]` 的访问 |
| **相对寻址** | EA = (PC) + A，程序计数器 + 偏移量 | 指令地址与 PC 相关，代码位置无关 | **转移指令**（跳转到前后 A 条指令处） |
| **堆栈寻址** | EA = 栈顶 | 隐式操作栈顶 | 表达式求值、函数调用 |

### 4.3 CISC vs RISC

这是指令集设计的两大哲学。CISC 试图用复杂的指令简化编程，RISC 则用简单的指令追求硬件效率。

| 对比 | CISC (复杂指令集) | RISC (精简指令集) |
|------|------|------|
| 指令数量 | 多（>200 条） | 少（<100 条） |
| 指令长度 | 可变长（1-15 字节） | **定长**（通常 4 字节） |
| 寻址方式 | 多种（>10 种） | 少（通常 3-4 种） |
| 内存访问 | 任何指令都可以访问内存 | 只有 **Load/Store** 指令可以访问内存 |
| 执行周期 | 不同指令差异大（1-100+ 周期） | 大部分指令**单周期**完成 |
| 流水线 | 难以实现高效流水线 | **易于流水线化** |
| 寄存器 | 少（x86 最初只有 8 个通用寄存器） | 多（通常 32 个） |
| 代表 | x86 (Intel/AMD) | ARM, MIPS, RISC-V |

现代实际中，x86 处理器内部将 CISC 指令翻译为类 RISC 的微操作 (μops) 执行，兼取两者之长。

## 五、CPU

### 5.1 指令周期

CPU 执行一条指令的完整过程称为一个指令周期，包含若干机器周期（也称 CPU 周期）。

指令周期 = 取指周期 + [间址周期] + 执行周期 + [中断周期]

- **取指周期**：从内存中取出指令，放入指令寄存器 IR，PC 自增
- **间址周期**（可选）：如果使用间接寻址，需要额外访存获取有效地址
- **执行周期**：执行指令规定的操作
- **中断周期**（可选）：如果有中断请求，保存现场并转入中断处理

### 5.2 数据通路

数据通路是 CPU 内部数据流动的路径，包括寄存器、ALU、多路选择器以及连接它们的总线或专用线路。

- **CPU 内部数据通路**：寄存器之间的数据传送、寄存器到 ALU 的数据传送
- **CPU 与主存之间**：通过 **MAR (Memory Address Register)** 传送地址，通过 **MDR (Memory Data Register)** 传送数据

### 5.3 控制器

控制器是 CPU 的"指挥中心"，根据指令的操作码产生一系列控制信号，驱动数据通路完成指令的执行。

**硬布线控制器**：
- 用组合逻辑电路（门电路）根据指令操作码、时序信号、状态条件等直接产生控制信号
- 优点：速度快（纯硬件逻辑）
- 缺点：设计复杂，修改困难（改一条指令可能要重新布线），适合 RISC

**微程序控制器**：
- 将每条机器指令的控制信号序列存储在一个专用的只读存储器——**控制存储器 (CM, Control Memory)** 中
- 每条机器指令对应一段**微程序**，微程序由若干**微指令**组成，每条微指令产生一组**微操作**控制信号
- 优点：灵活，易于修改和扩展（改微程序而非改硬件），适合 CISC
- 缺点：速度比硬布线稍慢（需要从 CM 中读取微指令）

### 5.4 流水线 (Pipeline)

**基本思想**：将指令执行分为多个阶段，让不同指令的不同阶段同时执行——就像工厂流水线上同时处理多个产品的不同工序。

**经典五级流水线**：

| 阶段 | 缩写 | 功能 |
|------|------|------|
| 取指 | **IF** (Instruction Fetch) | 从内存取指令 |
| 译码 | **ID** (Instruction Decode) | 译码指令 + 读寄存器 |
| 执行 | **EX** (Execute) | ALU 运算 |
| 访存 | **MEM** (Memory Access) | 读写内存（Load/Store 指令） |
| 写回 | **WB** (Write Back) | 将结果写入目标寄存器 |

**理想加速比**：k 级流水线的理想加速比为 k（5 级流水线理想上比非流水线快 5 倍）。但实际受到**流水线冲突 (Hazard)** 的制约。

**三种流水线冲突**：

| 冲突类型 | 原因 | 解决方法 |
|----------|------|----------|
| **结构冲突** | 两条指令同时需要同一硬件资源。例如取指和访存同时需要访问内存 | 资源复制（如将 Cache 分为指令 Cache 和数据 Cache）；或插入气泡等待 |
| **数据冲突** | 后续指令需要使用前面指令还未产生的结果 | **转发/旁路 (Forwarding/Bypassing)**：EX 阶段产生的结果直接转发给下一条指令的 EX 输入；插入气泡 (Stall)；编译器指令调度 |
| **控制冲突** | 分支指令改变了 PC，但后续指令已经被取入流水线 | 分支预测、延迟分支（在分支后放一条与分支无关的指令）、分支目标缓冲 (BTB) |

**数据冲突的三种类型**：

| 类型 | 含义 | 发生条件 | 频率 |
|------|------|----------|------|
| **RAW (Read After Write)** | 读后写，真数据依赖 | 后面的指令要读前面指令写的结果 | **最常见** |
| **WAR (Write After Read)** | 写后读，反依赖 | 后面的指令要写前面指令读的寄存器 | 乱序执行时可能出现 |
| **WAW (Write After Write)** | 写后写，输出依赖 | 两条指令写同一个寄存器 | 乱序执行时可能出现 |

**分支预测 (Branch Prediction)**：

- **静态预测**：
  - 总是预测不跳转（对顺序代码有效）
  - 总是预测跳转（对循环有效——循环末尾的分支大部分时候都跳回循环体）
  - 根据分支方向预测：向后跳转（循环）预测跳转，向前跳转预测不跳转

- **动态预测**：基于运行时历史信息
  - **1-bit 预测器**：记录上次分支是否跳转，预测这次也一样
  - **2-bit 饱和计数器**：需要连续两次预测错误才改变预测方向（比 1-bit 更稳定，对循环效果好——循环最后一次退出时预测错误，但不会影响下一次循环的预测）
  - **分支历史表 (BHT, Branch History Table)**：为多个分支指令维护各自的预测器

现代 CPU 的分支预测准确率可达 95% 以上。

## 六、总线

总线 (Bus) 是连接计算机各功能部件的公共通信线路。多个部件共享总线进行信息传输。

### 6.1 总线分类

**按功能分**：
- **数据总线 (Data Bus)**：传输数据，**双向**。总线宽度决定一次传输的数据量（如 64 位数据总线一次传 8 字节）
- **地址总线 (Address Bus)**：传输地址，**单向**（CPU → 内存/I/O）。宽度决定可寻址空间（32 位地址总线 → 4GB 地址空间）
- **控制总线 (Control Bus)**：传输控制信号（读/写、中断、总线请求等），部分单向部分双向

**按层次分**：
- **片内总线**：CPU 内部，连接寄存器和 ALU
- **系统总线**：CPU、主存、I/O 接口之间的总线
- **通信总线/I/O 总线**：连接外设（如 PCIe、USB、SATA）

### 6.2 总线仲裁

多个设备可能同时请求使用总线，需要仲裁机制决定谁先使用。

- **集中仲裁**：由总线控制器统一决定
  - **链式查询（菊花链）**：总线授权信号沿链传递，离控制器近的设备优先级高。简单但不公平
  - **计数器定时查询**：控制器用计数器轮询设备。可以设定起始值来调整优先级
  - **独立请求**：每个设备有独立的请求和授权线。最快但线数最多
- **分布仲裁**：各设备自行判断（如通过比较优先级编号）

### 6.3 总线传输

- **同步传输**：所有操作由统一时钟控制。简单但受限于最慢的设备
- **异步传输**：通过请求-应答握手协议。灵活但控制复杂。分为不互锁、半互锁、全互锁三种
- **半同步传输**：同步 + WAIT 信号。慢设备通过拉低 WAIT 信号请求等待

**总线带宽 (Bandwidth)** = 总线宽度 × 总线工作频率。例如 64 位宽、800MHz 的总线带宽 = 64 × 800M / 8 = 6.4 GB/s。

## 七、I/O 系统

### 7.1 I/O 接口

I/O 接口（也称设备控制器/适配器）是 CPU 与外部设备之间的"翻译官"，负责速度匹配、数据格式转换和信号电平转换。

I/O 接口包含：
- **数据缓冲寄存器**：暂存传输的数据
- **状态/控制寄存器**：设备状态信息和 CPU 发出的控制命令
- **地址译码逻辑**：识别 CPU 发出的 I/O 地址

**编址方式**：
- **统一编址（内存映射 I/O, Memory-Mapped I/O）**：I/O 端口和内存共用同一个地址空间。用普通的 Load/Store 指令访问 I/O。简单统一，ARM 采用
- **独立编址（I/O 映射 I/O, Port-Mapped I/O）**：I/O 端口有独立的地址空间，需要专门的 I/O 指令（如 x86 的 IN/OUT）。地址空间不冲突但需要额外的指令

### 7.2 中断系统

**中断 (Interrupt)** 是 CPU 暂停当前程序，转去处理紧急事件的机制。中断使得 CPU 不必轮询设备状态，可以在等待 I/O 的同时执行其他程序。

**中断处理过程**：
1. **中断请求**：外设完成操作后通过中断请求线发出信号
2. **中断响应**：CPU 在**当前指令执行完毕后**（不是执行过程中）检查中断请求
3. **中断处理**：
   - **保护现场**：将 PC、PSW（程序状态字）、关键寄存器压栈
   - 通过**中断向量**（中断服务程序的入口地址）跳转到中断服务程序
   - 执行中断服务程序
   - **恢复现场**：从栈中恢复寄存器
   - **中断返回**：执行 IRET 指令，恢复 PC 和 PSW，继续被中断的程序

**中断向量表 (Interrupt Vector Table)**：存放各中断源对应的中断服务程序入口地址的数组。CPU 根据中断类型号查表获得跳转地址。

**中断优先级**：不同中断源的紧急程度不同。通常：硬件故障 > I/O 中断 > 程序中断 > 外部中断

**中断嵌套**：高优先级的中断可以打断正在处理的低优先级中断。通过**中断屏蔽字**实现——每个中断源对应一个屏蔽位，1 表示屏蔽（禁止该中断），0 表示允许。通过设置不同中断服务程序的屏蔽字，可以灵活控制中断嵌套的行为。
