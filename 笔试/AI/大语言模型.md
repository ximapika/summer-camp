# 大语言模型（LLM）

---

## 一、预训练语言模型演进

### 1.1 Word2Vec（2013）

将词映射为低维稠密向量（Word Embedding），捕获语义关系。

**两种架构**：
- **CBOW（Continuous Bag-of-Words）**：由上下文预测中心词
- **Skip-gram**：由中心词预测上下文词

**训练技巧**：负采样（Negative Sampling），避免计算全词表 Softmax。

$$L = \log \sigma(v_{w_O}^T v_{w_I}) + \sum_{i=1}^k \mathbb{E}_{w_i \sim P_n(w)}[\log \sigma(-v_{w_i}^T v_{w_I})]$$

**经典性质**：vec(king) - vec(man) + vec(woman) ≈ vec(queen)

**局限性**：静态词向量，无法处理一词多义（如"苹果"是水果还是公司）。

### 1.2 ELMo（2018）

**Embeddings from Language Models**：使用双向 LSTM 预训练，为每个词生成上下文相关的动态词向量。

每个词的表示是各层表示的加权和：ELMo_k = γ Σ s_j h_{k,j}

首次实现上下文感知（Contextualized）的词表示。

### 1.3 演进路线

```
Word2Vec (2013) → ELMo (2018) → GPT (2018) → BERT (2018) →
GPT-2 (2019) → GPT-3 (2020) → InstructGPT/ChatGPT (2022) →
GPT-4 (2023) → Claude/LLaMA/Gemini (2023-2024)
```

关键转折点：
- **Transformer 的引入**（2017）：替代 RNN 成为主流架构
- **预训练 + 微调范式**（GPT/BERT, 2018）：大规模预训练 + 下游任务微调
- **大规模 Scaling**（GPT-3, 2020）：175B 参数，In-Context Learning 涌现
- **对齐训练**（InstructGPT/ChatGPT, 2022）：RLHF 使模型遵循人类指令

## 二、BERT（Bidirectional Encoder Representations from Transformers）

**架构**：Transformer **Encoder-only**，双向编码。

**预训练任务**：

1. **MLM（Masked Language Model）**：随机遮盖输入中 15% 的 token，让模型预测被遮盖的 token。
   - 80% 替换为 [MASK]
   - 10% 替换为随机 token
   - 10% 保持不变
   - 这种策略减少预训练和 Fine-tuning 的不一致

2. **NSP（Next Sentence Prediction）**：判断两个句子是否是连续的。
   - 输入：[CLS] 句子A [SEP] 句子B [SEP]
   - 后续研究（RoBERTa）发现 NSP 任务可能并不必要

**输入表示**：Token Embedding + Segment Embedding + Position Embedding

**Fine-tuning**：在预训练模型上加一个任务特定的输出层，用下游任务数据微调全部参数。

- 分类：用 [CLS] token 的表示
- 序列标注：用每个 token 的表示
- 问答：预测答案在原文中的起止位置

**BERT-Base**：12 层、768 维、12 头、110M 参数
**BERT-Large**：24 层、1024 维、16 头、340M 参数

## 三、GPT 系列

**架构**：Transformer **Decoder-only**，自回归生成（Autoregressive）。

**自回归生成**：逐个生成 token，每步基于已生成的所有 token 预测下一个：

$$P(x_1, ..., x_n) = \prod_{t=1}^n P(x_t | x_1, ..., x_{t-1})$$

使用 Causal Mask 确保每个位置只能看到前面的 token。

### GPT-1（2018）：预训练 + 微调

- 12 层 Transformer Decoder，117M 参数
- 无监督预训练（语言建模）+ 有监督微调

### GPT-2（2019）：Zero-shot 能力

- 1.5B 参数
- 提出"语言模型是无监督多任务学习者"
- 通过增大模型和数据规模，实现零样本迁移

### GPT-3（2020）：In-Context Learning 涌现

- **175B 参数**，96 层，d=12288，96 头
- 关键发现：**In-Context Learning（ICL）**——不更新参数，仅通过在 prompt 中提供示例即可完成任务

**Few-shot / Zero-shot / One-shot**：
- **Zero-shot**：仅给出任务描述，无示例
- **One-shot**：给出 1 个示例
- **Few-shot**：给出 2~几十个示例

ICL 的涌现（Emergent Ability）：只有模型规模足够大时才出现，小模型无此能力。

### GPT-4（2023）及之后

- 多模态（文本 + 图像输入）
- 更强的推理能力和更长的上下文窗口
- 更好的对齐和安全性

## 四、大模型关键技术

### 4.1 Tokenization（分词）

将文本切分为 token（模型处理的最小单元）。

**BPE（Byte Pair Encoding）**：GPT 系列使用。
1. 初始化词表为所有字符
2. 统计相邻 token 对的频率
3. 合并最高频 token 对为新 token
4. 重复 2-3 直到达到目标词表大小

**WordPiece**：BERT 使用。类似 BPE 但选择使训练数据似然最大的合并。

**SentencePiece**：LLaMA 等使用。在原始文本（不预分词）上直接训练，支持多语言。

**Byte-level BPE**：以字节为基本单位，GPT-2/3 使用，可处理任何 Unicode 文本。

词表大小通常 32K ~ 128K。中文在 BPE 中通常被拆分为多个 token（UTF-8 字节）。

### 4.2 预训练 → SFT → RLHF → DPO

**预训练（Pre-training）**：
- 在大规模语料上进行无监督语言建模
- 数据：网页、书籍、代码、学术论文等，数万亿 token
- 目标：Next Token Prediction
- 计算量巨大：GPT-3 约 3640 PetaFLOP-days

**SFT（Supervised Fine-Tuning，监督微调）**：
- 使用人类编写的高质量指令-回答对进行微调
- 数据量相对较小（几万到几十万条）
- 使模型学会遵循指令、以对话形式回答

**RLHF（Reinforcement Learning from Human Feedback）**：
1. 训练 **Reward Model（奖励模型）**：人类对模型的多个回答进行排序，训练一个模型来预测人类偏好
2. 使用 **PPO（Proximal Policy Optimization）** 强化学习算法，以 Reward Model 的打分为奖励信号，优化语言模型
3. 关键约束：KL 散度惩罚，防止策略偏离原始模型太远

$$\max_\pi \mathbb{E}_{x \sim D, y \sim \pi}[R(x, y)] - \beta \cdot D_{KL}(\pi \| \pi_{ref})$$

**DPO（Direct Preference Optimization）**：
- 绕过显式的 Reward Model，直接从偏好数据优化策略
- 将 RLHF 的优化问题转化为分类问题

$$L_{DPO}(\theta) = -\mathbb{E}_{(x, y_w, y_l)}\left[\log \sigma\left(\beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right)\right]$$

其中 y_w 为偏好回答，y_l 为非偏好回答。

DPO 训练更稳定、更简单，不需要单独训练 Reward Model 和运行 RL 循环。

### 4.3 Prompt Engineering（提示工程）

通过设计输入提示来引导 LLM 完成任务，无需修改模型参数。

**Chain-of-Thought（CoT，链式思考）**：
- 在 prompt 中引导模型逐步推理
- "Let's think step by step" 可显著提升数学推理等任务的性能
- Zero-shot CoT vs Few-shot CoT

**Few-shot Prompting**：在 prompt 中提供少量示例，引导模型理解任务格式和期望输出。

**System Prompt（系统提示）**：设定模型的角色、行为规范和约束条件。

**其他技巧**：
- Self-Consistency：多次采样 + 多数投票
- Tree-of-Thought：搜索多条推理路径
- ReAct：结合推理（Reasoning）和行动（Acting）

### 4.4 RAG（Retrieval-Augmented Generation，检索增强生成）

**动机**：LLM 的知识固化在参数中，存在知识截断、幻觉等问题。

**流程**：
1. **检索（Retrieve）**：将用户查询通过 Embedding 模型向量化，从知识库中检索相关文档
2. **增强（Augment）**：将检索到的文档作为上下文拼接到 prompt 中
3. **生成（Generate）**：LLM 基于增强后的 prompt 生成回答

**关键组件**：
- **Embedding 模型**：将文本映射为向量（如 text-embedding-ada-002）
- **向量数据库**：存储和检索向量（如 FAISS、Pinecone、Milvus）
- **Chunking 策略**：如何将长文档切分为适当大小的块

**优点**：知识可更新、减少幻觉、可追溯来源、成本低于微调。

### 4.5 AI Agent

LLM 作为核心推理引擎，具备**自主规划和执行**能力。

**核心组件**：
- **规划（Planning）**：将复杂任务分解为子任务（Task Decomposition）
- **工具使用（Tool Use）**：调用外部工具（搜索引擎、代码执行器、API）
- **记忆（Memory）**：短期记忆（上下文窗口）+ 长期记忆（外部存储）
- **反思（Reflection）**：对执行结果进行评估和调整

**典型范式**：ReAct（Reasoning + Acting）、AutoGPT、LangChain Agent。

**Function Calling / Tool Use**：模型生成结构化的工具调用请求，由系统执行并返回结果。

### 4.6 上下文窗口与长文本处理

**上下文窗口（Context Window）**：模型一次能处理的最大 token 数。

- 早期模型：2K-4K tokens
- 当前主流：128K-200K tokens（Claude 200K, GPT-4 Turbo 128K）
- 研究前沿：1M+ tokens

**长文本处理技术**：
- **RoPE 外推**：通过修改旋转位置编码的频率基底，扩展上下文长度
- **Sliding Window Attention**：只关注局部窗口内的 token（如 Mistral）
- **Sparse Attention**：稀疏注意力模式减少计算量
- **Ring Attention**：分布式环形注意力处理超长序列

## 五、大模型推理优化

### 5.1 KV Cache

**问题**：自回归生成时，每生成一个 token 都需要重新计算所有已生成 token 的 K、V。

**解决**：缓存已计算的 Key 和 Value，每步只计算新 token 的 Q、K、V。

将每步计算从 O(n²d) 降为 O(nd)（n 为已生成序列长度）。

**代价**：内存占用随序列长度线性增长。KV Cache 是推理时的主要内存瓶颈。

**优化方法**：
- **MQA（Multi-Query Attention）**：所有 head 共享同一组 K、V
- **GQA（Grouped-Query Attention）**：将 head 分组，组内共享 K、V（LLaMA 2 使用）
- **PagedAttention**：类似操作系统分页管理，减少 KV Cache 内存碎片（vLLM）

### 5.2 量化（Quantization）

将模型权重从高精度（FP32/FP16）转为低精度（INT8/INT4），减少内存和计算开销。

- **INT8 量化**：权重和激活值用 8 位整数表示，模型大小减半，精度损失极小
- **INT4 量化**：权重用 4 位表示，模型大小减至 1/4，有一定精度损失
- **GPTQ/AWQ/GGUF**：各种量化算法和格式

**量化类型**：
- **训练后量化（Post-Training Quantization, PTQ）**：对训练好的模型直接量化
- **量化感知训练（Quantization-Aware Training, QAT）**：在训练中模拟量化误差

### 5.3 推测解码（Speculative Decoding）

使用小模型（Draft Model）快速生成候选 token 序列，大模型（Target Model）并行验证。

**流程**：
1. Draft Model 自回归生成 k 个候选 token
2. Target Model 一次性并行验证这 k 个 token
3. 接受所有通过验证的 token，从第一个被拒绝的位置开始重新生成

**加速原理**：将 k 次大模型的串行推理压缩为 1 次并行验证。

**理论保证**：通过特定的接受/拒绝策略，可以保证最终生成的分布与直接使用大模型生成完全一致。

### 5.4 其他推理优化

- **Flash Attention**：利用 GPU 内存层次结构（SRAM vs HBM），减少内存访问次数，加速注意力计算
- **Continuous Batching**：动态批处理，不同请求完成时立即填入新请求
- **Tensor Parallelism / Pipeline Parallelism**：多 GPU 并行推理

## 六、多模态模型

### 6.1 Vision Transformer（ViT）

将 Transformer 应用于图像：
1. 将图像切分为固定大小的 patch（如 16x16）
2. 每个 patch 线性投影为一个 token
3. 加入 [CLS] token 和位置编码
4. 送入标准 Transformer Encoder
5. 用 [CLS] token 的输出做分类

**重要意义**：证明纯 Transformer 架构（无 CNN）在大规模数据上可以达到甚至超过 CNN 的性能。

### 6.2 CLIP（Contrastive Language-Image Pre-training）

**目标**：学习图像和文本之间的对齐表示。

**架构**：Image Encoder（ViT/ResNet） + Text Encoder（Transformer）

**训练**：对比学习——使匹配的图文对的向量表示接近，不匹配的远离。

$$L = -\frac{1}{N}\sum_i \left[\log \frac{\exp(\text{sim}(I_i, T_i)/\tau)}{\sum_j \exp(\text{sim}(I_i, T_j)/\tau)}\right]$$

**应用**：零样本图像分类、图文检索、为生成模型提供引导信号（如 Stable Diffusion 的文本条件）。

### 6.3 多模态 LLM

将视觉信息融入 LLM：
- **LLaVA**：Visual Encoder（CLIP ViT）+ Projection Layer + LLM
- **GPT-4V/4o**：原生多模态理解与生成
- **Flamingo**：通过 Cross-Attention 将视觉特征注入 LLM

**关键挑战**：视觉与语言模态的对齐、视觉 token 的高效表示、多模态推理。
