# 自然语言处理

---

## 一、基本任务

### 1.1 文本分类（Text Classification）

**定义**：将文本自动归入预定义的类别。

**应用**：情感分析、主题分类、意图识别、垃圾邮件过滤。

**方法演进**：
1. **传统方法**：TF-IDF 特征 + SVM / Naive Bayes
2. **深度学习**：TextCNN（Kim, 2014）/ BiLSTM + Attention
3. **预训练模型**：BERT Fine-tuning（[CLS] token + 分类头）
4. **大模型时代**：Zero/Few-shot Prompting

**TF-IDF（Term Frequency - Inverse Document Frequency）**：

$$\text{TF-IDF}(t, d) = \text{TF}(t, d) \times \text{IDF}(t)$$

- TF(t, d)：词 t 在文档 d 中出现的频率
- IDF(t) = log(N / df(t))：逆文档频率，N 为总文档数，df(t) 为包含词 t 的文档数
- 核心思想：一个词在某文档中频繁出现但在整个语料库中很少出现，则它对该文档很有代表性

**TextCNN**：
- 用多个不同大小的一维卷积核（如 3/4/5）捕获不同 n-gram 特征
- 每个卷积核后接 Max-over-time Pooling
- 拼接后通过全连接层分类

### 1.2 序列标注（Sequence Labeling）

对序列中每个 token 分配一个标签。

**命名实体识别（NER, Named Entity Recognition）**：识别文本中的实体（人名、地名、机构名、时间等）。

**标注体系**：
- **BIO**：B-XXX（实体开头）、I-XXX（实体内部）、O（非实体）
- **BIOES**：增加 E-XXX（实体结尾）、S-XXX（单字实体），更精确

**经典方法**：BiLSTM + CRF
- BiLSTM：捕获双向上下文特征
- CRF（条件随机场）：建模标签间的转移约束（如 B-PER 后不能跟 I-LOC）

**现代方法**：BERT + Token 分类层 / Prompt + 结构化输出

#### 词性标注（POS Tagging）

标注每个词的词性（名词、动词、形容词等）。是许多 NLP 任务的基础预处理步骤。

#### 分词（Word Segmentation）

中文等无自然分隔符的语言需要将连续文本切分为词语。
- 基于词典的方法：正向/逆向最大匹配
- 基于统计的方法：HMM、CRF
- 基于深度学习的方法：BiLSTM + CRF、BERT

### 1.3 机器翻译（Machine Translation）

**方法演进**：
1. **基于规则的翻译**（RBMT）：语言学家手写翻译规则
2. **统计机器翻译**（SMT）：基于统计模型（词对齐、短语翻译表）
3. **神经机器翻译**（NMT）：Seq2Seq + Attention → Transformer
4. **大模型直接翻译**：Zero-shot prompting

**评价指标**：
- **BLEU（Bilingual Evaluation Understudy）**：计算 n-gram 精确率的几何平均 × 简短惩罚因子（Brevity Penalty）
- **METEOR**：考虑同义词和词形变化
- **COMET**：基于预训练模型的评估，与人类判断更一致

### 1.4 文本生成

**任务类型**：
- **摘要生成（Summarization）**：抽取式（选择原文关键句）vs 生成式（生成新句子）
- **对话生成（Dialogue Generation）**：开放域对话、任务型对话
- **问答系统（Question Answering）**：抽取式（从文档中找答案 span）vs 生成式

**解码策略**：
- **Greedy Decoding**：每步选择概率最高的 token。简单但容易生成重复内容
- **Beam Search**：维护 k 个候选序列，选择总概率最高的。k=1 退化为 greedy
- **Sampling**：按概率分布随机采样，增加多样性
- **Top-k Sampling**：只从概率最高的 k 个 token 中采样
- **Top-p (Nucleus) Sampling**：从累积概率超过 p 的最小 token 集合中采样
- **Temperature Scaling**：p_i = exp(z_i/T) / Σexp(z_j/T)。T > 1 更随机，T < 1 更确定

### 1.5 信息抽取（Information Extraction）

**关系抽取（Relation Extraction）**：从文本中识别实体间的关系（如"北京 → 首都 → 中国"）。
- Pipeline 方法：先 NER 再关系分类
- 联合抽取：同时识别实体和关系

**事件抽取（Event Extraction）**：识别文本中的事件及其要素（触发词、论元角色）。

## 二、词嵌入（Word Embedding）

### 2.1 Word2Vec

- **CBOW**：由上下文词预测中心词，速度快
- **Skip-gram**：由中心词预测上下文词，在生僻词上表现更好

### 2.2 GloVe（Global Vectors）

结合全局共现统计信息和局部上下文窗口：

$$J = \sum_{i,j=1}^V f(X_{ij})(w_i^T \tilde{w}_j + b_i + \tilde{b}_j - \log X_{ij})^2$$

X_ij 为词 i 和词 j 的共现次数。f 为加权函数，限制高频词的影响。

### 2.3 FastText

Facebook 提出，利用**子词（Subword）**信息：
- 将每个词表示为字符 n-gram 的集合
- 词向量是所有 n-gram 向量的和
- 能处理未登录词（OOV）：通过组合已知 n-gram

### 2.4 对比总结

| 方法 | 类型 | 特点 |
|---|---|---|
| Word2Vec | 预测模型 | 局部上下文窗口 |
| GloVe | 计数模型 | 利用全局共现矩阵 |
| FastText | 预测模型 | 利用子词信息，处理 OOV |
| ELMo | 上下文动态表示 | 双向 LSTM，首次实现多义词区分 |
| BERT | 上下文动态表示 | Transformer，双向，性能最强 |

**静态 vs 动态词向量**：
- 静态（Word2Vec/GloVe/FastText）：每个词只有一个固定向量，无法区分多义
- 动态（ELMo/BERT）：词的表示依赖上下文，同一个词在不同句子中有不同向量

## 三、预训练模型的 NLP 应用

**预训练模型彻底改变了 NLP 范式**：

传统：特征工程 + 特定模型 → 预训练 + 微调 → Prompt + In-Context Learning

| 任务 | BERT 时代做法 | LLM 时代做法 |
|---|---|---|
| 文本分类 | Fine-tune BERT + 分类头 | Zero/Few-shot Prompting |
| NER | BERT + Token 分类层 | Prompt + 结构化输出 |
| 问答 | BERT + Span 预测 | 直接生成答案 |
| 摘要 | BART/T5 Fine-tune | 直接 Prompting |
| 翻译 | mBART/mT5 Fine-tune | 直接 Prompting |

### 3.1 三种预训练架构

| 架构 | 代表模型 | 预训练任务 | 适用场景 |
|---|---|---|---|
| **Encoder-only** | BERT, RoBERTa | MLM, NSP | 理解任务（分类、NER、QA） |
| **Decoder-only** | GPT, LLaMA | Next Token Prediction | 生成任务（文本生成、对话） |
| **Encoder-Decoder** | T5, BART | Span Corruption / Denoising | 序列到序列任务（翻译、摘要） |

### 3.2 微调技术

**全参数微调（Full Fine-tuning）**：更新模型所有参数。效果最好，但存储成本高（每个任务需要一份完整模型）。

**参数高效微调（PEFT, Parameter-Efficient Fine-Tuning）**：

| 方法 | 原理 | 可训练参数量 |
|---|---|---|
| **LoRA** | 在权重矩阵上加低秩分解矩阵 ΔW = AB（A: d×r, B: r×d, r << d） | 0.1%~1% |
| **Adapter** | 在每层中插入小型 MLP 模块 | 2%~5% |
| **Prefix Tuning** | 在每层 attention 前拼接可训练的前缀向量 | <1% |
| **Prompt Tuning** | 只优化输入前面的连续向量 | <0.1% |

**LoRA 详解**：
- 原始：h = Wx
- LoRA：h = Wx + ΔWx = Wx + ABx
- 推理时可将 ΔW 合并到 W 中，无额外推理开销
- 通常应用于 attention 层的 W_q, W_v 矩阵

## 四、语言模型基础

### 4.1 统计语言模型

给定词序列 w_1, ..., w_n，语言模型计算其概率：

$$P(w_1, ..., w_n) = \prod_{i=1}^n P(w_i | w_1, ..., w_{i-1})$$

**N-gram 模型**：用前 N-1 个词近似条件概率。

$$P(w_i | w_1, ..., w_{i-1}) \approx P(w_i | w_{i-N+1}, ..., w_{i-1})$$

- Unigram (N=1)：P(w_i)
- Bigram (N=2)：P(w_i | w_{i-1})
- Trigram (N=3)：P(w_i | w_{i-2}, w_{i-1})

**问题**：数据稀疏（很多 N-gram 组合在训练语料中未出现）。
**解决**：平滑技术（Add-k Smoothing, Kneser-Ney Smoothing）。

**困惑度（Perplexity, PPL）**：评估语言模型质量的指标。

$$\text{PPL} = P(w_1, ..., w_N)^{-1/N} = \exp\left(-\frac{1}{N}\sum_{i=1}^N \log P(w_i | w_{<i})\right)$$

困惑度越低，模型越好。直观含义：模型在每个位置平均需要从多少个候选中选择。

### 4.2 Attention 机制的变体

| 类型 | 描述 | 应用 |
|---|---|---|
| **Self-Attention** | 序列内部各位置之间的注意力 | Transformer Encoder/Decoder |
| **Cross-Attention** | Decoder 对 Encoder 输出的注意力 | Encoder-Decoder 架构 |
| **Causal/Masked Attention** | 只允许关注当前及之前位置 | 自回归生成（GPT） |
| **Multi-Head Attention** | 多组并行的注意力 | Transformer 标准组件 |
| **Sparse Attention** | 只计算部分位置对的注意力 | 长序列处理（Longformer） |
| **Flash Attention** | IO 感知的精确注意力实现 | 训练和推理加速 |
