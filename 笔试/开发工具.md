# 开发工具

## 一、Git 版本控制

### 1.1 基本概念

Git 是一个分布式版本控制系统，用于跟踪文件的修改历史、支持多人协作开发。与集中式版本控制（如 SVN）不同，Git 的每个开发者都拥有完整的仓库副本。

**核心概念**：

| 概念 | 说明 |
|------|------|
| **Repository (仓库)** | 存储项目所有文件和历史记录的地方 |
| **Commit (提交)** | 一个快照，记录某个时间点所有文件的状态 |
| **Branch (分支)** | 指向某个 commit 的可移动指针，用于并行开发 |
| **HEAD** | 指向当前所在分支的最新 commit |
| **Working Directory (工作区)** | 本地文件系统中的文件 |
| **Staging Area (暂存区)** | 准备提交的文件变更集合（也叫 Index） |
| **Remote (远程仓库)** | 托管在服务器上的仓库（如 GitHub, GitLab） |

### 1.2 工作流程

```
工作区 ──git add──→ 暂存区 ──git commit──→ 本地仓库 ──git push──→ 远程仓库
                                                    ←─git pull/fetch─
```

### 1.3 常用命令

**初始化与克隆**：
```bash
git init                    # 初始化本地仓库
git clone <url>             # 克隆远程仓库
```

**日常操作**：
```bash
git status                  # 查看工作区状态
git add <file>              # 添加到暂存区
git add .                   # 添加所有修改
git commit -m "message"     # 提交到本地仓库
git log --oneline --graph   # 查看提交历史（简洁图形化）
git diff                    # 查看工作区与暂存区的差异
git diff --cached           # 查看暂存区与上次 commit 的差异
```

**分支管理**：
```bash
git branch                  # 查看本地分支
git branch <name>           # 创建分支
git checkout <name>         # 切换分支
git checkout -b <name>      # 创建并切换
git switch <name>           # 切换分支（Git 2.23+ 推荐）
git switch -c <name>        # 创建并切换
git merge <branch>          # 将指定分支合并到当前分支
git rebase <branch>         # 变基（将当前分支的 commit 接到指定分支后面）
git branch -d <name>        # 删除已合并的分支
```

**远程操作**：
```bash
git remote -v               # 查看远程仓库
git remote add origin <url> # 添加远程仓库
git push origin <branch>    # 推送到远程
git pull origin <branch>    # 拉取并合并（= fetch + merge）
git fetch                   # 只下载远程更新，不合并
```

**撤销与回退**：
```bash
git checkout -- <file>      # 撤销工作区修改（恢复到暂存区版本）
git restore <file>          # 同上（Git 2.23+ 推荐）
git reset HEAD <file>       # 取消暂存
git restore --staged <file> # 同上
git reset --soft HEAD~1     # 回退一个 commit，保留修改在暂存区
git reset --hard HEAD~1     # 回退一个 commit，丢弃所有修改（危险！）
git revert <commit>         # 创建一个新 commit 来撤销指定 commit（安全）
```

**暂存工作**：
```bash
git stash                   # 暂存当前未提交的修改
git stash pop               # 恢复最近一次暂存
git stash list              # 查看所有暂存
```

### 1.4 merge vs rebase

| 对比 | merge | rebase |
|------|-------|--------|
| 方式 | 创建一个合并 commit | 将 commit 移到目标分支后面 |
| 历史 | 保留分支合并的历史（有分叉） | 线性历史（无分叉） |
| 冲突处理 | 一次处理所有冲突 | 逐个 commit 处理 |
| 适用 | 公共分支合并 | 个人分支同步主分支 |

**黄金法则**：不要在公共分支上 rebase（会改写历史）。

### 1.5 解决冲突

当两个分支修改了同一文件的同一区域时产生冲突：

```
<<<<<<< HEAD
当前分支的内容
=======
合并进来的分支的内容
>>>>>>> feature-branch
```

手动编辑文件选择保留内容 → `git add` → `git commit`。

### 1.6 .gitignore

指定 Git 忽略的文件（不纳入版本控制）：

```
*.o              # 忽略所有 .o 文件
build/           # 忽略 build 目录
!important.o     # 但不忽略 important.o
*.log            # 忽略日志文件
__pycache__/     # Python 缓存
node_modules/    # Node.js 依赖
.env             # 环境变量文件（含敏感信息）
```

## 二、编译器与构建工具

### 2.1 GCC / G++

GCC (GNU Compiler Collection) 是最常用的 C/C++ 编译器。

```bash
# 基本编译
g++ -o program main.cpp           # 编译并输出可执行文件
g++ -std=c++17 -o program main.cpp  # 指定 C++ 标准

# 常用选项
-O2          # 优化级别 2（竞赛推荐）
-O0          # 无优化（调试用）
-g           # 生成调试信息（配合 gdb）
-Wall        # 显示所有警告
-Wextra      # 显示额外警告
-DLOCAL      # 定义宏 LOCAL（用于本地调试）
-fsanitize=address,undefined  # 地址/未定义行为检查（调试利器）

# 竞赛常用编译命令
g++ -std=c++17 -O2 -Wall -o sol sol.cpp
```

**编译过程**：
```
源代码(.cpp) → 预处理(.i) → 编译(.s汇编) → 汇编(.o目标文件) → 链接(可执行文件)
```

```bash
g++ -E main.cpp -o main.i    # 只做预处理
g++ -S main.cpp -o main.s    # 只编译到汇编
g++ -c main.cpp -o main.o    # 编译到目标文件
g++ main.o -o main           # 链接
```

### 2.2 Clang

LLVM 项目的 C/C++ 编译器，与 GCC 命令行兼容，错误信息更友好：

```bash
clang++ -std=c++17 -O2 -o program main.cpp
```

### 2.3 Make 与 Makefile

Make 是自动化构建工具。Makefile 定义了编译规则和依赖关系。

```makefile
# 基本语法：
# 目标: 依赖
#     命令（前面必须是 Tab）

CC = g++
CFLAGS = -std=c++17 -O2 -Wall

program: main.o utils.o
	$(CC) $(CFLAGS) -o program main.o utils.o

main.o: main.cpp utils.h
	$(CC) $(CFLAGS) -c main.cpp

utils.o: utils.cpp utils.h
	$(CC) $(CFLAGS) -c utils.cpp

clean:
	rm -f *.o program

.PHONY: clean
```

```bash
make           # 编译
make clean     # 清理
```

### 2.4 CMake

跨平台的构建系统生成器，比 Makefile 更现代化。通过 CMakeLists.txt 描述项目：

```cmake
cmake_minimum_required(VERSION 3.10)
project(MyProject)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

add_executable(program main.cpp utils.cpp)
```

```bash
mkdir build && cd build
cmake ..       # 生成构建文件
make           # 编译
```

## 三、调试工具

### 3.1 GDB

GDB (GNU Debugger) 是 Linux 下最常用的调试器。

```bash
# 编译时加 -g 选项
g++ -g -o program main.cpp

# 启动 GDB
gdb ./program
```

**常用命令**：

| 命令 | 缩写 | 功能 |
|------|------|------|
| `run` | `r` | 运行程序 |
| `break main` | `b main` | 在 main 函数设断点 |
| `break 42` | `b 42` | 在第 42 行设断点 |
| `next` | `n` | 执行下一行（不进入函数） |
| `step` | `s` | 执行下一行（进入函数） |
| `continue` | `c` | 继续执行到下一个断点 |
| `print x` | `p x` | 打印变量 x 的值 |
| `backtrace` | `bt` | 查看调用栈 |
| `watch x` | | 监视变量 x，值改变时暂停 |
| `info locals` | | 查看所有局部变量 |
| `quit` | `q` | 退出 |

### 3.2 Valgrind

内存错误检测工具，可以发现内存泄漏、越界访问、使用未初始化变量等问题：

```bash
valgrind --leak-check=full ./program
```

### 3.3 AddressSanitizer (ASan)

编译器内建的内存错误检测，比 Valgrind 更快：

```bash
g++ -fsanitize=address -g -o program main.cpp
./program  # 运行时自动检测内存错误
```

## 四、其他常用工具

### 4.1 文本编辑器与 IDE

| 工具 | 特点 |
|------|------|
| **Vim / Neovim** | 终端编辑器，高效但学习曲线陡峭 |
| **VS Code** | 轻量级 IDE，插件丰富，最流行 |
| **CLion** | JetBrains 的 C++ IDE，功能强大 |
| **IntelliJ IDEA** | Java 开发首选 |
| **PyCharm** | Python 开发首选 |

### 4.2 Shell 工具

```bash
time ./program           # 测量程序运行时间
diff output.txt expected.txt  # 比较两个文件的差异
wc -l file.cpp           # 统计代码行数
```

### 4.3 Docker

Docker 是容器化工具，将应用及其依赖打包成轻量级容器，确保"在我这里能运行"。

```bash
docker run ubuntu:22.04   # 运行一个 Ubuntu 容器
docker build -t app .     # 根据 Dockerfile 构建镜像
docker ps                 # 查看运行中的容器
```

核心概念：
- **镜像 (Image)**：只读模板，包含应用和依赖
- **容器 (Container)**：镜像的运行实例
- **Dockerfile**：定义如何构建镜像

### 4.4 正则表达式工具

```bash
grep -E "pattern" file    # 扩展正则搜索
sed -E 's/old/new/g' file # 正则替换
```

### 4.5 网络工具

```bash
curl -X GET http://api.example.com    # HTTP 请求
wget http://example.com/file.zip      # 下载文件
ssh user@host                         # 远程登录
scp file user@host:/path              # 远程复制
```

### 4.6 包管理与虚拟环境

**Python**：
```bash
pip install package       # 安装包
pip freeze > requirements.txt  # 导出依赖
python -m venv myenv      # 创建虚拟环境
source myenv/bin/activate  # 激活虚拟环境
```

**Node.js**：
```bash
npm install package       # 安装包
npm init                  # 初始化项目
```
