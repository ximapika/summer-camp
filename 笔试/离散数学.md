# 离散数学

离散数学是计算机科学的数学基础。与微积分研究的连续对象不同，离散数学研究的是**离散的**（不连续的、可数的）数学结构，如整数、图、逻辑命题、集合等。计算机中的数据本质上是离散的（0 和 1），因此离散数学为算法设计、数据结构、编程语言理论、密码学等计算机核心领域提供了理论工具。

## 一、数理逻辑

数理逻辑用形式化的方法研究推理的规律。在计算机科学中，逻辑是程序正确性验证、数据库查询、人工智能推理等领域的基础。

### 1.1 命题逻辑

**命题 (Proposition)** 是一个可以判断真假的陈述句。例如"2 + 3 = 5"是命题（真），"今天下雨"是命题（真或假），但"x > 3"不是命题（x 未确定）、"请关门"也不是命题（不是陈述句）。

**联结词 (Connective)** 用于将简单命题组合成复合命题：

| 联结词 | 符号 | 名称 | 含义 | 真值条件 |
|--------|------|------|------|----------|
| 否定 | ¬P | NOT | 非 P | P 为真则假，P 为假则真 |
| 合取 | P ∧ Q | AND | P 且 Q | 都真才真 |
| 析取 | P ∨ Q | OR | P 或 Q | 都假才假 |
| 蕴涵 | P → Q | IF...THEN | 若 P 则 Q | **仅当 P 真 Q 假时为假**，其余为真 |
| 等价 | P ↔ Q | IFF | P 当且仅当 Q | P 和 Q 同真同假时为真 |

**蕴涵 P → Q 的理解**：这是最容易混淆的联结词。关键点在于：当前提 P 为假时，无论 Q 如何，P → Q 都为真（"从假前提可以推出任何结论"）。例如"如果 1+1=3，那么月亮是方的"是一个真命题。在编程中，if 条件为 false 时不执行，也不算"违反"了条件。

**重要等值式（逻辑恒等式）**：
- **双重否定**：¬¬P ≡ P
- **德·摩根律 (De Morgan's Laws)**：¬(P ∧ Q) ≡ ¬P ∨ ¬Q，¬(P ∨ Q) ≡ ¬P ∧ ¬Q。在编程中非常常用：`!(a && b)` 等价于 `!a || !b`
- **蕴涵等值**：P → Q ≡ ¬P ∨ Q。蕴涵可以用否定和析取表示
- **逆否命题**：P → Q ≡ ¬Q → ¬P。原命题与逆否命题等价。"如果下雨则地湿"等价于"如果地没湿则没下雨"
- **分配律**：P ∧ (Q ∨ R) ≡ (P ∧ Q) ∨ (P ∧ R)；P ∨ (Q ∧ R) ≡ (P ∨ Q) ∧ (P ∨ R)
- **吸收律**：P ∨ (P ∧ Q) ≡ P；P ∧ (P ∨ Q) ≡ P

**范式 (Normal Form)**：将命题公式标准化的形式，便于比较和推理。

- **析取范式 (DNF, Disjunctive Normal Form)**：合取式的析取。例如 (P ∧ Q) ∨ (¬P ∧ R)。每个合取式称为**极小项**
- **合取范式 (CNF, Conjunctive Normal Form)**：析取式的合取。例如 (P ∨ Q) ∧ (¬P ∨ R)。每个析取式称为**极大项**
- **主析取范式**：每个极小项中，所有命题变量都恰好出现一次（正或负）。主析取范式是唯一的，可以从真值表直接写出——取所有使公式为真的行对应的极小项
- **主合取范式**：类似地，取所有使公式为假的行对应的极大项

### 1.2 谓词逻辑

命题逻辑将命题视为不可分割的整体，无法表达命题内部的结构。例如"所有学生都通过了考试"在命题逻辑中只能用一个符号表示，无法进一步推理。**谓词逻辑**（一阶逻辑）引入**个体**和**谓词**的概念，可以描述个体的性质和个体之间的关系。

**谓词 (Predicate)**：描述个体性质或关系的函数。P(x) 表示"x 具有性质 P"，如 Student(x) 表示"x 是学生"；R(x, y) 表示"x 与 y 有关系 R"，如 GreaterThan(x, y) 表示"x > y"。

**量词 (Quantifier)**：
- **全称量词 ∀ (for all)**：∀x P(x) 表示"对论域中所有的 x，P(x) 都成立"
- **存在量词 ∃ (there exists)**：∃x P(x) 表示"论域中存在某个 x，使得 P(x) 成立"

**量词否定规则**（极其重要，类似于德·摩根律）：
- ¬∀x P(x) ≡ ∃x ¬P(x)（"并非所有人都及格" = "存在不及格的人"）
- ¬∃x P(x) ≡ ∀x ¬P(x)（"不存在及格的人" = "所有人都不及格"）

**推理规则**：
- **全称实例化 (UI, Universal Instantiation)**：∀x P(x) → P(a)，对任意具体个体 a 成立
- **存在实例化 (EI, Existential Instantiation)**：∃x P(x) → P(c)，其中 c 是一个新的常量（Skolem 常量），不能是已出现过的
- **全称推广 (UG, Universal Generalization)**：如果 P(a) 对论域中任意 a 都成立，则 ∀x P(x)
- **存在推广 (EG, Existential Generalization)**：P(a) → ∃x P(x)，已知某个具体 a 满足 P，则存在满足 P 的 x

## 二、集合论

集合论是现代数学的基础，也是数据库理论（关系代数）、类型理论等计算机领域的基础。

### 2.1 集合运算

集合是由确定的、互不相同的对象组成的整体。

| 运算 | 符号 | 定义 | 直觉理解 |
|------|------|------|----------|
| 并集 | A ∪ B | {x \| x ∈ A 或 x ∈ B} | A 和 B 中所有元素的合集 |
| 交集 | A ∩ B | {x \| x ∈ A 且 x ∈ B} | A 和 B 共有的元素 |
| 差集 | A - B | {x \| x ∈ A 且 x ∉ B} | 在 A 中但不在 B 中的元素 |
| 补集 | A̅ 或 Aᶜ | U - A（U 为全集） | 不在 A 中的所有元素 |
| 对称差 | A ⊕ B | (A - B) ∪ (B - A) = (A ∪ B) - (A ∩ B) | 在 A 或 B 中但不同时在两者中 |
| 幂集 | P(A) | A 的所有子集构成的集合 | \|P(A)\| = 2^\|A\|（n 个元素有 2^n 个子集） |
| 笛卡尔积 | A × B | {(a,b) \| a ∈ A, b ∈ B} | 所有有序对的集合，\|A×B\| = \|A\|×\|B\| |

集合运算的性质与命题逻辑有对应关系（∪ 对应 ∨，∩ 对应 ∧，补集对应 ¬），德·摩根律同样成立：
- (A ∪ B)ᶜ = Aᶜ ∩ Bᶜ
- (A ∩ B)ᶜ = Aᶜ ∪ Bᶜ

### 2.2 关系

**关系 R ⊆ A × B** 是 A 到 B 的笛卡尔积的子集。直观理解：关系描述了 A 中的元素和 B 中的元素之间的某种联系。当 A = B 时，称为 A 上的二元关系。例如，"小于"关系定义了自然数之间哪些对 (a, b) 满足 a < b。

**关系的性质**（R 是集合 A 上的关系）：

| 性质 | 定义 | 直觉理解 | 矩阵特征 | 图特征 |
|------|------|----------|----------|--------|
| 自反 | ∀a ∈ A, (a,a) ∈ R | 每个元素都与自己有关系 | 主对角线全 1 | 每个顶点有自环 |
| 反自反 | ∀a ∈ A, (a,a) ∉ R | 没有元素与自己有关系 | 主对角线全 0 | 无自环 |
| 对称 | (a,b) ∈ R → (b,a) ∈ R | a 与 b 有关系则 b 与 a 也有 | M = Mᵀ（转置相等） | 有向边都成对出现 |
| 反对称 | (a,b) ∈ R ∧ (b,a) ∈ R → a = b | a≠b 时不可能双向都有关系 | M 与 Mᵀ 非对角线位置不同时为 1 | 不同顶点间无双向边 |
| 传递 | (a,b) ∈ R ∧ (b,c) ∈ R → (a,c) ∈ R | 关系可以"传递"——a 到 b、b 到 c，则 a 到 c | M² 中 1 的位置 M 也为 1 | 有两步路径就有直达边 |

**闭包 (Closure)**：给定关系 R，可能不满足某个性质。闭包是**包含 R 的、满足该性质的最小关系**。

- **自反闭包** r(R) = R ∪ I_A（I_A 是恒等关系，即加上所有自环）
- **对称闭包** s(R) = R ∪ R⁻¹（R⁻¹ 是反关系，即加上所有反向边）
- **传递闭包** t(R) = R ∪ R² ∪ R³ ∪ ...（R² = R∘R，即两步可达也算直接可达。可用 **Warshall 算法** O(n³) 计算）

**等价关系**：同时满足自反、对称、传递三个性质的关系。等价关系将集合 A 划分为若干互不相交的**等价类 (Equivalence Class)**，所有等价类的并集等于 A。例如"模 3 同余"是整数上的等价关系，将整数划分为三个等价类：{...,-3,0,3,6,...}、{...,-2,1,4,7,...}、{...,-1,2,5,8,...}。

**偏序关系 (Partial Order)**：同时满足自反、反对称、传递三个性质的关系。偏序集记为 (A, ≤)。偏序中并非所有元素都可以比较（例如集合的包含关系——{1,2} 和 {2,3} 不可比较）。

- **全序 (Total Order)**：偏序 + 任意两个元素都可比较。例如实数上的 ≤
- **哈斯图 (Hasse Diagram)**：偏序关系的简化图示——省略自环、传递边和方向（约定方向向上）
- 相关概念：
  - **极大元/极小元**：集合中没有比它更大/更小的元素（可以有多个）
  - **最大元/最小元**：大于/小于集合中所有元素（如果存在则唯一）
  - **上界/下界**：大于等于/小于等于子集中所有元素的元素
  - **上确界 (LUB, Least Upper Bound) / 下确界 (GLB, Greatest Lower Bound)**：最小的上界 / 最大的下界

### 2.3 函数

函数 f: A → B 是一种特殊的关系，满足 A 中**每个**元素恰好对应 B 中**一个**元素。A 是定义域 (Domain)，B 是值域 (Codomain)，f(A) 是像 (Range)。

| 类型 | 定义 | 直觉理解 | 别名 |
|------|------|----------|------|
| **单射 (Injection)** | f(a₁) = f(a₂) → a₁ = a₂ | 不同的输入映射到不同的输出（"一对一"） | 一一映射 |
| **满射 (Surjection)** | ∀b ∈ B, ∃a ∈ A, f(a) = b | B 中每个元素都有原像（"映满"） | 到上映射 |
| **双射 (Bijection)** | 单射 + 满射 | 一一对应，A 和 B 元素完美匹配 | 一一对应 |

**重要性质**：
- 双射函数存在逆函数 f⁻¹
- |A| = |B| 当且仅当存在 A 到 B 的双射（这是比较集合大小的数学定义）
- 可数集：与自然数集合之间存在双射的集合。有理数是可数的，实数是不可数的（Cantor 对角线论证）

## 三、图论

图论研究由顶点和边组成的图结构，在计算机科学中有广泛应用——网络拓扑、社交网络、路由算法、编译器优化（控制流图）等都依赖图论。

### 3.1 基本概念

- **图 G = (V, E)**：V 是顶点集合，E 是边集合
- **无向图**：边是无序的顶点对 {u, v}
- **有向图**：边是有序的顶点对 (u, v)，从 u 到 v
- **简单图**：无自环（顶点到自身的边）和重边（两顶点间多条边）
- **完全图 K_n**：n 个顶点，任意两顶点间都有边。边数 = C(n,2) = n(n-1)/2
- **二部图 (Bipartite Graph)**：顶点可以分为两组，边只在两组之间。等价条件：**不含奇数长度的环**
- **度 (Degree)**：与顶点关联的边数。有向图分入度 (in-degree) 和出度 (out-degree)
- **路径 (Path)**：从 u 到 v 的顶点序列，相邻顶点间有边
- **连通图**：任意两顶点间都有路径

### 3.2 握手定理

**无向图**：所有顶点的度之和 = 2|E|（每条边贡献两个度）

**有向图**：所有顶点的入度之和 = 出度之和 = |E|（每条有向边贡献一个出度和一个入度）

**推论**：无向图中度为奇数的顶点个数一定是**偶数**。

### 3.3 欧拉图与哈密顿图

**欧拉图**涉及的是**边**——能否经过每条边恰好一次？这个问题起源于 Euler 在 1736 年解决的柯尼斯堡七桥问题。

| | 欧拉回路（回到起点） | 欧拉路径（不必回到起点） |
|--|---------|---------|
| 定义 | 经过每条**边**恰好一次，回到起点 | 经过每条**边**恰好一次 |
| 无向图 | 连通 + 所有顶点的度都是偶数 | 连通 + 恰好 0 或 2 个奇数度顶点 |
| 有向图 | 强连通 + 每个顶点的入度 = 出度 | 弱连通 + 至多一个顶点出度-入度=1（起点），至多一个入度-出度=1（终点），其余入度=出度 |

**哈密顿图**涉及的是**顶点**——能否经过每个顶点恰好一次？

- **哈密顿回路**：经过每个顶点恰好一次并回到起点
- **哈密顿路径**：经过每个顶点恰好一次（不必回到起点）
- 没有简单的充要条件（判定哈密顿回路是否存在是 NP 完全问题）
- **Dirac 定理**（充分条件）：n ≥ 3 的简单图，若每个顶点的度 ≥ n/2，则存在哈密顿回路
- **Ore 定理**（充分条件）：n ≥ 3 的简单图，若任意不相邻的两顶点度之和 ≥ n，则存在哈密顿回路

### 3.4 平面图

**平面图**是可以画在平面上使得**边不相交**的图。判断一个图是否为平面图在电路设计（PCB 布线）等领域有重要应用。

**欧拉公式**：对于连通平面图，V - E + F = 2（V 顶点数，E 边数，F 面数，包括无界面）

**推论**（判断非平面图的必要条件）：
- 简单连通平面图：E ≤ 3V - 6（至少有 6 个顶点时适用）
- 无三角形（无长度为 3 的环）的简单连通平面图：E ≤ 2V - 4

由此可以证明 K₅（5 个顶点的完全图）和 K₃,₃（完全二部图）不是平面图（验证 E > 3V-6 即可）。

**库拉托夫斯基定理**：图 G 是平面图 **当且仅当** G 不包含 K₅ 或 K₃,₃ 的**细分**（subdivision，即在边上插入顶点得到的图）作为子图。

### 3.5 图着色

**顶点着色**：给图的顶点着色，要求相邻顶点颜色不同。所需的最少颜色数称为**色数 χ(G)**。

- **四色定理 (Four Color Theorem)**：任何平面图的色数 ≤ 4。这个定理在 1976 年首次由计算机辅助证明
- **二部图**当且仅当 χ(G) ≤ 2，当且仅当图中不含奇数长度的环
- **贪心着色**：按某种顺序给顶点着色，每次用最小的可用颜色。结果不一定最优，但给出上界 χ(G) ≤ Δ(G) + 1（Δ 是最大度）

图着色在编译器中有重要应用：**寄存器分配**可以建模为图着色问题——变量是顶点，同时活跃的变量之间连边，用 k 种颜色（k 个寄存器）着色。

### 3.6 树的性质

树是连通无环图，是图论中最简单也最重要的结构。以下性质互为充要条件（任选一个作为定义，其余可推导）：

- 连通无环图
- n 个顶点，恰好 n-1 条边
- 任意两点之间**恰好一条**路径
- 连通，但删去任一条边就变为不连通
- 无环，但添加任一条边就产生恰好一个环

**生成树 (Spanning Tree)**：图 G 的一个子图，包含 G 的所有顶点并构成一棵树。连通图一定有生成树。

## 四、代数结构

代数结构研究集合及其上定义的运算。在计算机科学中，群论用于密码学，格论用于程序分析，代数结构的思想贯穿于编程语言设计（类型系统、代数数据类型）。

### 4.1 群 (Group)

群是最基本的代数结构之一。集合 G 上的二元运算 · 构成群需要满足四条公理：

1. **封闭性**：∀a, b ∈ G，a · b ∈ G（运算结果仍在集合内）
2. **结合律**：(a · b) · c = a · (b · c)
3. **单位元（幺元）**：∃e ∈ G，∀a ∈ G，a · e = e · a = a（存在"不变元素"）
4. **逆元**：∀a ∈ G，∃a⁻¹ ∈ G，a · a⁻¹ = a⁻¹ · a = e（每个元素都有"反元素"）

若还满足**交换律**（a · b = b · a），则称为**交换群（阿贝尔群, Abelian Group）**。

**常见群的例子**：
- (ℤ, +)：整数在加法下构成阿贝尔群（单位元 0，n 的逆元 -n）
- (ℤₙ, +)：模 n 整数加法群
- (ℤₙ*, ×)：模 n 的乘法群（与 n 互素的元素），在密码学（如 RSA）中极为重要

**相关概念**：
- **群的阶 |G|**：群中元素的个数
- **子群**：G 的非空子集 H，在同一运算下也构成群
- **拉格朗日定理**：有限群 G 中，子群 H 的阶整除 G 的阶。即 |H| 能整除 |G|

### 4.2 环 (Ring)

环在一个集合上定义了**两种**运算（加法和乘法）：
- (R, +) 是**阿贝尔群**
- (R, ·) 满足封闭性和结合律（但不要求逆元和交换律）
- 乘法对加法满足**分配律**：a·(b+c) = a·b + a·c

例如 (ℤ, +, ×) 构成环。若乘法也满足交换律，则称为**交换环**。若存在乘法单位元（1），则称为**含幺环**。

### 4.3 域 (Field)

域是一种"运算最完备"的代数结构：加法和乘法都构成阿贝尔群（乘法去掉零元素后）。

- (F, +) 是阿贝尔群
- (F - {0}, ·) 是阿贝尔群
- 分配律成立

简单说：域中可以自由地做加减乘除（除以零除外）。

**常见域**：有理数 ℚ、实数 ℝ、复数 ℂ、模素数 p 的整数 ℤₚ。注意 ℤ 不是域（整数除法不封闭）。ℤₚ 是有限域，在密码学和编码理论中极为重要。

### 4.4 格 (Lattice)

格是偏序集 (L, ≤)，其中任意两个元素都有**最小上界（上确界/join，记为 a ∨ b）** 和**最大下界（下确界/meet，记为 a ∧ b）**。

格在程序分析中有重要应用——数据流分析框架本质上就是在格上进行的不动点计算。

### 4.5 同态与同构

**同态 (Homomorphism)**：保持运算结构的映射。如果 f: G → G'，使得 f(a · b) = f(a) · f(b)，则 f 是同态。

**同构 (Isomorphism)**：双射的同态。如果两个代数系统之间存在同构映射，则它们"结构完全相同"——只是元素的名字不同。例如 ({0,1}, XOR) 和 (ℤ₂, +) 是同构的。

**同态基本定理**：群同态 f: G → G' 的核 ker(f) 是 G 的正规子群，且 G / ker(f) ≅ im(f)。

## 五、组合数学

组合数学研究计数问题——"有多少种方式？"。在算法分析中，组合数学用于分析算法的复杂度、计算概率、设计编码方案等。

### 5.1 排列与组合

- **排列 (Permutation)**：从 n 个不同元素中取出 r 个排成一列的方式数
  - P(n,r) = n! / (n-r)!
  - 全排列：P(n,n) = n!

- **组合 (Combination)**：从 n 个不同元素中取出 r 个组成一组的方式数（不考虑顺序）
  - C(n,r) = n! / (r! × (n-r)!)
  - 也记为 $\binom{n}{r}$

- **多重集排列**：n 个元素中有 k 种不同类型，各有 n₁, n₂, ..., nₖ 个，全排列数为：
  - n! / (n₁! × n₂! × ... × nₖ!)
  - 例如 "aab" 的排列数 = 3! / (2! × 1!) = 3

- **多重集组合**（隔板法/星与条模型）：从 k 种元素中取 r 个（可重复），方式数为：
  - C(k+r-1, r)

### 5.2 常用公式

- **二项式定理**：(a+b)ⁿ = Σ_{k=0}^{n} C(n,k) × aⁿ⁻ᵏ × bᵏ
- **组合数恒等式**：
  - C(n,0) + C(n,1) + ... + C(n,n) = 2ⁿ（取 a=b=1 即得）
  - C(n,r) = C(n-1,r-1) + C(n-1,r)（**帕斯卡恒等式**，杨辉三角的递推关系）
  - C(n,r) = C(n, n-r)（对称性）
  - Σ_{k=0}^{r} C(m,k)C(n,r-k) = C(m+n,r)（**Vandermonde 恒等式**）

### 5.3 容斥原理

容斥原理用于计算多个集合并集的大小，解决"至少满足一个条件"的计数问题。

|A₁ ∪ A₂ ∪ ... ∪ Aₙ| = Σ|Aᵢ| - Σ|Aᵢ ∩ Aⱼ| + Σ|Aᵢ ∩ Aⱼ ∩ Aₖ| - ... + (-1)ⁿ⁺¹|A₁ ∩ ... ∩ Aₙ|

直觉理解：先把每个集合的大小加起来（多算了交集部分），再减去两两交集（多减了三重交集），再加上三三交集......交替加减。

**经典应用——错位排列 (Derangement)**：
n 个元素的全排列中，没有任何元素在原来位置上的排列数。

Dₙ = n! × (1 - 1/1! + 1/2! - 1/3! + ... + (-1)ⁿ/n!)

当 n 较大时，Dₙ ≈ n!/e。

### 5.4 递推关系

递推关系定义了序列中每项与前面几项的关系。解递推方程是分析递归算法复杂度的基本工具。

**线性齐次递推**：aₙ = c₁aₙ₋₁ + c₂aₙ₋₂ + ... + cₖaₙ₋ₖ

求解步骤：
1. 写出**特征方程**：xᵏ - c₁xᵏ⁻¹ - c₂xᵏ⁻² - ... - cₖ = 0
2. 求特征根 r₁, r₂, ..., rₖ
3. 若特征根互不相等：通解 aₙ = α₁r₁ⁿ + α₂r₂ⁿ + ... + αₖrₖⁿ
4. 若特征根 r 有重数 m：对应项为 (α₁ + α₂n + ... + αₘnᵐ⁻¹) × rⁿ
5. 用初始条件确定系数 α₁, α₂, ...

**非齐次递推**：通解 = 齐次通解 + 特解。特解的形式取决于非齐次项的形式。

**经典例子——斐波那契数列**：F(n) = F(n-1) + F(n-2)，F(0)=0, F(1)=1
- 特征方程：x² - x - 1 = 0
- 特征根：(1±√5)/2
- 通解（Binet 公式）：F(n) = (φⁿ - ψⁿ) / √5，其中 φ = (1+√5)/2 ≈ 1.618（黄金比例）

### 5.5 生成函数

生成函数是组合数学的强大工具，将序列 {a₀, a₁, a₂, ...} 映射到形式幂级数 A(x) = Σ aₙxⁿ。这样就可以用代数方法（乘法、求导、部分分式分解等）来解决计数和递推问题。

例如，斐波那契数列的生成函数为 F(x) = x / (1 - x - x²)。

### 5.6 鸽巢原理

**鸽巢原理 (Pigeonhole Principle)**：如果将 n+1 个物品放入 n 个容器中，则至少有一个容器包含 ≥ 2 个物品。

**推广形式**：将 n 个物品放入 k 个容器中，至少有一个容器包含 ≥ ⌈n/k⌉ 个物品。

虽然原理简单，但在证明存在性问题时非常有力。

**经典应用**：
- 在 13 个人中，至少有 2 个人生日在同一个月（13 个人放入 12 个月）
- 在任意 5 个整数中，至少有 2 个整数模 4 同余（5 个数放入 4 个余数类）
- 在 n+1 个不超过 2n 的正整数中，一定有两个互素的（连续整数互素）
